# 需求文档

本文档定义 v1.0 之前要交付的功能与非功能需求。

## 1. 背景与目标

### 1.1 背景

当前已有的方案位于 `D:\English-training\skill`，基于 OpenClaw 框架 + Tampermonkey 浏览器用户脚本实现英语口语陪练。该方案依赖较重：

- 需要 OpenClaw 网关常驻
- 语音走 Tampermonkey userscript（必须开着浏览器）
- 框架插件系统带来额外的认知成本

### 1.2 目标

构建一个**独立、轻量、本地运行**的 AI 口语陪练智能体，具备：

- 自己的 AI 界面（不依赖外部框架）
- 长期记忆
- 课程阶段状态机
- 快速上下文检索
- 自动提示词注入
- 语音输入输出

未来在 GitHub 开源，供其他有类似需求的家庭使用。

## 2. 用户与场景

### 2.1 用户（可配置）

用户画像不是写死的，而是通过 `prompts/USER.md`（或未来的 UI 配置面板）在安装 / 首次运行时设定。智能体根据其中的信息调整难度、话题选择、互动风格、课程节奏。

**示例配置 A：中学生小张**（13 岁，初一）

- 学校有英语课，课堂上口语练习机会少
- 性格偏内向，习惯自己练习
- 家长希望他每天有 25-35 分钟英语对话时间
- 每周 3-5 次

**示例配置 B：备考高中生**（17 岁，高二）

- 半年后考雅思，目标分 6.5
- 已有较好基础，但缺乏长篇表达练习
- 每次 45-60 分钟，强度更高

**示例配置 C：成人自学者**（28 岁，程序员）

- 利用通勤碎片时间
- 偏好技术话题（编程、AI、远程工作）
- 每次 15-20 分钟即可

> v1.0 不做多用户 / 多账号（见 §5 范围外），但**单实例的可配置画像**是核心需求。

### 2.2 典型场景

1. **打开应用 → 开始会话**：学生点击"开始今天的练习"，智能体根据记忆挑选一个未讲过的话题，自动进入 WARM_UP 阶段
2. **对话中**：智能体主动引导、提问、控制节奏；学生用文字或语音回复
3. **达到时长 → 自动收尾**：智能体判断"今天差不多了"，做总结、布置小作业、写入会话记录
4. **下次打开**：智能体记得上次聊到哪儿、什么话题讲过、有什么作业没完成

## 3. 功能需求

### 3.1 对话核心

- **F1.1** 用英语进行自然对话，难度匹配 13-14 岁学生水平
- **F1.2** 主动引导：使用开放性问题；连续 3 次短答后切换话题
- **F1.3** 鼓励风格：肯定努力；以引导式纠错为主，不直接打断
- **F1.4** 必要时可使用中文辅助解释（受 `prompts/SOUL.md` 控制）

### 3.2 状态机

- **F2.1** 四阶段：`WARM_UP` (0-5 min) → `MAIN_ACTIVITY` (5-25 min) → `WRAP_UP` (25-30 min) → `END` (30+ min)
- **F2.2** 每个阶段有明确的智能体行为指引
- **F2.3** WRAP_UP / END 阶段自动写入会话总结
- **F2.4** 用户明确表示要停时立即进入收尾
- **F2.5** 跨天自动重置计时器

### 3.3 提示词注入

- **F3.1** 每次 LLM 调用前自动注入 `[System Context]` 块（当前时间、阶段、阶段指引）
- **F3.2** 注入器可插拔，便于后续扩展（记忆片段、相关话题等）

### 3.4 记忆系统

- **F4.1** 长期记忆学生基本信息（姓名、年龄、水平、目标）—— 来自 `prompts/USER.md`
- **F4.2** 短期记忆当前会话上下文（最近 N 轮对话）
- **F4.3** 跨会话记忆每次会话总结、作业、表现
- **F4.4** 话题去重：自动避免重复讲同一话题
- **F4.5** 所有数据本地存储（SQLite + markdown），不联网

### 3.5 快速查询

- **F5.1** 结构化查询：按日期 / 阶段 / 话题类型查会话
- **F5.2** 语义查询：输入"上次讲了什么关于宠物的话题"，返回相关片段（向量检索）

### 3.6 语音 I/O

- **F6.1** 语音输入：学生说话转文字（STT）
- **F6.2** 语音输出：智能体回复转语音播放（TTS）
- **F6.3** 语音队列：支持增量合成、避免长回答卡顿

### 3.7 话题库

- **F7.1** 内置 PETS / FCE 等考试话题模板（从旧项目 `D:\English-training\skill\topic-library\` 迁移）
- **F7.2** 管理员可手动增删话题（编辑 markdown 即可，初期不需要 UI）

### 3.8 会话归档

- **F8.1** 每次会话结束自动生成 `conversation-history.md`（轻量摘要）
- **F8.2** 完整对话记录写入 `teaching-transcript.md`
- **F8.3** 文件命名和格式与旧项目保持兼容（便于迁移历史数据）

### 3.9 UI

- **F9.1** 独立 AI 界面（具体形态：Tauri / Electron / 纯 Web，待定）
- **F9.2** 显示对话历史、当前阶段、计时器
- **F9.3** 一键开始 / 结束会话
- **F9.4** 语音按钮可点可关

## 4. 非功能需求

| 类别 | 要求 |
|---|---|
| 性能 | LLM 首 token 延迟 < 3s（在线时） |
| 隐私 | 学生数据全部本地；API key 仅在 `.env` 中 |
| 稳定性 | 30 分钟以上会话不中断；LLM 错误可降级（不卡死） |
| 可移植 | 学生数据可一键拷贝到另一台机器恢复 |
| 可维护 | 提示词（markdown）与代码分离；改提示词不需要 rebuild |
| 国际化 | UI 中英双语（中文为主） |

## 5. 范围外（v1.0 不做）

- 多学生 / 多账号系统
- 云同步、跨设备同步
- 移动端 App
- 语音发音评分 / 纠音
- 视频、图像能力
- 教师 / 家长后台仪表盘
- 商业化部署、付费功能

## 6. 验收标准（v1.0）

- [ ] 学生能独立完成 30 分钟会话不报错
- [ ] 状态机正确切换 4 个阶段，时间边界 ±1 分钟
- [ ] 跨天后再次打开，智能体能引用上次会话内容
- [ ] 关闭应用 → 重启 → 学生数据完整恢复
- [ ] 删掉 `data/`，应用能正常启动并创建新数据库
- [ ] GitHub README 让新用户 10 分钟内能跑起来

## 7. 风险与约束

| 风险 | 缓解 |
|---|---|
| LLM API 成本 | 优先小 token 模型；v1.x 评估本地小模型 |
| LLM 偶发中断 | 失败重试一次；仍失败则提示并保留会话 |
| 学生误关应用 | 每次对话自动落盘，崩溃后可恢复 |
| 长期记忆膨胀 | v0.x 不处理；v1.x 引入定期总结 |
| 第三方依赖供应链 | 锁版本；CI 跑 build；季度升级 |
