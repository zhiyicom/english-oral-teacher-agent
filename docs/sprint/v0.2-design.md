## 2. 模块设计

### 2.1 `src/config/env.ts` —— 环境变量加载器

**职责**：从 `process.env`（已被 `dotenv/config` 预加载）读取、zod 校验、返回强类型 `Env` 对象。

**接口**：

```ts
import { z } from 'zod';

const EnvSchema = z.object({
  LLM_PROVIDER: z.enum(['minimax']).default('minimax'),
  MINIMAX_API_KEY: z.string().min(1, 'MINIMAX_API_KEY is required'),
  ANTHROPIC_BASE_URL: z.string().url().default('https://api.minimaxi.com/anthropic'),
  LLM_MODEL_MAIN: z.string().default('MiniMax-M3'),
  LLM_MODEL_SUMMARIZER: z.string().default('MiniMax-M3'),
  LLM_TEMPERATURE: z.coerce.number().min(0).max(2).default(0.7),
  LLM_MAX_TOKENS: z.coerce.number().int().positive().default(2048),
  APP_DATA_DIR: z.string().default('./data'),
  APP_LOG_LEVEL: z.enum(['debug', 'info', 'warn', 'error']).default('info'),
  RUN_LIVE_LLM: z.enum(['0', '1']).optional(),
});

export type Env = z.infer<typeof EnvSchema>;

export function loadEnv(): Env {
  const result = EnvSchema.safeParse(process.env);
  if (!result.success) {
    const issues = result.error.issues
      .map(i => `  - ${i.path.join('.')}: ${i.message}`)
      .join('\n');
    throw new Error(`Invalid environment:\n${issues}`);
  }
  return result.data;
}
```

**关键决策**：

- `LLM_PROVIDER` 在 v0.2 限定为 `'minimax'`。这是 zod 枚举，未来加 provider 时放开即可。
- `MINIMAX_API_KEY` 缺则直接抛错（fail-fast），不在 LLM 调用时才报。
- `ANTHROPIC_BASE_URL` 给默认值（即使是 MiniMax 的 URL），方便没配 `.env` 也能跑（但 API key 仍必需）。
- `z.coerce.number()` 让 `.env` 里 `"0.7"` 自动转成 number。

### 2.2 `src/prompts/loader.ts` —— 提示词加载器

**职责**：从 `prompts/` 目录读三个文件（SOUL / AGENTS / USER），返回拼好的 system prompt。

**接口**：

```ts
export interface UserProfile {
  name: string;
  age: number;
  level: 'beginner' | 'intermediate' | 'advanced';
  goals: string[];
  interests: string[];
  // 来自 PRD §1 的其他字段，v0.2 暂只取核心
}

export interface SystemPrompt {
  soul: string;          // 来自 SOUL.md
  agents: string;        // 来自 AGENTS.md
  user: string;          // 来自 USER.md (or USER.md.example)
  userProfile: UserProfile;  // 解析后的 frontmatter
}

export function loadSystemPrompt(dataDir?: string): SystemPrompt;
```

**实现要点**：

- `SOUL.md` / `AGENTS.md`：纯文本，直接 `fs.readFileSync`。
- `USER.md` 优先用 gitignored 的真实文件；不存在则 fallback 到 `USER.md.example`（dev 时用），保证 L1 测试不需要写真实 USER.md。
- `USER.md` 用 `gray-matter` 解析 frontmatter（YAML），body 留作 `user` 字符串。
- 最终拼装：`# SOUL\n\n${soul}\n\n# AGENTS\n\n${agents}\n\n# STUDENT\n\n${user}`。
- LLM client 拿到的是 **string**（拼好的），不知道来源细节。

### 2.3 `src/llm/types.ts` —— LLM 抽象

```ts
export type Role = 'system' | 'user' | 'assistant';

export interface Message {
  role: Role;
  content: string;
}

export interface ChatOpts {
  system: string;             // 拼好的 system prompt
  messages: Message[];        // 已有对话历史
  temperature?: number;
  maxTokens?: number;
}

export interface ChatChunk {
  type: 'text' | 'thinking';
  delta: string;
}

export interface ChatResult {
  content: string;            // 拼好的最终回复（不含 thinking）
  thinking?: string;          // 可选：thinking 内容（v0.2 不写文件）
  usage?: { inputTokens: number; outputTokens: number };
}

export interface LLMClient {
  /** 流式（v0.2 CLI 用流式） */
  chatStream(opts: ChatOpts): AsyncIterable<ChatChunk>;
  /** 非流式（测试、Summarizer 用） */
  chat(opts: ChatOpts): Promise<ChatResult>;
}
```

**关键决策**：

- CLI 用 `chatStream`（边生成边打印，体验更好）。
- 测试用 `chat`（断言更简单）。
- `ChatResult.content` **不包含** thinking —— caller 不会看到 thinking。thinking 单独字段留作未来 debug 用。

### 2.4 `src/llm/anthropic.ts` —— Anthropic 协议 Provider

**职责**：用 `@anthropic-ai/sdk` 调 MiniMax（协议兼容），把我们的 `Message` 转成 Anthropic 格式，处理 `thinking` 块。

**接口**：

```ts
export function createAnthropicProvider(env: Env): LLMClient;
```

**关键代码**（伪）：

```ts
import Anthropic from '@anthropic-ai/sdk';

export function createAnthropicProvider(env: Env): LLMClient {
  const client = new Anthropic({
    apiKey: env.MINIMAX_API_KEY,
    baseURL: env.ANTHROPIC_BASE_URL,
  });

  async function* chatStream(opts: ChatOpts): AsyncIterable<ChatChunk> {
    const stream = client.messages.stream({
      model: env.LLM_MODEL_MAIN,
      max_tokens: opts.maxTokens ?? env.LLM_MAX_TOKENS,
      temperature: opts.temperature ?? env.LLM_TEMPERATURE,
      system: opts.system,
      messages: opts.messages.map(m => ({
        role: m.role === 'system' ? 'user' : m.role,  // 不会发生
        content: m.content,
      })),
    });

    for await (const event of stream) {
      if (event.type === 'content_block_delta') {
        if (event.delta.type === 'text_delta') {
          yield { type: 'text', delta: event.delta.text };
        } else if (event.delta.type === 'thinking_delta') {
          yield { type: 'thinking', delta: event.delta.thinking };
        }
      }
    }
  }

  async function chat(opts: ChatOpts): Promise<ChatResult> {
    let text = '';
    let thinking = '';
    for await (const chunk of chatStream(opts)) {
      if (chunk.type === 'text') text += chunk.delta;
      else thinking += chunk.delta;
    }
    return { content: text, thinking: thinking || undefined };
  }

  return { chatStream, chat };
}
```

**关键决策**：

- 用 `messages.stream` 而非 `messages.create`，CLI 流式更友好。
- `thinking_delta` 在 Anthropic SDK 里实际叫 `thinking_delta`（v0.2 实装时以实际 SDK 文档为准，必要时调）。
- `baseURL` 走 `env.ANTHROPIC_BASE_URL`，让 MiniMax 路径走自定义 endpoint。
- 不实现工具调用、prompt caching、PDF/image —— v0.2 范围外。

### 2.5 `src/llm/testing.ts` —— Replay Provider

**职责**：CI 不打外网，从 JSON fixture 读响应，模拟流式 chunk。

**接口**：

```ts
export function createReplayProvider(fixturesDir: string): LLMClient;
```

**Fixture 格式**（`tests/fixtures/replay/greeting.json`）：

```json
{
  "name": "greeting",
  "match": {
    "userSaysContains": "hi"
  },
  "chunks": [
    { "type": "thinking", "delta": "The user is greeting me. " },
    { "type": "thinking", "delta": "I should respond warmly." },
    { "type": "text", "delta": "Hi there! " },
    { "type": "text", "delta": "How are you today?" }
  ],
  "result": {
    "content": "Hi there! How are you today?",
    "thinking": "The user is greeting me. I should respond warmly."
  }
}
```

**匹配策略**：

- 遍历 `fixturesDir/*.json`，找第一个 `match.userSaysContains` 子串匹配 user 最后一条消息的 fixture。
- 没匹配上 → 抛错（避免 L3 测试"沉默通过"）。
- 匹配上 → 按 chunks 顺序 yield（每 5ms 一个，模拟真实流式），最后返回 `result`。

**为什么不直接 mock SDK**：

- mock SDK = mock 一个第三方库，未来 SDK 升级会失配。
- Fixture = 业务级断言（"看到 hi 应该有 greeting 回复"），更稳定、更可读。

### 2.6 `src/cli.ts` —— REPL

**职责**：readline 循环，输入消息 → 调 LLM → 打印回复。

**伪代码**：

```ts
import { createInterface } from 'node:readline';
import { loadEnv } from './config/env.js';
import { loadSystemPrompt } from './prompts/loader.js';
import { createAnthropicProvider } from './llm/anthropic.js';
import { createReplayProvider } from './llm/testing.js';

async function main() {
  const env = loadEnv();
  const systemPrompt = loadSystemPrompt();
  const client = process.env.RUN_LIVE_LLM === '1'
    ? createAnthropicProvider(env)
    : createReplayProvider('tests/fixtures/replay');

  const history: Message[] = [];
  const rl = createInterface({ input: process.stdin, output: process.stdout });

  console.log('English Oral Teacher Agent — CLI mode');
  console.log(`Persona: ${systemPrompt.userProfile.name} (age ${systemPrompt.userProfile.age})`);
  console.log('Type "exit" to quit.\n');

  const ask = (q: string) => new Promise<string>(r => rl.question(q, r));

  while (true) {
    const input = (await ask('> ')).trim();
    if (input === 'exit' || input === '') break;
    history.push({ role: 'user', content: input });

    process.stdout.write('[Teacher]: ');
    let response = '';
    for await (const chunk of client.chatStream({
      system: JSON.stringify(systemPrompt),  // 或拼好的 string
      messages: history,
    })) {
      if (chunk.type === 'text') {
        process.stdout.write(chunk.delta);
        response += chunk.delta;
      }
      // thinking 不打 stdout
    }
    process.stdout.write('\n\n');
    history.push({ role: 'assistant', content: response });
  }
  rl.close();
}

main().catch(err => {
  console.error(err);
  process.exit(1);
});
```

**关键决策**：

- `system` 字段传拼好的 string（不是 JSON），更简单。`prompts/loader.ts` 直接返回 string 也行 —— 看实现方便。
- 思考不打印，但保留在变量里（v0.2 不写文件）。
- `exit` 退出时不持久化（v0.3 才加 SQLite）。
- `Ctrl+D`（EOF）走 readline 的 `close` 事件，同样能退出。

import { describe, it, expect } from 'vitest';
import { spawn } from 'node:child_process';
import path from 'node:path';

describe('CLI REPL (Replay mode)', () => {
  it('responds to "hi" with greeting', async () => {
    const cli = path.resolve('src/cli.ts');
    const child = spawn('node', ['--import', 'tsx', cli], {
      env: { ...process.env, RUN_LIVE_LLM: '0', MINIMAX_API_KEY: 'sk-test' },

    let stdout = '';
    child.stdout.on('data', d => stdout += d.toString());

    child.stdin.write('hi\n');
    await new Promise(r => setTimeout(r, 500));

## 5. 错误处理

| 场景 | 行为 |
|---|---|
| `MINIMAX_API_KEY` 缺失 | `loadEnv` 抛错，CLI 立即退出 1 |
| `.env` 文件不存在 | 用 `process.env`，缺关键变量再抛错 |
| API 401/403 | Anthropic SDK 抛 `AuthenticationError`，CLI 打印错误退出 |
| API 429 | 不重试（v0.7 才加），打印错误退出 |
| `thinking_delta` 字段不存在于 SDK | 静默忽略（不抛错），只 yield text |
| Replay fixture 不匹配 user 输入 | 抛错（防止沉默通过） |

## 8. 风险与对策（设计层）

| 风险 | 对策 |
|---|---|
| `@anthropic-ai/sdk` 升级改 streaming event 字段 | 锁版本 `^0.40.0`（定版时选稳定版）；L1 测试 spy 守住关键字段 |
| MiniMax-M3 实际不支持 `thinking` 块 | 静默忽略（不抛错），只取 text；Live 验证时人工确认 |
| `tsx` 在 Windows spawn 失败 | L3 测试用 `node --import tsx` 而非 `tsx` 命令 |
| Replay fixture 跟 Live 行为漂移 | 每次 Live 验证后人工更新 fixture；CI 只信 fixture |
| `process.env` 跨测试污染 | 每个 L1 test `beforeEach` 重置关键 env 变量 |
