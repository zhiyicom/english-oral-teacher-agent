## 6. CLI 改造

`src/cli.ts` 从"纯 readline 循环"扩成"读 + 落盘"：

| 时机 | 行为 |
|---|---|
| 启动 | `openDb({ dataDir })` → `applyMigrations(db)` → `sessions.create()` 建一条新 session |
| 每个 user turn | `messages.append({ sessionId, role: 'user', content })` |
| 每个 assistant turn | `messages.append({ sessionId, role: 'assistant', content })` |
| 退出（EOF / SIGINT / 用户输入 exit）| `sessions.markEnded(sessionId)` → `db.close()` |

### 6.1 readline bug 修复（DoD #5 调试发现）

Live 验证时第一轮 LLM 调用（~2-4s）期间，测试 driver 已经把 5 行输入全部灌入 stdin。readline 把这 5 个 `'line'` 事件都 emit 了，但 `rl.question()` 的 callback 在第 1 个事件返回后就 resolve，后续 4 个事件"丢失"了。结果是 turn 1 后 CLI 跳到第 4 行（"i built castles and farms"），turns 2、3、5 凭空消失。

**根因**：readline `'line'` 事件是 fire-and-forget，事件触发时若没有 listener，就永久丢失。

**修复**：在 `src/cli.ts` 加了 `createLineReader()`——一次性挂 `'line'` 和 `'close'` 监听，push 进 queue；ask() 先从 queue 取，没有就等 resolver。这样所有进入 stdin 的行都被收下，consumer 准备好就立刻拿到下一行。

修复后 Live 验证 5 轮全部落盘（见 §7.1）。

## 8. 风险与遗留问题

| 风险 / 遗留 | 影响 | 处置 |
|---|---|---|
| **readline `'line'` 事件丢失** | Live 慢 LLM 时丢 turn（v0.3 调试发现） | 已用 `createLineReader` queue 方案修好（见 §6.1） |
| **`better-sqlite3` Windows 编译** | pnpm 11 + cmd.exe PATH 找不到 `node`，prebuild 脚本跑不起来 | `.npmrc` 写 `allow-builds[]=better-sqlite3`；手动用绝对路径 `node` 跑 `prebuild-install`（已在 [[feedback-better-sqlite3-windows-build]] 记录） |
| **WAL 模式下 `rmSync` 偶发 EPERM** | `tests/storage/integrity.test.ts` cleanup 偶发 | `safeRm()` helper 重试 3 次，最终失败只 warn 不 fail；测试本身不影响 |
| **L3 测试用 `setTimeout` 间隔** | 行间 700ms 在慢 CI 上可能 flaky | 已改为"等 stdout 出现第 N 个 `[Teacher]` 后再 end stdin"，对 LLM 速度不再敏感 |
| **L3 测试用 `setTimeout` 启动延迟** | 200ms 是经验值，太短可能子进程还没挂监听 | 200ms 在本机稳；如 CI 慢再调 |
| **Migrations 目录定位** | 测试用 `resolveMigrationsDirForTesting()` 走 `<repo>/src/storage/migrations`，跟生产代码一致 | OK，测试与生产共源码 |
| **DB schema 与 PRD v0.5 词汇/错例表** | 本次只用了 core 列；其它列已建表占位但未消费 | OK，v0.5 时直接填 |
| **Live LLM 流式 thinking 块** | 本次 5 轮全无 extended thinking；如未来需要，CLI 已正确分流 `text` vs 非 `text` chunk | 不需改 |
| **`sessions.markEnded` 计时** | `duration_min` 是 `Math.floor` 的分钟数，5 轮对话总是 0 | OK，v0.3 不做分钟级精度；v0.4 看是否要 second 级 |
