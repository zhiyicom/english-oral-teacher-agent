## 2. 数据结构

### 2.1 `Phase` 枚举

```ts
export type Phase = 'WARM_UP' | 'MAIN_ACTIVITY' | 'WRAP_UP' | 'END'
```

### 2.2 `PhaseTransition` 记录

```ts
export interface PhaseTransition {
  phase: Phase         // 进入的新 phase
  at: number           // 自 session 开始的分钟数（1 位小数精度）
  reason?: 'time' | 'silence' | 'user_stop'  // 为什么切换（debug 用）
}
```

### 2.3 `SessionState` 内部态

```ts
export interface SessionState {
  phase: Phase
  startedAt: number       // ms epoch（clock.now()）
  lastUserMsgAt: number   // ms epoch
  // —— 以下为派生，每次 tick 重算 ——
  elapsedMin: number      // (now - startedAt) / 60_000
  silenceMin: number      // (now - lastUserMsgAt) / 60_000
  lastTransitionAt: number // 上次 phase 切换时的 elapsedMin（用于 "Last transition" 显示）
}
```

> `elapsedMin` / `silenceMin` / `lastTransitionAt` 存进 state（而非每次现算）——`applyEvent` 返回新 state 时一并更新，调用方拿到的永远是"快照"。

### 2.4 `Clock` 接口

```ts
export interface Clock {
  now(): number  // ms epoch
}

export const realClock: Clock = {
  now: () => Date.now(),
}

export function mockClock(initialNow: number): Clock {
  let now = initialNow
  return {
    now: () => now,
    set: (t: number) => { now = t },
    advance: (ms: number) => { now += ms },
  }
}
```

> `mockClock` 额外暴露 `set` / `advance` 是给测试用；生产只用 `realClock`。

### 2.5 `Event` 联合

```ts
export type Event =
  | { type: 'TICK' }                  // 每 turn 开头（CLI 主动算 elapsed/silence）
  | { type: 'USER_MSG' }              // 用户发了消息
  | { type: 'USER_STOP' }             // 用户说 stop（命中关键词）
  | { type: 'INIT'; now: number }     // 初始化（CLI 启动时一次）
```

`INIT` 让 state machine 纯函数化：state 起步只靠 `now` 算 elapsed，不调 clock。

## 3. 模块设计

### 3.1 `src/agent/clock.ts`

**职责**：定义 `Clock` 接口 + 3 个工厂。

**接口**：

```ts
export interface Clock {
  now(): number
}

export const realClock: Clock  // 用 Date.now()
export function mockClock(initial: number): MockClock  // 可 set/advance
```

**决策**：

- `MockClock` 在 test 文件里**显式 import**，不在 production 用
- `realClock` 是**单例**，不传 clock 的地方默认用它（虽然 `applyEvent` 强制要 clock，避免隐式全局）

### 3.2 `src/agent/state-machine.ts`

**职责**：纯函数计算 phase + 触发 transition。

**接口**：

```ts
export function getPhase(elapsedMin: number): Phase
export function initState(now: number): SessionState
export function applyEvent(state: SessionState, event: Event, clock: Clock): SessionState
```

**`getPhase` 实现**：

```ts
const BOUNDARIES: Array<[number, Phase]> = [
  [0, 'WARM_UP'],
  [5, 'MAIN_ACTIVITY'],
  [25, 'WRAP_UP'],
  [30, 'END'],
]

export function getPhase(elapsedMin: number): Phase {
  if (elapsedMin < 5) return 'WARM_UP'
  if (elapsedMin < 25) return 'MAIN_ACTIVITY'
  if (elapsedMin < 30) return 'WRAP_UP'
  return 'END'
}
```

**`applyEvent` 实现**（model C：silence 只算不切）：

```ts
export function applyEvent(state: SessionState, event: Event, clock: Clock): SessionState {
  const now = clock.now()
  const elapsedMin = (now - state.startedAt) / 60_000
  const silenceMin = (now - state.lastUserMsgAt) / 60_000

  switch (event.type) {
    case 'INIT': {
      return {
        phase: 'WARM_UP',
        startedAt: event.now,
        lastUserMsgAt: event.now,
        elapsedMin: 0,
        silenceMin: 0,
        lastTransitionAt: 0,
      }
    }

    case 'TICK': {
      // model C：phase 切换**仅**依赖时间。silence 仍计算（塞进 state 给 [System Context] 看），但不触发切 phase。
      const nextPhase = getPhase(elapsedMin)

      const transitioned = nextPhase !== state.phase
      return {
        ...state,
        phase: nextPhase,
        elapsedMin,
        silenceMin,
        lastTransitionAt: transitioned ? elapsedMin : state.lastTransitionAt,
      }
    }

    case 'USER_MSG': {
      return {
        ...state,
        lastUserMsgAt: now,
        silenceMin: 0,  // 重置 silence（给 LLM 看的 hint 也重置）
      }
    }

    case 'USER_STOP': {
      // 唯一绕过 time 边界的 phase 切换。USER_STOP 立即到 END。
      return {
        ...state,
        phase: 'END',
        elapsedMin,
        silenceMin,
        lastTransitionAt: elapsedMin,
      }
    }
  }
}
```

> **不变性**：`applyEvent` 是纯函数——给定相同 (state, event, now) 必返回相同新 state，不修改入参。
>
> **model C 关键不变量**：state.phase 永远等于 `getPhase(elapsedMin)`，除非发生过 `USER_STOP`。无 `phase: 'WRAP_UP', elapsedMin: 0` 这种不一致态。

**违规转换检测**：

`applyEvent` 接受**任意** `state.phase` 入参，但**TICK 总是按 time 算**——你喂 state.phase='WARM_UP' 但 elapsed=100 进来，下一 tick 立即被 getPhase 拽回 END。**不需要在 `applyEvent` 内对 state.phase 做校验**。

但是 plan §5.1 L1 #16 要求"状态转换不合法时报错"——这里指**手工 transition**（非 TICK），比如 CLI 想要"强制跳到 WRAP_UP"（v0.4 不需要，但预留接口）。提供独立 helper `validatePhaseTransition(from, to)` 给上层用。

```ts
// 允许的转换（time-based 顺序，**不**含 silence skip —— model C 删了）
const ALLOWED: Record<Phase, Phase[]> = {
  WARM_UP: ['MAIN_ACTIVITY', 'WRAP_UP', 'END'],  // 显式 jump 仍允许（admin command 之类）
  MAIN_ACTIVITY: ['WRAP_UP', 'END'],
  WRAP_UP: ['END'],
  END: [],  // END 是终态
}

export function validatePhaseTransition(from: Phase, to: Phase): void {
  if (from === to) return  // 同 phase 合法（self-loop）
  if (!ALLOWED[from].includes(to)) {
    throw new Error(`Invalid phase transition: ${from} → ${to}`)
  }
}
```

> L1 #16 调用 `validatePhaseTransition('WARM_UP', 'WRAP_UP')` 应通过；`validatePhaseTransition('END', 'WARM_UP')` 应抛错（END 是终态）；`validatePhaseTransition('MAIN_ACTIVITY', 'WARM_UP')` 应抛错（不能后退）。

### 3.3 `src/agent/context-injector.ts`

**职责**：把 `SessionState` 拼成 `[System Context]` 字符串。

**接口**：

```ts
export function buildSystemContext(state: SessionState): string
```

**实现**：

```ts
export function buildSystemContext(state: SessionState): string {
  return [
    '[System Context]',
    `- Phase: ${state.phase}`,
    `- Elapsed: ${state.elapsedMin.toFixed(1)} min`,
    `- Silence: ${state.silenceMin.toFixed(1)} min`,
    `- Last transition: ${(state.elapsedMin - state.lastTransitionAt).toFixed(1)} min ago (entered ${state.phase})`,
  ].join('\n')
}
```

输出示例：

```
[System Context]
- Phase: WARM_UP
- Elapsed: 2.3 min
- Silence: 0.5 min
- Last transition: 2.3 min ago (entered WARM_UP)
```

### 4.2 每 turn 流程

```
┌─ while (true) ─────────────────────────────────────────────┐
│  raw = await reader.next()                                │
│  if (raw === '') break   // EOF                           │
│                                                             │
│  // 1. tick（算 elapsed / silence / 可能 phase 切换）       │
│  const prevPhase = state.phase                             │
│  state = applyEvent(state, { type: 'TICK' }, clock)        │
│                                                             │
│  // 2. 写 phase_history（如果切换了）                       │
│  if (state.phase !== prevPhase && state.phase === 'END') { │
│    // END 切换，记一条                                     │
│    phaseHistory.push({                                     │
│      phase: 'END',                                         │
│      at: state.elapsedMin,                                 │
│      reason: '<推断>',                                     │
│    })                                                      │
│  }                                                         │
│                                                             │
│  // 3. 如果 phase 是 END，直接退出                          │
│  if (state.phase === 'END') {                              │
│    console.log('[Session ended: <reason>]')                │
│    break                                                    │
│  }                                                         │
│                                                             │
│  // 4. 处理用户输入                                         │
│  const input = raw.trim()                                  │
│  if (input === '' || input === 'exit') break              │
│                                                             │
│  // 5. stop 关键词检测                                      │
│  const isStop = STOP_REGEX.test(input)                     │
│  if (isStop) {                                             │
│    state = applyEvent(state, { type: 'USER_STOP' }, clock) │
│    phaseHistory.push({ phase: 'END', at: state.elapsedMin, │
│                          reason: 'user_stop' })             │
│  } else {                                                  │
│    state = applyEvent(state, { type: 'USER_MSG' }, clock)  │
│  }                                                         │
│                                                             │
│  // 6. 拼 system prompt                                     │
│  const system = buildFinalSystem(systemPrompt, state)      │
│                                                             │
│  // 7. 调 LLM（即使 stop 也调，让 agent 说再见）             │
│  ... 调 client.chatStream({ system, messages: history })   │
│                                                             │
│  // 8. append user + assistant 消息到 history + DB          │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 Stop 关键词正则

```ts
const STOP_REGEX = /(?:^|\s)(stop|quit|end|bye|done|结束|停)(?:$|[!.?,\s])/i
```

**测试用例**：

| 输入 | 匹配？ | 理由 |
|---|---|---|
| `stop` | ✅ | 单词 |
| `STOP` | ✅ | 大写不敏感 |
| `stop.` | ✅ | 句号结尾 |
| `okay stop, thanks` | ✅ | 句中，前后是空格/逗号 |
| `nonstop` | ❌ | 不是整词（前面是字母） |
| `I don't want to stop` | ❌ | 前面是空格但 "stop" 后面是 ` `, 但是前面是 `to ` → 实际是 `to stop` → 看正则是 `(?:^|\s)(stop)...(?:$|[!.?\s])` → 前面是空格 + stop 后面是字符串末尾？不对，"stop" 后是句号。"I don't want to stop." → stop 后是 `.` → 匹配！**这是误杀** |
| `let's stop and continue` | ❌ | 后面是 ` `，是 `[\s]` → 实际会匹配（这正是 plan #11 要求的"不应误杀"！） |

⚠️ **plan §5.1 L1 #11 明确要求**：`let's stop and continue` **不**应被识别为 stop。

需要更严格的正则：仅当 stop 是**完整短句**时才算。

**改进版**：

```ts
// stop 必须是整句（句首或前面是句末标点）
const STOP_REGEX = /(?:^|[.!?]\s+|\n)(stop|quit|end|bye|done|结束|停)\b[.!?\s]*$/i
```

- `(?:^|[.!?]\s+|\n)` —— 句首 / 句末标点后 / 换行
- `(stop|quit|end|bye|done|结束|停)` —— 关键词
- `\b` —— 词边界
- `[.!?\s]*$` —— 关键词后只有句末标点 / 空白，到字符串末尾

**测试**：

| 输入 | 匹配？ |
|---|---|
| `stop` | ✅ |
| `stop!` | ✅ |
| `ok stop` | ❌（前面是空格不是句末标点） |
| `let's stop and continue` | ❌（stop 后面不是末尾） |
| `I don't want to stop.` | ❌（stop 后面是 `.`，但 stop 前面是空格不是句末标点） |
| `okay. stop` | ✅（前面是 `. `） |
| `goodbye` | ❌（"bye" 不在列表） |

> 但这样**降低召回**——用户说"okay stop" 不会被识别。trade-off: 取 plan 指定的精确性优先。

### 4.5 markEnded 扩展（`src/storage/sessions.ts`）

```ts
interface MarkEndedOptions {
  endedAt?: Date
  durationMin?: number
  phaseHistory?: PhaseTransition[]  // ← 新增
  reason?: string                    // ← 新增
}

function markEnded(id: string, options: MarkEndedOptions = {}): Session {
  const endedAt = options.endedAt ?? new Date()
  const startedAt = get(id)?.started_at ?? endedAt
  const durationMin = options.durationMin ?? computeDurationMin(startedAt, endedAt)

  db.prepare(`
    UPDATE sessions
    SET ended_at = ?,
        duration_min = ?,
        phase_history = ?,
        summary = COALESCE(?, summary)  -- 预留 v0.5
    WHERE id = ?
  `).run(
    endedAt.toISOString(),
    durationMin,
    options.phaseHistory ? JSON.stringify(options.phaseHistory) : null,
    options.reason ?? null,
    id,
  )

  return get(id)!
}
```

> 不改 schema；只填已有 `phase_history` 列。

## 7. 决策记录

| 决策 | 替代方案 | 为什么选这个 |
|---|---|---|
| `Clock` 接口注入而非 `Date.now()` 全局 | 直接调 `Date.now()` | 测可控；MOCK_TIME demo 不用 patch global |
| Time boundaries: 5/25/30 min | 4/24/30 / 3/20/30 / 6/30/... | plan 测试断言 (0/4/6/24/26/29/30/31) 唯一匹配这组 |
| **model C: silence 不切 phase，只作 [System Context] hint** | model A: silence 2/5/10 min 切 phase | 13 岁小孩沉默多在思考，state machine 硬切 phase 会打断；让 LLM 看到 silence 数字自己决定怎么回应（更聪明） |
| Stop 关键词: 严格（整句） | 宽松（任意出现） | plan 明确要求 "let's stop and continue" 不命中 |
| `phaseHistory` 存为 JSON TEXT | 新建 `phase_transitions` 表 | v0.3 已建 TEXT 列；30 min session 只有 3-4 条切换，JSON 足够 |
| `applyEvent` 纯函数 + 强制传 clock | 单例 state 对象 + 全局 clock | 可测试；race-free |
| `validatePhaseTransition` 作为独立 helper | 在 applyEvent 内报错 | 区分"自动算 phase"（time-based 不报错）和"手工 transition"（需要校验） |
| END 不发 LLM，直接 break | END 也发"goodbye"消息 | "stop" 事件走 USER_STOP + 仍发 LLM；自然 END（30 min）不调 LLM，省钱省时 |
| 中文关键词支持 | 仅英文 | 用户群体是 13 岁小孩，混合使用常见；正则简单加 2 个字符 |
| state.phase 永远等于 `getPhase(elapsedMin)`（除 USER_STOP） | 允许 phase 任意跳 | model C 的不变量：time 是唯一自动驱动；规则简单、可预测 |
