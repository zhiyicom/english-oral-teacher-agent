## 1. Sprint 目标

**用一句话**：30 分钟 session 里有结构——4 个 phase（WARM_UP / MAIN_ACTIVITY / WRAP_UP / END）按时切换，每轮 LLM 调用都能看到当前 phase + elapsed + silence 的 [System Context] 块，并能被"用户说停"或"沉默太久"提前触发。

**为什么是 v0.4**：

- v0.3 把消息落盘了，但 session 没有"形状"——一个 30 分钟的课和一个 3 分钟的闲聊写出来是一模一样的两条消息
- 没有 phase，agent 就不会**主动收尾**（WRAP_UP）、不会**提前结束**（silence kill switch）、也不会**让用户说停就停**（stop override）
- v0.5 要做"上次回顾摘要"作为开场白，依赖 v0.4 的 phase 切换 + 落盘 phase_history
- v0.6 的 summarizer 需要 phase_history 知道"什么时候真正 END"

**Demo 形态**：

```bash
$ MOCK_TIME=true pnpm dev
# 0 分钟：WARM_UP
> hi
[System Context] Phase: WARM_UP | Elapsed: 0.0 min | Silence: 0.0 min
[Teacher]: Hi there! How are you today?

# 6 分钟（mock）：MAIN_ACTIVITY
> i built a castle
[System Context] Phase: MAIN_ACTIVITY | Elapsed: 6.0 min | Silence: 0.5 min
[Teacher]: Castles are cool! Do you build in survival or creative?

# 26 分钟（mock）：WRAP_UP
> ok bye
[System Context] Phase: WRAP_UP | Elapsed: 26.0 min | Silence: 0.3 min
[Teacher]: Great chat today! See you next time. 👋

# 退出后查 DB
$ sqlite3 data/oral-teacher.db "SELECT phase_history FROM sessions;"
[{"phase":"WARM_UP","at":0},{"phase":"MAIN_ACTIVITY","at":5},{"phase":"WRAP_UP","at":25},{"phase":"END","at":30}]
```

### 2.2 Phase 边界（时间阈值，决策结果）

| Phase | elapsedMin 范围 | 备注 |
|---|---|---|
| `WARM_UP` | `[0, 5)` | 开场闲聊 |
| `MAIN_ACTIVITY` | `[5, 25)` | 主体（讲主题 / 练口语） |
| `WRAP_UP` | `[25, 30)` | 收尾总结 |
| `END` | `≥ 30` | 结束 |

测试断言（plan §v0.4 指定）：

```
getPhase(0)  === 'WARM_UP'
getPhase(4)  === 'WARM_UP'
getPhase(6)  === 'MAIN_ACTIVITY'
getPhase(24) === 'MAIN_ACTIVITY'
getPhase(26) === 'WRAP_UP'
getPhase(29) === 'WRAP_UP'
getPhase(30) === 'END'
getPhase(31) === 'END'
```

### 2.3 边界检测（提前触发，提前于时间阈值）

| 触发条件 | 行为 | 设计动机 |
|---|---|---|
| 用户说 `stop` / `quit` / `end` / `bye` / `done` / `结束` / `停` | 立即 → `END` | 用户主动收尾 |
| `WARM_UP` 阶段沉默 ≥ 2 分钟 | → `WRAP_UP` | 孩子开场没话聊，别硬撑 |
| `MAIN_ACTIVITY` 阶段沉默 ≥ 5 分钟 | → `WRAP_UP` | 主题卡住，主动收 |
| `WRAP_UP` 阶段沉默 ≥ 2 分钟 | → `END` | 收尾说完了就走 |
| **任意阶段**沉默 ≥ 10 分钟 | → `END` | kill switch（用户走了 / 卡死） |

> 关键词匹配走**大小写不敏感 + 整词匹配**：`/(?:^|\s)(stop|quit|end|bye|done|结束|停)(?:$|[!.?\s])/i`。中文关键词同处理。
>
> "沉默"定义为 `now - lastUserMsgAt`，不包含 assistant 消息。

### 2.4 Schema 影响（最小）

v0.3 已经建好 `sessions.phase_history TEXT` 字段。v0.4 **不创建新表**，只：

- 在 `markEnded(sessionId, { phaseHistory })` 入参里加 `phaseHistory: Array<{ phase: Phase; at: number }>`
- 写入时 `JSON.stringify()` 存到 `phase_history` 列
- `sessions.create()` 不变

### 2.5 [System Context] 块格式

每轮 LLM 调用前，在 `system` 字符串末尾追加：

```
[System Context]
- Phase: WARM_UP
- Elapsed: 2.3 min
- Silence: 0.5 min
- Last transition: 0.0 min ago (entered WARM_UP)
```

字段：
- `Phase` —— 当前 phase
- `Elapsed` —— 从 session 开始到现在的分钟数（1 位小数）
- `Silence` —— 从用户最后一条消息到现在的分钟数
- `Last transition` —— 上次 phase 切换到现在的分钟数（agent 可感知"刚进 MAIN"还是"快 END 了"）

> 块用 `buildSystemContext(state)` 拼，state 由 state machine 提供；CLI 不手工拼字符串。

### 2.6 Time source

- **生产**：`realClock` 用 `Date.now()`，注入到 `applyEvent(state, event, clock)`。
- **测试**：`mockClock(now)` 工厂 + vitest 的 `vi.useFakeTimers()` + `vi.setSystemTime(now)`。State machine **不直接调** `Date.now()`，保证可测。
- **手动 demo**：`MOCK_TIME=true` 环境变量启用 CLI 内的 mock clock，启动时从 `MOCK_NOW` 读 ISO 时间戳（默认 `2026-01-01T00:00:00Z`），后续每轮 +1 分钟——5 轮对话走完正好 5 分钟。

### 2.7 CLI 行为

- 启动：建 session → state = `{ phase: 'WARM_UP', startedAt: now, lastUserMsgAt: now }`
- 每 turn 之前：
  1. 调 `applyEvent(state, { type: 'TICK' }, clock)` → 算 elapsed/silence/phase
  2. 若 phase 从非 END → END：**break loop**（不再要输入，直接打印 `[Session ended: <reason>]` + markEnded）
  3. 若 phase 在本 turn 切换了：把旧 phase 入 `phaseHistory` 数组
  4. 拼 `buildSystemContext(state)` → 拼到 system prompt
- 用户发消息时：
  1. 先发 stop 关键词检测；若命中 → 走 stop 事件，**仍然把消息送到 LLM** 让 agent 说再见，下一轮 break
  2. 正常消息 → 调 `applyEvent(state, { type: 'USER_MSG' }, clock)`（重置 `lastUserMsgAt`）
- 退出（exit / SIGINT / EOF）：`markEnded({ phaseHistory, reason: 'user_exit' | 'phase_end' | 'silence_kill' })`

| 性能优化（高频 tick） | 量起来再说；30 分钟 1 tick 足够 |

| 项 | 决定 |

## 6. 风险与回退

| 风险 | 影响 | 应对 |
|---|---|---|
| **Stop 关键词误杀** | "I don't want to stop" 被识别为 stop | 整词 + 边界匹配（前后是 `^/$/[.!?\s]`），且只对**完整短语**触发（不命中 = noop） |
| **时间 mock 漏一处** | state machine 在某处直接 `Date.now()` 而非 clock.now() | L1 测试用 vi.spyOn(Date, 'now') 验证 `realClock` 不会调 Date.now（已隔离） |
| **Phase 切换导致 system prompt 变化太大** | LLM 措辞前后差异大 | [System Context] 块是**追加**到 system prompt 末尾，不改 SOUL/AGENTS/USER 内容 |
| **CLI break 早于 LLM 收到 stop** | 用户说 stop 没听到回复就被断开 | stop 事件**仍把当前 turn 走完**（送 LLM + 收响应），下轮才 break |
| **Live 5 轮不够跨 phase** | 主验证看不到 phase 切换 | L3 用 mock 走 6 轮（强制进入 MAIN_ACTIVITY），Live 是补充 |
| **30 分钟阈值在小学生场景不实用** | 真实场景是 5-10 分钟 | 阈值做成常量，sprint 末看反馈再调；v0.4 暂写死 |
| **Silence 计算精度** | `now - lastUserMsgAt` 用 ms，60s 内精度足够 | OK |
| **多轮 silence 累积** | 第一次用户说完沉默 5 min → WRAP_UP；之后用户又回一句 → 应该重置 silence | USER_MSG 事件重置 `lastUserMsgAt` |
