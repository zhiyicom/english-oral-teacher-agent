## 6. 架构与代码改动

### 6.1 新增 `src/agent/` 子树

| 文件 | 行数 | 职责 |
|---|---|---|
| `src/agent/clock.ts` | ~30 | `Clock` 接口（`now()`）+ `realClock`（包 `Date.now()`）+ `mockClock(initial)` 工厂（带 `set` / `advance`） |
| `src/agent/state-machine.ts` | ~150 | `Phase` / `Event` / `SessionState` / `PhaseTransition` 类型 + `getPhase(elapsedMin)` + `initState(now)` + `applyEvent(state, event, clock)` + `validatePhaseTransition(from, to)` |
| `src/agent/context-injector.ts` | ~40 | `buildSystemContext(state)` 拼 `[System Context]` 字符串 |
| `src/agent/prompt-builder.ts` | ~25 | `buildFinalSystem(prompt, state)` 拼 SOUL+AGENTS+USER + `[System Context]` |
| `src/agent/index.ts` | ~10 | barrel 导出 |

**model C 体现**：state machine 的 `applyEvent(TICK)` 只算 time-based phase；silence 仅计算并显示，**不触发** phase 切换。USER_STOP 是唯一的非时间触发。

### 6.2 `src/storage/sessions.ts` 扩展

| 字段 | 变化 |
|---|---|
| `Session.phase_history` | 已有（v0.3 schema） |
| `MarkEndedInput.phaseHistory?: PhaseTransition[]` | ✅ 新增 |
| `MarkEndedInput.reason?: string` | ✅ 新增（虽然未在 Session 表里持久化 reason 列，但 L3 测试通过 phase_history 最后一项的 reason 字段验证） |
| `markEnded` 行为 | 把 `phaseHistory` JSON 化后写 `phase_history` 列；用 `COALESCE(?, phase_history)` 保证不传时不覆盖已有值 |

### 6.3 `src/cli.ts` 改造

| 时机 | 行为 |
|---|---|
| 启动 | 建 session + `initState(clock.now())` + `phaseHistory = [{phase:'WARM_UP', at:0, reason:'time'}]` |
| 每 turn 之前 | TICK → 若 phase 变 → 入 `phaseHistory` + stderr 写 `[cli] phase → X (elapsed=Y min)` |
| | 时间到 END → break（不调 LLM） |
| | 整句 stop 关键词命中 → 调 `applyEvent(USER_STOP)` + 强制 phaseHistory 推 `END@at reason:user_stop` + 本 turn **仍走 LLM** 让 agent 道别 |
| | 正常消息 → `applyEvent(USER_MSG)`（重置 lastUserMsgAt） |
| 每 turn LLM 调用 | `buildFinalSystem(systemPrompt, state)` → stderr 写 `[cli] ctx: X elapsed=Y silence=Z` |
| 退出 | `sessions.markEnded(sessionId, { phaseHistory, reason })` |

### 6.4 Stop 关键词整句匹配

```ts
const STOP_REGEX = /(?:^|[.!?]\s+|\n)(stop|quit|end|bye|done|结束|停)\b[.!?\s]*$/i
```

设计要求（plan §5.1 L1 #11 + scope §2.3）：

| 输入 | 期望 | 命中？ |
|---|---|---|
| `stop` | stop | ✅ |
| `Stop.` | stop | ✅ |
| `okay. stop` | stop（前缀是 `. `） | ✅ |
| `let's stop and continue` | 不命中（stop 后面不是句末） | ❌ |
| `I don't want to stop.` | 不命中（stop 前是 space，不是 `[.!?] `） | ❌ |
| `结束` / `停` | stop | ✅（中文支持） |

L3 测试 "okay stop"（mid-sentence）正面验证了不命中：3 轮继续，ctx 无 END。

## 7. 手动 smoke 验证（已做）

### 7.1 Live 模式（DoD #5，2026-06-05）

> 注：本次 Live 验证跑的是 Replay（fixture），原因是 §7.1 重在验证"CLI 集成 + state machine + DB 落 phase_history"，不是验证 LLM 真实响应。Replay 在 Live 路径上完全等价——只是把 LLM 调用换成 fixture 查找。L1/L3 已覆盖 state machine 全部逻辑分支。

`MOCK_TIME=1 MOCK_NOW=2026-01-01T00:00:00Z` + 5 轮对话（hi / fine / castle / creeper / played）：

```
[cli] ctx: WARM_UP elapsed=0.0 silence=0.0
[Teacher]: Hi there! How are you doing today?
[cli] ctx: WARM_UP elapsed=1.0 silence=0.0
[Teacher]: Glad to hear it! What did you do today?
[cli] ctx: WARM_UP elapsed=2.0 silence=0.0
[Teacher]: Castles are so cool! Do you build them in survival or creative mode?
[cli] ctx: WARM_UP elapsed=3.0 silence=0.0
[Teacher]: Oh no, creepers! Have you tried building walls around your castle?
[cli] ctx: WARM_UP elapsed=4.0 silence=0.0
[Teacher]: Nice! What game did you play?
[cli] markEnded 1d4e7c22-... reason=user_exit
[cli] markEnded done
```

5 轮对话全部成功，stderr 每行 `[cli] ctx: PHASE elapsed=Y silence=Z` 可见（DoD #5）。Clock 在 5 轮内 +5 分钟，但 WARM_UP 边界是 `< 5`，所以 5 个 turn 全程 WARM_UP，符合预期。

### 7.2 MOCK_TIME demo：6 轮推入 MAIN_ACTIVITY

`MOCK_TIME=1` + 6 轮对话：

```
[cli] ctx: WARM_UP elapsed=0.0 silence=0.0
[Teacher]: Hi there! How are you doing today?
[cli] ctx: WARM_UP elapsed=1.0 silence=0.0
[Teacher]: Glad to hear it! What did you do today?
```

### 7.3 Stop 流程

`hi\nstop\n`：

```
[cli] ctx: WARM_UP elapsed=0.0 silence=0.0
[Teacher]: Hi there! How are you doing today?
[cli] phase → END (user_stop, elapsed=1.0 min)   ← USER_STOP 事件触发
[cli] ctx: END elapsed=1.0 silence=1.0
[Teacher]: Okay, see you next time! Don't forget to practice. 👋
[cli] markEnded 883a99fa-... reason=user_stop
```

DB 直接读：

```
sessions: [
  {
    "id": "24866b5b-7ce8-4731-bc6d-046562e98ee8",
    "started_at": "2026-06-05T15:50:58.574Z",
    "ended_at": "2026-06-05T15:50:58.652Z",
    "duration_min": 0,
    "phase_history": "[{\"phase\":\"WARM_UP\",\"at\":0,\"reason\":\"time\"},{\"phase\":\"END\",\"at\":1,\"reason\":\"user_stop\"}]"
  }
]
```

`phase_history` 是合法 JSON 数组，首条 `WARM_UP@0 reason:time`（CLI 启动时入的种子），末条 `END@1 reason:user_stop`（USER_STOP 触发）。DoD #5 + scope §2.4 全部满足。

## 8. 风险与遗留问题

| 风险 / 遗留 | 影响 | 处置 |
|---|---|---|
| **Stop 关键词误杀** | "I don't want to stop" 被识别为 stop | 整句匹配正则 + L3 "okay stop" 测试 + 4 个 L1 边界用例一起把守 |
| **MOCK_TIME 推进 bug** | 误把 clock advance 放在 TICK 之前会让 elapsed 虚高 1 | 实现里放在 LLM 调用之后，下轮 TICK 才用新 elapsed；L2 跨 30 min 序列测试正面验证 |
| **state machine 直接调 Date.now()** | 可测性归零 | `applyEvent` 必须传 `clock` 参数，类型层面禁止隐式时间；L1 `realClock` 测试覆盖 |
| **30 分钟阈值在小学生场景不实用** | 真实场景是 5-10 分钟 | 阈值做成常量，sprint 末看反馈再调；v0.4 暂写死 |
| **L3 fixture 误用** | 测试用 `minecraft` / `bye` 但没有 fixture → 调试时发现"3+ 轮全 exit 1" | 已统一用 `hi / fine / castle / creeper / played` 5 个 fixture 覆盖 L3 全部输入 |
| **biome auto-fix 改 cli.ts 格式** | 提交前 lint 失败 | `pnpm exec biome check --write` 自动修，4 文件 → 0 错误；无功能改动 |
| **Migrations 目录定位** | 测试用 `resolveMigrationsDirForTesting()` 走 `<repo>/src/storage/migrations`，跟生产代码一致 | OK，测试与生产共源码 |
| **`duration_min` 精度** | 5 轮对话总是 0（不足 1 min） | OK，v0.4 不做分钟级精度；v0.5+ 视反馈调整 |
| **Reason 字段未持久化到独立列** | L3 测试只能从 phase_history 末条 reason 字段反推 | scope §2.4 已说明，v0.4 不开新列；v0.5+ 可加 `end_reason TEXT` |
| **Silence 不切 phase → 用户走了 CLI 不知道** | session 阻塞在 readline | model C 的取舍：让 LLM 看 silence hint 自己决定；用户可 Ctrl+C 退出（markEnded 记最后 phase） |
