## 6. 架构与代码改动

### 6.1 新增 `src/agent/topic-matcher.ts`

| 元素 | 职责 |
|---|---|
| `jaccard(a, b)` | 大小写不敏感的 Jaccard 相似度；空数组返 0；重复元素 Set 去重 |
| `matchTopic(sessionKeywords, topics, threshold=0.3)` | 返 best match（最高 jaccard）+ tie 按字典序；空 keywords 返 null；无 topic 过阈值返 null |
| `TopicMatch` interface | `{topic, jaccard, shared}` —— shared 给调试 / 日志用 |

### 6.2 新增 `src/storage/topics.ts` + DAO

| 类 | 方法 | 说明 |
|---|---|---|
| `TopicsDao` | `list()` / `get(name)` | 读 topic 库 |
| `TopicStatsDao` | `get(topic)` / `all()` / `incrementAndUpdate(topic, now)` | UPSERT pattern：`INSERT ... ON CONFLICT(topic) DO UPDATE SET count=count+1, last=excluded.last`；first_discussed_at 首次写入后不变 |

### 6.3 迁移 `003_topic_stats.sql`

```sql
CREATE TABLE topics (name PK, keywords_json, description, created_at);
CREATE TABLE topic_stats (topic PK, discussion_count DEFAULT 0, first_discussed_at, last_discussed_at);
-- seed 7 starter topics: minecraft/school/sports/food/family/movies/music
```

字段对齐 `001_init.sql` 已预留的 `topics_used`（v0.6 暂不用，留给 v0.7 工具）。

### 6.4 `src/agent/context-injector.ts` 改造

`buildSystemContext(state, lastReview, activeTopics?, now?)` 追加第三段：
- `activeTopics.length === 0` → 整段不出现
- `activeTopics.slice(0, 5)` 截断到 5
- 每条格式：`${topic} (${count} ${time|times}, ${daysAgo})`
- `formatDaysAgo`: 0→"today", 1→"yesterday", 2+→"N days ago"
- `computeDaysAgo` 用 `now: Date` 参数（默认 `new Date()`）保证可测

### 6.5 `src/agent/prompt-builder.ts` 改造

`buildFinalSystem(prompt, state, lastReview, activeTopics?)` 透传。

### 6.6 `src/cli.ts` 改造

| 时机 | 行为 |
|---|---|
| 启动 | `topicStats.all()` → `activeTopics: TopicStat[]`（whole-session，与 lastReview first-turn-only 不同） |
| 每 turn | `buildFinalSystem(..., isFirstTurn ? lastReview : null, activeTopics)` |
| finally 块 | `matchTopic(summaryKeywords, topics.list())` → 命中 → `topicStats.incrementAndUpdate()` + stderr `[cli] topic match: X jaccard=Y shared=[...]`；未命中 → stderr `[cli] topic match: none (keywords=[...])` |

**关键设计选择**：
- `activeTopics` 是 whole-session 而非 first-turn-only——topic 视角是聚合横截面，跨整轮都相关
- topic match **在** markEnded **之后**——match 失败不会阻塞 session 持久化
- topic match 在 summarize 失败时（`summaryKeywords=[]`）自动返 null，零成本跳过

### 6.7 Barrel 扩

- `src/agent/index.ts`: 加 `export * from './topic-matcher.js'`
- `src/storage/index.ts`: 加 TopicsDao / TopicStatsDao / Topic / TopicStat 重导出

## 7. 手动 smoke 验证（已做）

### 7.1 Live 模式 — session A 5 turns（DoD #5，2026-06-09）

`APP_DATA_DIR=./data/live-demo-v06` + 5 轮对话（hi / fine / castle / creeper / played）：

```
English Oral Teacher Agent — CLI mode
Persona target: Sample Student (age 13, intermediate)
Type "exit" to quit.

[cli] ctx: WARM_UP elapsed=0.0 silence=0.0
[Teacher]: Hi there! How are you doing today?

[cli] ctx: WARM_UP elapsed=0.0 silence=0.0
[Teacher]: Glad to hear it! What did you do today?

[cli] ctx: WARM_UP elapsed=0.0 silence=0.0
[Teacher]: Castles are so cool! Do you build them in survival or creative mode?

[cli] ctx: WARM_UP elapsed=0.0 silence=0.0
[Teacher]: Oh no, creepers! Have you tried building walls around your castle?

[cli] ctx: WARM_UP elapsed=0.0 silence=0.0
[Teacher]: Nice! What game did you play?

[cli] summarize ok summary=109c keywords=5
[cli] markEnded dc3df5cf-e12c-40a6-884d-141acdf084bb reason=user_exit
[cli] markEnded done
[cli] topic match: minecraft jaccard=0.50 shared=[minecraft,castle,creeper,wall,build]
```

关键证据：
- `topic match: minecraft` — 关键词 5 个全在 minecraft topic 里，Jaccard=0.5，shared 完整列出
- `markEnded done` 出现 **在** `topic match` 之前——topic match 失败不会阻塞 markEnded
- `summary=109c keywords=5` — v0.5 summarizer 正常工作

### 7.2 Live 模式 — session B 5 turns（DoD #6，2026-06-09）

同一 `data/live-demo-v06` 起新 session，跑 5 轮相同输入：

```
[cli] topic match: minecraft jaccard=0.50 shared=[minecraft,castle,creeper,wall,build]
```

DB 查 `topic_stats`：

```
=== topic_stats ===
 - minecraft | count=2 | first=2026-06-09T14:02:17.433Z | last=2026-06-09T14:02:23.766Z
```

`count=2`（2 个 session 匹配），`first ≠ last`（UPSERT 累加成功）。

### 7.3 Active topics 注入验证（DoD #7，2026-06-09）

CLI 不会把 `[System Context]` 写 stderr（按 v0.5 决定）。验证靠：
1. **L1 单测覆盖**：`context-injector.test.ts` 9 个 case 全过，含 2 个 top 5 行为、formatDaysAgo 边界、段顺序
2. **L3 端到端 wiring**：`activeTopics` 启动时 `topicStats.all()` 一次性读，main loop 每轮 `buildFinalSystem(..., activeTopics)` 透传——见 L3 `topics.test.ts` 的 "7 seed topics loaded" 验证 DAO 可用
3. **Live 间接证据**：§7.1 §7.2 跑通 = `matchTopic` 拿到正确 topics + match 后 `incrementAndUpdate` 写入 DB；这覆盖了"DAO + 匹配器 + CLI 集成"全链

为直观看到 active topics 段拼到 `[System Context]` 末尾，临时跑 ad-hoc 脚本（用 v0.5 §7.2 同款办法）：

```
$ loadTopicStats + buildSystemContext（注入 session B 的 1 个 active topic）
[System Context]
- Phase: WARM_UP
- Elapsed: 0.0 min
- Silence: 0.0 min
- Last transition: 0.0 min ago (entered WARM_UP)
- Active topics: minecraft (2 times, today)
```

"Active topics" 段确实在 [System Context] 末尾，top 1 格式 `minecraft (2 times, today)` 符合 §3.6 设计。`2 times` 复数 + `today` 0 days 的格式化都正确。

## 8. 风险与遗留问题

| 风险 / 遗留 | 影响 | 处置 |
|---|---|---|
| **Jaccard 阈值 0.3 太松** | 无关 keywords 误匹 sports/food | L1 fixture 测 1 个反例；live demo 验证；v0.6.1 调阈值 |
| **Jaccard 阈值 0.3 太严** | 真聊 minecraft 但关键词分散 | seed 关键词 10 个覆盖常用词；live 看实际数据 |
| **一次 session 只匹配 1 个 topic** | 多主题场景丢信息 | 设计权衡：v0.6 简化；v0.7 工具发现时再说 |
| **active topics 在 session 中途不更新** | LLM 不知道"本次匹配上 minecraft 了" | 设计选择：active topics 是 session 外视角；v0.7 再说 |
| **Topic seed 7 个不够** | 真实学生聊到 seed 之外的话题 | DAO 预留 addTopic；v0.7 工具自动发现 |
| **`computeDaysAgo` 用 `new Date()` 默认** | 测试不稳定 | 实际加了 `now: Date` 注入；L1 全用 `new Date('2026-06-09T12:00:00.000Z')` 固定时间 |
| **`description` 字段在 v0.6 暂未展示** | 话题描述在 [System Context] 中看不到 | v0.6 暂不展示；v0.7 工具用 description 做决策 |
| **`topics_used` 字段 001 已留但 v0.6 没用** | 浪费 1 列 | v0.7 工具记录"本 session 用了哪些 topic"时启用 |
| **biome auto-fix 改 5 个文件** | 提交前 lint 失败 | `biome check --write` 自动修（import 排序 / 模板字面量 / 类型推断）；无功能改动 |
