## 6. 架构与代码改动

### 6.1 新增 `src/storage/migrations/004_mistakes.sql`

```sql
CREATE TABLE mistakes (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  session_id TEXT NOT NULL,
  original TEXT NOT NULL,
  corrected TEXT NOT NULL,
  category TEXT NOT NULL CHECK (category IN ('grammar', 'vocabulary', 'spelling')),
  ts TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (session_id) REFERENCES sessions(id) ON DELETE CASCADE
);
CREATE INDEX idx_mistakes_session ON mistakes(session_id, ts);
```

### 6.2 新增 `src/storage/mistakes.ts` + DAO

| 方法 | 说明 |
|---|---|
| `append(input)` | INSERT + 返完整行；可选 `ts` 默认 `new Date().toISOString()` |
| `getBySession(sessionId)` | 按 `ts ASC, id ASC` 排，给单 session 时间线 |
| `getRecent(limit)` | 跨 session `ts DESC, id DESC` LIMIT N，给 v0.7.1 correction 注入用 |

### 6.3 新增 `src/agent/tool-registry.ts`

| 元素 | 职责 |
|---|---|
| `Tool` interface | `name / description / schema (z.ZodTypeAny) / execute(args)` |
| `ToolRegistry` | `register(tool)` —— 重名 throw；`get(name)` —— 未注册 undefined；`list()` —— 插入顺序数组 |
| `createToolRegistry()` | 工厂返 `ToolRegistry` 实例（Map 内部存储） |

### 6.4 新增 `src/agent/tool-parser.ts`

| 元素 | 职责 |
|---|---|
| `TOOL_REGEX` | `/<tool>(\w+)\(\s*(\{[\s\S]*?\})\s*\)<\/tool>/` —— 支持多行 body，非贪婪 |
| `parseToolCall(response)` | 返第一个匹配的 `ParsedToolCall` 或 null；JSON 无效 throw；非对象 throw |
| `stripToolCall(response, parsed)` | 删 rawMatch 子串 + 折叠 3+ 连续换行为 2 + trim |

### 6.5 新增 `src/agent/tools/mark-mistake.ts`

| 元素 | 职责 |
|---|---|
| `MarkMistakeArgsSchema` | zod object：`original.min(1).max(500) / corrected.min(1).max(500) / category.enum(grammar,vocabulary,spelling)` |
| `createMarkMistakeTool(db, sessionId)` | factory：sessionId 由 CLI 启动时绑定，**不**让 LLM 传 |

**关键设计选择**：sessionId 工厂注入而非 schema 字段——LLM 不用关心，跨 session 攻击不可能。

### 6.6 新增 `prompts/tools.md`

教 LLM `<tool>name({json})</tool>` 语法，含调用规则和 3 个示例（grammar/spelling/无错例）。

### 6.7 `src/prompts/loader.ts` 改造

`SystemPrompt` interface 加 `tools: string | null`；`loadSystemPrompt()` 读 `prompts/tools.md`（不强制）；`buildSystemString()` 在末尾追加 `# TOOLS` 段。

### 6.8 `src/cli.ts` 改造

| 时机 | 行为 |
|---|---|
| 启动 | 创 `ToolRegistry`，注册 `createMarkMistakeTool(db, session.id)` |
| 每 turn 流 | **改 buffer 模式**：先把 `chatStream` 所有 text chunk 拼到 `response`，**不**实时 stdout |
| stream 后 | `parseToolCall(response)`；解析失败 → stderr 告警；解析成功 → `tool.execute(args)`，失败 → stderr 告警；成功 → stderr `[cli] tool call: name({args})` |
| 显示 | `display = parsed ? stripToolCall(response, parsed) : response`；`stdout.write(display + '\n\n')` |
| 持久化 | history + messages.append 用 `display`（不含 tool 块），LLM 上下文是干净的 |
| finally | 不动；topic match + markEnded 流程跟 v0.6 一致 |

**核心 trade-off**：buffer 模式让 stdout 干净（学生看不到 tool 块），代价是失去 streaming 体验。v0.7 单 LLM call 50-200 token，buffer 延迟可忽略。现有 L3 测试只看最终 stdout，无 breaking change。

### 6.9 Barrel 扩

- `src/agent/index.ts`: 加 tool-registry / tool-parser / tools/mark-mistake 重导出
- `src/storage/index.ts`: 加 mistakes / MistakeCategory 重导出

## 7. 手动 smoke 验证（已做）

### 7.1 Live 模式 — 5 turns 含 1 个 mistake（DoD #5 + #6，2026-06-09）

`APP_DATA_DIR=./data/live-demo-v07` + 5 轮（hi / fine / **I go to school yesterday** / played minecraft / creeper）：

```
English Oral Teacher Agent — CLI mode
Persona target: Sample Student (age 13, intermediate)
Type "exit" to quit.

[cli] ctx: WARM_UP elapsed=0.0 silence=0.0
[Teacher]: Hi there! How are you doing today?

[cli] ctx: WARM_UP elapsed=0.0 silence=0.0
[Teacher]: Glad to hear it! What did you do today?

[cli] ctx: WARM_UP elapsed=0.0 silence=0.0
[Teacher]: [cli] tool call: mark_mistake({"original":"I go to school yesterday","corrected":"I went to school yesterday","category":"grammar"})
Oh, you went to school yesterday! What did you do there?

[cli] ctx: WARM_UP elapsed=0.0 silence=0.0
[Teacher]: Nice! What game did you play?

[cli] ctx: WARM_UP elapsed=0.0 silence=0.0
[Teacher]: Oh no, creepers! Have you tried building walls around your castle?

[cli] summarize ok summary=109c keywords=5
[cli] markEnded cac07b0e-5fd9-43da-b04c-6a038250df72 reason=user_exit
[cli] markEnded done
[cli] topic match: minecraft jaccard=0.50 shared=[minecraft,castle,creeper,wall,build]
```

关键证据：
- 第 3 turn 后 stderr 出现 `[cli] tool call: mark_mistake(...)`，args 正确（original 是学生原句、corrected 是正确版、category="grammar"）
- 第 3 turn 的 [Teacher] 输出**只**有 "Oh, you went to school yesterday! What did you do there?"，**没有** `<tool>...</tool>` 标签（DoD #6 ✅）
- markEnded 仍然成功；topic match v0.6 行为不变；summarize 不受影响

**注意**：上面 `[cli] tool call: ...` 这行是 stderr，跟 stdout 的 `[Teacher]: ` 在同一终端混在一起；但分流到管道后 stdout 完全干净（参看 L3 测试 §4.3 "1 turn with yesterday"：`expect(stdout).not.toContain('<tool>')` 通过）。

## 8. 风险与遗留问题

| 风险 / 遗留 | 影响 | 处置 |
|---|---|---|
| **LLM 漏写 `<tool>` 标签** | live mode 不调 tool | prompt 强约束（tools.md 含 EXACTLY 字样 + 3 示例）；L3 走 Replay fixture 保证测试可重现；live demo 没法 100% 保证（看真实 LLM 训练） |
| **LLM 输出多个 `<tool>` 块** | 后续块被忽略 | regex 取第一个；prompt 写 "At most one tool call per turn" |
| **stream → buffer breaking change** | 学生失去流式体验 | 单 LLM call 50-200 token，buffer 延迟 < 200ms 不可感；L3/L4 测试不受影响 |
| **Tool execute 抛错阻塞对话** | DB 故障 / zod 失败 | try/catch + stderr 告警 + 跳过；mark_mistake 是 best-effort，丢一条不影响 |
| **mark_mistake 失败 LLM 不知道** | 同样错例下次再调（重复） | v0.7 简化；v0.7.1+ 如果需要可加 tool_result 反馈 |
| **学生输入有 "yesterday" 但其实没错** | fixture 模式下永远调 mark_mistake | live mode 不会（LLM 看上下文判断）；测试模式接受这种"假阳" |
| **Replay fixture 误匹配** | "I played yesterday" 命中 hobby-played 而非 mistake-yesterday | fixture loader 是 first-match-wins，按文件名字典序；测试输入特意避开 "played" |
| **biome auto-fix 改 3 个文件** | 提交前 lint 失败 | `biome check --write` 自动修（长字符串拆行、链式调用换行）；无功能改动 |
| **tool 块在 stderr/stdout 同终端混行** | 视觉上 `[cli] tool call:` 出现在 `[Teacher]:` 之后看着奇怪 | live demo 表象问题；分流后干净；可选 v0.7.1 把 `[Teacher]:` 移到 tool 解析之后再 write |
| **prompts/USER.md 不存在时 fallback** | v0.7 不影响；loader 用 USER.md.example | 已有逻辑，v0.7 不动 |
