## 2. context-injector 改动

### 2.1 函数签名

```ts
export function buildSystemContext(
  state: SessionState,
  lastReview: LastReview | null = null,
  activeTopics: TopicStat[] = [],
  recentMistakes: Mistake[] = [],   // v0.7.1 NEW
  now: Date = new Date(),
): string
```

### 2.2 渲染规则

- `recentMistakes.length === 0` → 整段不出现
- 截到 top 5（防 N 传超过 5 的边界）
- 每条：`  - "{original}" → "{corrected}" ({category})`
- 段头：`- Recent mistakes (N={count}):`（{count} 是实际渲染条数，不是 limit）

```ts
function renderRecentMistakes(mistakes: Mistake[]): string[] {
  if (mistakes.length === 0) return []
  const top = mistakes.slice(0, 5)
  const lines = [`- Recent mistakes (N=${top.length}):`]
  for (const m of top) {
    lines.push(`  - "${m.original}" → "${m.corrected}" (${m.category})`)
  }
  return lines
}
```

### 2.3 段顺序

```
[System Context]
- Phase: ...
- Elapsed: ...
- Silence: ...
- Last transition: ...
{lastReview 段 if first turn}
{active topics 段 if non-empty}
{recent mistakes 段 if non-empty}   ← v0.7.1 NEW
```

### 2.4 L1 测试

| # | case | 期望 |
|---|---|---|
| 1 | empty `[]` | 整段不出现 |
| 2 | 1 个 mistake | "Recent mistakes (N=1):" + 1 行 |
| 3 | 3 个 mistakes | "Recent mistakes (N=3):" + 3 行 |
| 4 | 5 个 mistakes | "Recent mistakes (N=5):" + 5 行（全显示） |
| 5 | 8 个 mistakes | "Recent mistakes (N=5):" + 5 行（截前 5） |
| 6 | 含特殊字符（引号 / 反斜杠）的 original | 不 break 格式 |
| 7 | 4 段共存 + 顺序：Phase/Elapsed/.../Last session/Active topics/Recent mistakes | 段顺序正确 |

## 4. cli.ts 改动

### 4.1 启动段加载

在现有 `activeTopics: TopicStat[] = topicStats.all()` 旁边加：

```ts
const mistakesDao = createMistakesDao(db)
const recentMistakes = mistakesDao.getRecent(5)
process.stderr.write(
  `[cli] loaded ${recentMistakes.length} recent mistakes (cross-session)\n`,
)
```

注意：v0.7 已经在 toolRegistry 里 `createMarkMistakeTool(db, session.id)`，那里也 createMistakesDao 了一次（内部）。v0.7.1 让 CLI 自己也 create 一个用来 getRecent。两个 DAO 实例没问题（DAO 是无状态的，prepare 语句独立）。

### 4.2 dedup set

```ts
const markedOriginals = new Set<string>()
```

放在 `while (true)` 主循环外，本 session 范围。

### 4.3 buildFinalSystem 调用

```ts
const system = buildFinalSystem(
  systemPrompt,
  state,
  isFirstTurn ? lastReview : null,
  activeTopics,
  recentMistakes,   // v0.7.1 NEW
)
```

### 4.4 Tool 调用 dedup

替换现有 `if (parsed)` 块：

```ts
if (parsed) {
  display = stripToolCall(response, parsed)
  const tool = toolRegistry.get(parsed.name)
  if (!tool) {
    process.stderr.write(`[cli] tool unknown: ${parsed.name}\n`)
  } else {
    const original = typeof parsed.args.original === 'string' ? parsed.args.original : ''
    if (parsed.name === 'mark_mistake' && original && markedOriginals.has(original)) {
      process.stderr.write(
        `[cli] tool dedup: skipped (already marked: "${original}")\n`,
      )
    } else {
      try {
        tool.execute(parsed.args)
        if (parsed.name === 'mark_mistake' && original) {
          markedOriginals.add(original)
        }
        process.stderr.write(
          `[cli] tool call: ${parsed.name}(${JSON.stringify(parsed.args)})\n`,
        )
      } catch (err) {
        process.stderr.write(`[cli] tool execute error: ${(err as Error).message}\n`)
      }
    }
  }
}
```

注意：dedup 检查在 tool.execute 之前。失败不算 marked（execute throw 时不 add 到 set）。

## 5. L3 测试改动

### 5.1 修改：v0.7 "2 turns both with yesterday: same session accumulates 2 mistake rows"

v0.7.1 后行为变了——dedup 应让 DB 只 1 行。把测试改成：

```ts
it('2 turns both with "yesterday": dedup blocks 2nd mark, DB has 1 row + stderr has dedup log', async () => {
  const result = await runCli('I go to school yesterday\nI see my friend yesterday\nexit\n', {
    MINIMAX_API_KEY: 'sk-test',
    APP_DATA_DIR: dataDir,
  })
  expect(result.exitCode).toBe(0)
  // First turn: tool call fires
  const toolCalls = result.stderr.match(/\[cli\] tool call: mark_mistake/g) ?? []
  expect(toolCalls.length).toBe(1)
  // Second turn: dedup catches it
  expect(result.stderr).toMatch(/\[cli\] tool dedup: skipped/)

  // DB only 1 row
  const db = openDb({ dataDir })
  applyMigrations(db, migrationsDir)
  const sessions = createSessionsDao(db)
  const mistakes = createMistakesDao(db)
  const session = sessions.list()[0]
  const rows = mistakes.getBySession(session?.id ?? '')
  expect(rows).toHaveLength(1)
  db.close()
})
```

### 5.2 新增：session A 写 mistake → session B 启动时加载 N 条

```ts
it('cross-session: session B startup loads mistakes from session A and logs N count', async () => {
  // Session A: 1 mistake
  const a = await runCli('I go to school yesterday\nexit\n', {
    MINIMAX_API_KEY: 'sk-test',
    APP_DATA_DIR: dataDir,
  })
  expect(a.exitCode).toBe(0)
  expect(a.stderr).toMatch(/\[cli\] tool call: mark_mistake/)
  expect(a.stderr).toMatch(/\[cli\] loaded 0 recent mistakes/)  // A starts empty

  // Session B: same dataDir, starts with 1 loaded mistake
  const b = await runCli('hi\nexit\n', {
    MINIMAX_API_KEY: 'sk-test',
    APP_DATA_DIR: dataDir,
  })
  expect(b.exitCode).toBe(0)
  expect(b.stderr).toMatch(/\[cli\] loaded 1 recent mistakes/)
})
```

### 5.3 维持：v0.7 "session A + session B both produce mistakes: total 2 mistakes across 2 sessions"

不动——这个测试是 2 个独立 session 各 1 mistake，dedup 是 session scope，跨 session 不影响。

### 5.4 维持：v0.7 "1 turn with yesterday: 1 row + stderr tool call + no tool block in stdout"

不动。
