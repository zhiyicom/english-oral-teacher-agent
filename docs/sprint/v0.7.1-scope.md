# Sprint v0.7.1 Scope — Correction Injection + Mistake Dedup

<!-- STATUS: PLANNING -->

> **本文件是 Sprint v0.7.1 的计划**。详细设计见 [v0.7.1-design.md](./v0.7.1-design.md)。
>
> v0.7.1 核心：**把 v0.7 已有的 mistakes 数据用起来**——下次 session 启动时拉最近 N 条错例注入 `[System Context]`，让 LLM 引导学生不再犯同样的错；同时给 mark_mistake 加 dedup 防止同一句话被反复 mark。

---

## 1. 背景

v0.7 落地了"记错例"的链条（`<tool>mark_mistake(...)</tool>` → DB），但**还没有人用这些数据**。v0.7 scope §2.2 把 Correction 注入显式推迟到 v0.7.1。

v0.7.1 把这条线接上：

1. **写 → 读**：v0.7 只写不读；v0.7.1 启动时读 `mistakes.getRecent(5)` 注入 [System Context]
2. **去重**：v0.7 没限制；同一句话 LLM 可能在 Replay 模式或上下文窗口模糊时反复调 mark_mistake。v0.7.1 加 in-process Set 缓存防同 session 重复

不做 live LLM 真实验证（独立任务，跟 v0.7.1 解耦）。

## 3. 关键决策

### 3.1 Dedup 范围：整 session vs 滑动窗口

**选整 session**。理由：
- 实现简单（in-process `Set<string>`，0 额外存储）
- 13 岁口语 session 通常 < 30 turn，整 session 去重不会"漏"老错例
- 滑动窗口"5 turn 内"语义需要 Map<original, turnNumber> + 维护 turn 计数器，复杂度翻倍
- 如果 user 真的想"同一句话学生 5 turn 后又错可以再 mark"，v0.7.2+ 单独做

### 3.2 注入刷新时机

**启动时一次性读，每 turn 传同一 list**（跟 v0.6 active topics 同款）。理由：
- 简单一致，跟现有 `activeTopics` 模式对齐
- 本 session 当前 turn mark 的错例下一 turn LLM 看不到 → **但 LLM 通过对话历史看得到自己写的纠错话术**（即"Oh, you went to school yesterday!"），效果近似
- 每 turn 重读虽然立即可见，但要么是 5 次 DB 查询 / session（轻量但浪费），要么搞 dirty-check（复杂）
- 如果中后期发现 LLM 真的"忘"了本 session 错例，v0.7.2+ 改

### 3.3 N=5

跟 active topics top 5 一致。token 成本可控，模式可读。

### 3.4 Recent mistakes 段格式

```
- Recent mistakes (N=5):
  - "I go to school yesterday" → "I went to school yesterday" (grammar)
  - "delicius" → "delicious" (spelling)
  ...
```

理由：
- 短：每条 1 行
- 自然语言：LLM 直观理解
- 含 category：LLM 知道是 grammar / vocab / spelling 哪类
- 不含 ts / id：噪音，LLM 用不上

### 3.5 Dedup 触发后 LLM 怎么知道？

**LLM 不知道**。tool 块仍然从 stdout 删除（看不到自己调失败），但 LLM 知道自己的输出包含 tool 块（因为它写出来了）。
- 实际效果：LLM 下一 turn 仍可能再调 mark_mistake，CLI 仍然 dedup
- 不影响对话（dedup 是 silent，不告诉 LLM）
- 如果未来要让 LLM 知道，参考 v0.7-protocol-compare.md §3 "Tool 失败时怎么办"

### 3.6 [System Context] 段顺序

```
[System Context]
- Phase: ...
- Elapsed: ...
- Silence: ...
- Last transition: ...
- Last session: ...      (v0.5, first-turn-only)
- Active topics: ...     (v0.6, whole-session)
- Recent mistakes (N=5):  (v0.7.1, whole-session)
  - "..." → "..." (...)
```

mistakes 段在最后，理由：
- 横截面信息（最新 → 最近 → 错例）从泛到精
- 错例段是"长内容"，放最后避免遮挡 phase 信息
- 跟 active topics 段相对位置稳定，回归测试稳定

## 5. CLI 集成

### 5.1 启动

```ts
// 跟 v0.6 activeTopics 同样位置
const activeTopics: TopicStat[] = topicStats.all()
const recentMistakes: Mistake[] = mistakes.getRecent(5)
process.stderr.write(
  `[cli] loaded ${recentMistakes.length} recent mistakes (cross-session)\n`,
)

// dedup set
const markedOriginals = new Set<string>()
```

### 5.2 每 turn

```ts
const system = buildFinalSystem(
  systemPrompt,
  state,
  isFirstTurn ? lastReview : null,
  activeTopics,
  recentMistakes,  // v0.7.1: 同一 list 每 turn 传
)
```

### 5.3 Tool 调用 dedup

```ts
if (parsed) {
  display = stripToolCall(response, parsed)
  const tool = toolRegistry.get(parsed.name)
  if (!tool) {
    process.stderr.write(`[cli] tool unknown: ${parsed.name}\n`)
  } else if (
    parsed.name === 'mark_mistake' &&
    typeof parsed.args.original === 'string' &&
    markedOriginals.has(parsed.args.original)
  ) {
    process.stderr.write(
      `[cli] tool dedup: skipped (already marked: "${parsed.args.original}")\n`,
    )
  } else {
    try {
      tool.execute(parsed.args)
      if (parsed.name === 'mark_mistake' && typeof parsed.args.original === 'string') {
        markedOriginals.add(parsed.args.original)
      }
      process.stderr.write(
        `[cli] tool call: ${parsed.name}(${JSON.stringify(parsed.args)})\n`,
      )
    } catch (err) {
      process.stderr.write(`[cli] tool execute error: ${(err as Error).message}\n`)
    }
  }
}
```

dedup 逻辑放 CLI 层（不在 tool 内部）的好处：
- tool 保持纯粹（factory 注入了 sessionId 已经够多了）
- v0.8+ 加 `mark_vocabulary` 等 tool 时 dedup 策略可以 per-tool 不一样
- L1 tool 测试不用关心 dedup

## 9. 风险

| 风险 | 影响 | 处置 |
|---|---|---|
| 启动时读 mistakes → session 多了 token 占满 | N=5 上限 + 每条 1 行短文本，可控 | top 5 截断（跟 active topics 一致） |
| Dedup 误杀（学生真的两次同样错应该都记） | 整 session 1 次即可，看上下文不算"丢"（LLM 知道学生说过） | v0.7.2+ 加滑动窗口可选 |
| LLM 在 [System Context] 见到错例后过度纠错 | prompt 里 AGENTS.md "Don't lecture" 仍生效；mistakes 只是参考 | 看 live demo 实际表现，必要时 v0.7.2 prompt 微调 |
| Session A 没有 mistake → session B 读到空 list | 段不显示（空数组隐藏），跟 active topics 一致行为 | L1 测试覆盖 |
| 跨 session 的 mistake 跟当前学生上下文无关 | 比如几天前的错，今天的话题不同；LLM 仍可能引用 | v0.8+ 用 vector / topic-bound retrieval 取代纯时间序 |
| recent mistakes 段在 system 里太重要被 LLM 当作"必答内容" | LLM 强迫每次都引用错例 | prompt 加"reference if relevant, don't force"；live 观察 |
