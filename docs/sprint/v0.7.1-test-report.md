## 8. 风险与遗留问题

| 风险 / 遗留 | 影响 | 处置 |
|---|---|---|
| **加载 N=5 错例增加 token 占位** | 单条 ~50 token × 5 = 250 token / turn | 可控；live demo 实际响应延迟无明显变化 |
| **Dedup 误杀**（学生两次同样错本应都记） | 整 session 1 次即可；LLM 知道学生说过 | v0.7.2+ 加滑动窗口可选（如果实际有需求） |
| **LLM 见到 Recent mistakes 段过度纠错** | prompt 里 AGENTS.md "Don't lecture" 仍生效 | live demo 观察；必要时 v0.7.2 prompt 加"reference if relevant, don't force" |
| **Loaded mistake 跟当前 session 话题无关** | LLM 可能引用 5 天前的错例 | v0.8+ 用 vector / topic-bound retrieval 取代纯时间序 |
| **每 turn 重读 mistakes 才能让 LLM 看到本 session 刚 mark 的** | LLM 通过对话历史看自己写的纠错话术，效果近似 | v0.7.2+ 改（如果实际有需求） |
| **CLI 自己的 mistakesDao 跟 tool 内 createMistakesDao 重复** | 2 个 DAO 实例访问同一 db | DAO 无状态，prepared statements 独立，无冲突（已写注释说明） |
| **biome auto-fix 改 3 个文件** | 提交前 lint 失败 | `biome check --write` 自动修（长字符串合并、缩进、regex 拆行）；无功能改动 |
| **v0.7 commit 留 3 个 typecheck 错误** | v0.7.1 首次跑 typecheck 暴露 | v0.7.1 一并修（fakePrompt.tools + @ts-expect-error 位置） |

## 9. 下一步建议

按 DEVELOPMENT_PLAN §3 + v0.7.1 scope §2.2 路线图：

**v0.7.2+（短期，条件性）**：
- 错例滑动窗口 dedup（"5 turn 内"语义）—— 只有在用户实际报"同样错想都记"才做
