## 2. `src/memory/embedder.ts` 改动

### 2.1 接口 & 公开 API

```ts
export interface Embedder {
  embed(text: string): Promise<Float32Array>
  /** dimension of vectors this embedder produces (e.g. 384) */
  readonly dim: number
}

export function createTransformersEmbedder(): Embedder
export function createMockEmbedder(opts?: {
  /** map text → fixed vec for determinism; default = hash-derived */
  table?: Record<string, Float32Array>
  /** vector dimension; default 384 to match MiniLM-L6-v2 */
  dim?: number
}): Embedder
```

### 2.2 transformers.js 懒加载实现

```ts
// 模块级单例 promise（不是 Promise<Pipeline>）—— null 表示 "尚未触发加载"
let extractorPromise: Promise<FeatureExtractionPipeline> | null = null

async function getExtractor(): Promise<FeatureExtractionPipeline> {
  if (!extractorPromise) {
    const { pipeline } = await import('@xenova/transformers')
    extractorPromise = pipeline('feature-extraction', 'Xenova/all-MiniLM-L6-v2', {
      quantized: true,
    })
  }
  return extractorPromise
}

export function createTransformersEmbedder(): Embedder {
  return {
    dim: 384,
    async embed(text: string): Promise<Float32Array> {
      const extractor = await getExtractor()
      const out = await extractor(text, { pooling: 'mean', normalize: true })
      // out.data is Float32Array of length 384
      return new Float32Array(out.data)
    },
  }
}
```

注意：
- **动态 `import('@xenova/transformers')`**：避免 CLI 启动时静态 import 触发加载；仅 `getExtractor()` 第一次调用时才解析模块
- **`pooling: 'mean', normalize: true`**：标准做法，输出已 L2-normalized → cosineSimilarity = dot product，但我们仍保留 `cosineSimilarity()` 函数以防未来换非归一化模型
- **`quantized: true`**：int8 量化版（~25MB 下载、~120MB 运行 RAM）

### 2.3 Mock factory（测试用）

```ts
export function createMockEmbedder(opts: {
  table?: Record<string, Float32Array>
  dim?: number
} = {}): Embedder {
  const dim = opts.dim ?? 384
  const table = opts.table ?? {}
  return {
    dim,
    async embed(text: string): Promise<Float32Array> {
      if (table[text]) return new Float32Array(table[text])
      // 默认：根据 text 字符 codepoint 总和派生确定性 vec
      const seed = [...text].reduce((s, c) => s + c.charCodeAt(0), 0)
      const vec = new Float32Array(dim)
      for (let i = 0; i < dim; i++) {
        vec[i] = Math.sin(seed + i) // 不归一化（让 cosineSimilarity 真实工作）
      }
      return vec
    },
  }
}
```

## 3. `src/memory/vector-store.ts` 改动

### 3.1 序列化

```ts
/** Float32Array → Buffer（共享底层 ArrayBuffer，无 copy） */
export function f32ToBuffer(vec: Float32Array): Buffer {
  return Buffer.from(vec.buffer, vec.byteOffset, vec.byteLength)
}

/** Buffer → Float32Array（拷贝出独立 ArrayBuffer，避免 SQLite buf 回收后 dangling） */
export function bufferToF32(buf: Buffer): Float32Array {
  // SQLite 返回的 Buffer 可能跟下一次查询共享底层内存，必须 copy
  const ab = buf.buffer.slice(buf.byteOffset, buf.byteOffset + buf.byteLength)
  return new Float32Array(ab)
}
```

理由：
- 写入用零拷贝（vec → buf）
- 读出必须拷贝（更安全；better-sqlite3 不保证 buf 长寿）

### 3.2 cosine

```ts
export function cosineSimilarity(a: Float32Array, b: Float32Array): number {
  if (a.length !== b.length) {
    throw new Error(`cosine: dim mismatch ${a.length} vs ${b.length}`)
  }
  let dot = 0
  let na = 0
  let nb = 0
  for (let i = 0; i < a.length; i++) {
    const x = a[i] as number
    const y = b[i] as number
    dot += x * y
    na += x * x
    nb += y * y
  }
  const denom = Math.sqrt(na) * Math.sqrt(nb)
  if (denom === 0) return 0
  return dot / denom
}
```

### 3.3 L1 测试

`tests/memory/vector-store.test.ts`：

| # | case | 期望 |
|---|---|---|
| 1 | roundtrip: `bufferToF32(f32ToBuffer(v))` 按位相等 | ✅ |
| 2 | cosine(a, a) === 1（同向量 → 1） | ✅（容差 1e-6） |
| 3 | cosine(a, b)，b = a × 2（缩放）→ 1（cosine 不受 magnitude 影响） | ✅ |
| 4 | cosine([1,0,0...], [0,1,0...]) → 0（正交） | ✅ |

## 4. Migration 005 + `sessions.ts` DAO 改动

### 4.1 Migration 005 文件

`src/storage/migrations/005_session_embeddings.sql`：

```sql
-- 005_session_embeddings.sql
-- v0.7.2: store session.summary embeddings as BLOB for in-memory cosine retrieval
-- BLOB layout: Float32Array of 384 floats (1536 bytes per session, MiniLM-L6-v2 dim)
-- NULL means not yet embedded (lookup excludes them via WHERE embedding IS NOT NULL)
```

注意：

### 4.3 实现

```ts
const updateEmbedding = raw.prepare('UPDATE sessions SET embedding = ? WHERE id = ?')
const selectWithEmbeddings = raw.prepare(
  `SELECT id, started_at, summary, keywords, embedding
   FROM sessions
   WHERE embedding IS NOT NULL AND summary IS NOT NULL
   ORDER BY started_at DESC`,
)

return {
  // ... existing
  setEmbedding(id: string, vec: Float32Array): void {
    updateEmbedding.run(f32ToBuffer(vec), id)
  },
  listWithEmbeddings(): SessionWithEmbedding[] {
    const rows = selectWithEmbeddings.all() as Array<{
      id: string
      started_at: string
      summary: string
      keywords: string | null
      embedding: Buffer
    }>
    return rows.map((r) => ({
      id: r.id,
      startedAt: r.started_at,
      summary: r.summary,
      keywords: r.keywords ? (JSON.parse(r.keywords) as string[]) : [],
      embedding: bufferToF32(r.embedding),
    }))
  },
}
```

## 5. `src/memory/retrieve-relevant.ts` 改动

### 5.1 接口

```ts
export interface RelevantSession {
  sessionId: string
  startedAt: string
  summary: string
  keywords: string[]
  similarity: number
  daysAgo: number
}

export function retrieveRelevant(opts: {
  candidates: SessionWithEmbedding[]
  queryVec: Float32Array
  topK: number
  excludeSessionId?: string
  now?: Date
}): RelevantSession[]
```

**注意**：函数接受 `candidates` 数组而不是 db handle —— 让函数纯（无 IO），调用方负责拿 candidates。理由：
- 单元测试无需建 DB
- CLI 只需 `sessions.listWithEmbeddings()` 一次，候选集复用
- 解耦 storage 跟 memory 模块的方向（memory 不依赖 storage）

### 5.2 实现

```ts
export function retrieveRelevant(opts: {
  candidates: SessionWithEmbedding[]
  queryVec: Float32Array
  topK: number
  excludeSessionId?: string
  now?: Date
}): RelevantSession[] {
  const now = opts.now ?? new Date()
  const filtered = opts.excludeSessionId
    ? opts.candidates.filter((c) => c.id !== opts.excludeSessionId)
    : opts.candidates
  const scored = filtered.map((c) => ({
    sessionId: c.id,
    startedAt: c.startedAt,
    summary: c.summary,
    keywords: c.keywords,
    similarity: cosineSimilarity(opts.queryVec, c.embedding),
    daysAgo: Math.max(
      0,
      Math.floor((now.getTime() - Date.parse(c.startedAt)) / 86_400_000),
    ),
  }))
  scored.sort((a, b) => b.similarity - a.similarity)
  return scored.slice(0, opts.topK)
}
```

### 6.2 渲染规则

- `relevantPast.length === 0` → 整段不出现
- 截到 top 2（防 caller 传多）
- 每条：`  - {daysAgo} days ago: "{summary 截 80 字}..." (keywords: k1, k2, k3)`
- 段头：`- Relevant past sessions (N={count}):`（{count} 是实际渲染条数）

```ts
function renderRelevantPast(items: RelevantSession[]): string[] {
  if (items.length === 0) return []
  const top = items.slice(0, 2)
  const lines = [`- Relevant past sessions (N=${top.length}):`]
  for (const r of top) {
    const truncated = r.summary.length > 80 ? `${r.summary.slice(0, 80)}...` : r.summary
    const kwStr = r.keywords.join(', ')
    const dayWord = r.daysAgo === 1 ? 'day' : 'days'
    lines.push(`  - ${r.daysAgo} ${dayWord} ago: "${truncated}" (keywords: ${kwStr})`)
  }
  return lines
}
```

### 6.3 段顺序（v0.7.2 后）

```
[System Context]
- Phase: ...
- Elapsed: ...
- Silence: ...
- Last transition: ...
{lastReview 段 if non-null}
{relevantPast 段 if non-empty}        ← v0.7.2 NEW（紧跟 Last session）
{active topics 段 if non-empty}
{recent mistakes 段 if non-empty}
```

注意：插入位置在 `lastReview` 之后、`activeTopics` 之前——历史块（lastReview + relevantPast）物理相邻、逻辑相关。

## 8. `cli.ts` 改动

### 8.1 启动段：embed query + retrieve top 2

在现有 `const activeTopics: TopicStat[] = topicStats.all()` 后面、`createToolRegistry()` 前面加：

```ts
// v0.7.2 — semantic retrieval of relevant past sessions. Skip if no
// lastReview (first session ever) or no embedded candidates exist. The
// embedder is lazy: first call here triggers transformers.js model load
// (~2-4s); subsequent embeds reuse the cached pipeline.
const embedder = createTransformersEmbedder()
let relevantPast: RelevantSession[] = []
if (lastReview && lastReview.keywords.length > 0) {
  try {
    const queryText = lastReview.keywords.join(' ')
    const queryVec = await embedder.embed(queryText)
    const candidates = sessions.listWithEmbeddings()
    relevantPast = retrieveRelevant({
      candidates,
      queryVec,
      topK: 2,
      excludeSessionId: lastReview.sessionId,
    })
    process.stderr.write(
      `[cli] retrieved ${relevantPast.length} relevant sessions (query="${queryText}")\n`,
    )
  } catch (err) {
    process.stderr.write(`[cli] retrieve relevant failed: ${(err as Error).message}\n`)
  }
}
```

- **lastReview 是已经存在的 v0.5 变量**（`loadLastReview(db)` 拉到的最近 session）

- embedder load 失败 → fallback 到空数组，CLI 不卡（DoD 不挂）

1. 启动时不 crash（embedder + retrieveRelevant 跑通）
2. stderr 有 `retrieved N relevant sessions` log
3. session END 后下次启动能拿到候选（说明 embedding 写入了）

## 13. 关键技术注意

### 13.1 ESM dynamic import

`@xenova/transformers` 是 ESM-only。Node 用 dynamic `import()` 即可（项目已是 `"type": "module"`）。**不要**用 `require()` 包装。

### 13.2 Float32Array byte order

`Buffer.from(f32.buffer)` 跟 `new Float32Array(buf.buffer, buf.byteOffset, dim)` 都依赖 V8 native byte order（x64 little-endian）。本项目锁定 Windows x64 + Linux x64（CI 未来用），无 ARM；所以一致。

如果未来上 ARM（树莓派部署？），需要显式用 DataView + `getFloat32(i, /* littleEndian */ true)` 写读，但 v0.7.2 不处理。

### 13.3 测试 DB 跟 live DB 隔离

L3 用临时目录（`mkdtempSync`），不污染 `./data/`。Live demo 用 `./data/live-demo-v072/`（gitignored，跟 v0.7.1 同模式）。

### 13.4 模型缓存路径

transformers.js 默认缓存到 `~/.cache/huggingface/`（Windows 是 `C:\Users\<user>\.cache\huggingface\`）。如要改路径，设 env var `HF_HOME` 或 `TRANSFORMERS_CACHE`，v0.7.2 不暴露这个配置（用户跑一次即缓存，没必要）。

### 13.5 ARCH + PRD 更新（DoD #9 + #10）

**ARCH §3.4** 改动（v0.7.2 commit 一并提交）：

> ~~lancedb（嵌入式，文件式，零外部依赖）~~
> **SQLite BLOB + 内存 cosine（嵌入式，零额外依赖；< 1 万 session 性能充足）；session 的 summary 嵌入向量存 `sessions.embedding` 列，查询时 `listWithEmbeddings()` 全读 + brute-force cosine 排序，top-K=2**

并在 ARCH 末尾变更记录加：

> 2026-06-XX: v0.7.2 偏离 §3.4 原方案 LanceDB → SQLite BLOB。理由：避免重演 better-sqlite3 Windows native binding 坑（[[feedback-better-sqlite3-windows-build]]）；当前场景 < 1 万 session，brute-force cosine < 5ms 充足；删 session 时 BLOB 自动级联无需双写。

**PRD §3.3** 改动：

> ~~向量维度 768~~ → **384（Xenova/all-MiniLM-L6-v2 int8）**
> ~~独立 collection `session_summaries`~~ → **`sessions` 表 `embedding` BLOB 列**
