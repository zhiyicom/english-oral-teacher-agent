## 2. `src/agent/tools/memory-search.ts` 设计

### 2.1 接口 & 公开 API

```ts
import type { DbHandle } from '../../storage/db.js'
import { createSessionsDao, type SessionsDao } from '../../storage/sessions.js'
import type { Embedder } from '../../memory/embedder.js'
import { retrieveRelevant, type RelevantSession } from '../../memory/retrieve-relevant.js'
import type { Tool } from '../tool-registry.js'
import { z } from 'zod'

const MemorySearchArgsSchema = z.object({
  query: z.string().min(1).max(200),
  top_k: z.number().int().min(1).max(5).default(2),
})

export function createMemorySearchTool(db: DbHandle, embedder: Embedder): Tool {
  const sessions: SessionsDao = createSessionsDao(db)
  return {
    name: 'memory_search',
    description: 'Search past sessions by semantic similarity. Returns top-K sessions with date, truncated summary, and keywords.',
    schema: MemorySearchArgsSchema,
    async execute(args: unknown): Promise<RelevantSession[]> {
      const { query, top_k } = MemorySearchArgsSchema.parse(args)
      const queryVec = await embedder.embed(query)
      const candidates = sessions.listWithEmbeddings()
      return retrieveRelevant({ candidates, queryVec, topK: top_k })
    },
  }
}
```

### 2.2 关键设计点

- **embedder 工厂注入**：跟 mark-mistake 的 sessionId 注入同模式。CLI 启动时 `createMemorySearchTool(db, embedder)` —— embedder 跟 cli.ts 已有的 v0.7.2 那个 embedder 同 instance（pipeline singleton 复用，避免重复加载）
- **top_k 范围 1-5**：跟 PRD 范围对齐（v0.7.2 startup 用 2；tool 默认 2；上限 5 防止 LLM 滥用）
- **query 长度 1-200**：防御 LLM 误传整本 summary；200 字符 ≈ 50 token embedding，<100ms cached
- **async execute**：embed 调用是 async（首次 1-2s 加载模型，后续 <100ms cached）；`Tool.execute` 返回类型宽化后兼容

### 2.3 错误处理

- zod schema 校验失败 → throw z.ZodError；CLI 现有 `try/catch` 块捕获，记 stderr `[cli] tool execute error: <msg>`，跳过该 turn tool
- embed 失败（网络断开 / 模型损坏）→ throw；同上捕获
- retrieveRelevant 永远不 throw（纯函数，candidates 为空返 []）

**fail-soft**：任何 tool 失败都不阻塞对话。学生继续聊。

## 3. `src/agent/tool-registry.ts` 改动

### 3.1 签名宽化

```ts
// Before (v0.7)
export interface Tool {
  readonly name: string
  readonly description: string
  readonly schema: z.ZodTypeAny
  execute(args: unknown): unknown
}

// After (v0.7.3)
export interface Tool {
  readonly name: string
  readonly description: string
  readonly schema: z.ZodTypeAny
  execute(args: unknown): unknown | Promise<unknown>   // ← widened
}
```

**向后兼容**：mark-mistake 的 `execute(args): unknown`（sync）仍然赋给 `unknown | Promise<unknown>`（sync 是 union 的子集）。`unknown` assignable to `unknown | Promise<unknown>` ✓。

CLI 改造时对所有 `tool.execute(...)` 调用统一加 `await`，sync 返回值 `await` 后仍是该值（V8 unwrap）。

### 3.2 不动的部分

- `ToolRegistry` class 本身（register / get / list）0 改
- `Tool` 的 name / description / schema 字段 0 改

### 4.2 新循环（v0.7.3，4 分支）

```ts
let response = ''
for await (const chunk of client.chatStream({ system, messages: history })) {
  if (chunk.type === 'text') response += chunk.delta
}
const parsed = parseToolCall(response)

if (!parsed) {
  // === 分支 1: 无 tool 块（v0.7 行为）===
  process.stdout.write(`${response}\n\n`)
  history.push({ role: 'assistant', content: response })
  messages.append({ sessionId: session.id, role: 'assistant', content: response })
} else if (parsed.name === 'mark_mistake') {
  // === 分支 2: mark_mistake（v0.7 sync 路径，含 dedup）===
  const display = stripToolCall(response, parsed)
  const original = typeof parsed.args.original === 'string' ? parsed.args.original : ''
  if (original && markedOriginals.has(original)) {
    process.stderr.write(`[cli] tool dedup: skipped (already marked: "${original}")\n`)
  } else {
    const tool = toolRegistry.get(parsed.name)
    if (!tool) {
      process.stderr.write(`[cli] tool unknown: ${parsed.name}\n`)
    } else {
      try {
        await tool.execute(parsed.args)  // 统一 await（兼容 sync）
        if (original) markedOriginals.add(original)
        process.stderr.write(`[cli] tool call: ${parsed.name}(${JSON.stringify(parsed.args)})\n`)
      } catch (err) {
        process.stderr.write(`[cli] tool execute error: ${(err as Error).message}\n`)
      }
    }
  }
  process.stdout.write(`${display}\n\n`)
  history.push({ role: 'assistant', content: display })
  messages.append({ sessionId: session.id, role: 'assistant', content: display })
} else if (parsed.name === 'memory_search') {
  // === 分支 3: memory_search（A+B 路径）===
  const tool = toolRegistry.get(parsed.name)
  if (!tool) {
    process.stderr.write(`[cli] tool unknown: ${parsed.name}\n`)
    const display = stripToolCall(response, parsed)
    process.stdout.write(`${display}\n\n`)
    history.push({ role: 'assistant', content: display })
    messages.append({ sessionId: session.id, role: 'assistant', content: display })
  } else {
    let result: unknown
    try {
      result = await tool.execute(parsed.args)
      process.stderr.write(`[cli] tool call: ${parsed.name}(${JSON.stringify(parsed.args)})\n`)
    } catch (err) {
      process.stderr.write(`[cli] tool execute error: ${(err as Error).message}\n`)
      // 失败也要走 2nd-call，让 LLM 知道搜不了（不能 silently drop）
      result = { error: (err as Error).message }
    }

    // 把原始 1st-call 推 history（带 tool 块，让 LLM 知道它自己叫了 tool）
    history.push({ role: 'assistant', content: response })
    messages.append({ sessionId: session.id, role: 'assistant', content: response })

    // 推 synthetic user message 含 result（marker 让 Replay fixture 路由到 2nd-call fixture）
    const resultText = formatToolResult(parsed.name, result)
    const syntheticUserContent = `[tool_result_v073]\n${resultText}`
    history.push({ role: 'user', content: syntheticUserContent })
    // 注：synthetic user message 不写 messages 表（不是真学生输入；只是 LLM 反馈循环用）

    // 2nd LLM call（非流式，匹配 summarize() 现有形状）
    process.stderr.write(
      `[cli] tool 2nd-call: ${parsed.name}(top_k=${typeof parsed.args.top_k === 'number' ? parsed.args.top_k : 2})\n`,
    )
    const followup = await client.chat({ system, messages: history })
    // Strip safety pass：万一 2nd-call response 又含 tool 块（prompt 禁了但 LLM 可能误调），strip 不 execute
    const followupParsed = parseToolCall(followup.content)
    const followupDisplay = followupParsed ? stripToolCall(followup.content, followupParsed) : followup.content
    if (followupParsed) {
      process.stderr.write(`[cli] tool 2nd-call strip: ignored <tool>${followupParsed.name}</tool> (no recursion in v0.7.3)\n`)
    }

    process.stdout.write(`${followupDisplay}\n\n`)
    history.push({ role: 'assistant', content: followupDisplay })
    messages.append({ sessionId: session.id, role: 'assistant', content: followupDisplay })
  }
} else {
  // === 分支 4: 未知 tool（v0.7 fallback 行为）===
  const display = stripToolCall(response, parsed)
  process.stderr.write(`[cli] tool unknown: ${parsed.name}\n`)
  process.stdout.write(`${display}\n\n`)
  history.push({ role: 'assistant', content: display })
  messages.append({ sessionId: session.id, role: 'assistant', content: display })
}
```

### 4.3 `formatToolResult` 辅助

```ts
function formatToolResult(toolName: string, result: unknown): string {
  if (toolName !== 'memory_search') {
    return JSON.stringify(result)  // fallback（v0.7.3 只有 memory_search，但留扩展点）
  }
  if (!Array.isArray(result) || result.length === 0) {
    return `[Tool result: memory_search]\n\nNo relevant past sessions found.`
  }
  const lines = [`[Tool result: memory_search]\n`, `Top ${result.length} most relevant past sessions (by semantic similarity to your query):\n`]
  for (let i = 0; i < result.length; i++) {
    const r = result[i]
    const dayWord = r.daysAgo === 1 ? 'day' : 'days'
    const truncated = r.summary.length > 80 ? `${r.summary.slice(0, 80)}...` : r.summary
    lines.push(`${i + 1}. ${r.daysAgo} ${dayWord} ago: "${truncated}" (keywords: ${r.keywords.join(', ')})`)
  }
  return lines.join('\n')
}
```

### 4.4 关键设计点

| 点 | 决策 | 理由 |
|---|---|---|
| **`messages.append` 只写 assistant 行为** | 1st-call 写（含 tool 块）、synthetic user message **不写**、2nd-call 写 | 学生在 UI / transcript 看 tool 过程不需要（最终回复已是连贯文本）；student-facing 视角只需 2 条 assistant turn |
| **2nd-call 用 `client.chat`（非流式）** | 不流式 | 跟 summarize() 现有形状对齐；streaming 价值不大（2nd-call 是 follow-up 而非开场，TTFT 容忍度高）；省流式事件处理代码 |
| **2nd-call 失败处理** | try/catch 包整个 2nd-call；失败时回退到 1st-call 的 strip 文本 | 跟 tool 失败 fail-soft 原则一致；不阻塞对话 |
| **tool 失败时仍推 synthetic user + 跑 2nd-call** | result = `{ error: msg }`，让 LLM 知道搜不了 | 不能 silently drop；LLM 应该能告诉学生"抱歉我搜不到" |
| **2nd-call 也有 tool 块** | strip safety pass，stderr 记 strip 日志 | prompt 防御 + 运行时防御双保险；不递归（v0.7.3 范围）|
| **synthetic user message 不写 messages 表** | 只进 history（LLM 视角）| messages 表是 student↔teacher transcript；tool feedback 是内部机制 |
| **stderr log `[cli] tool 2nd-call: memory_search(top_k=N)`** | 新增 | 跟 `[cli] tool call:` 形状对齐；live demo 可 grep |

### 4.5 历史同步（v0.7.1 recentMistakes、v0.7.2 relevantPast）

`buildFinalSystem(..., recentMistakes, isFirstTurn ? relevantPast : [])` 调用不变（行 271-278）。`isFirstTurn` 冻结逻辑不变。**唯一变化是 turn 内部 history 多推 1 条 assistant + 1 条 user**——但这 2 条只在 memory_search 分支推，不影响 [System Context] 渲染。

### 11.1 Case 1: "memory_search triggers 2nd LLM call"

```ts
it('memory_search: input triggers tool call + 2nd LLM call + stdout shows 2nd-call content', async () => {
  const dataDir = makeTmpDataDir()
  // Pre-seed 1 session (v0.7.2 path — so memory_search 1st-call has a non-empty candidate set)
  // Actually NOT needed: empty result is also valid, just want to verify the loop runs.
  // Skip pre-seed to keep test simple.

  const result = await runCli({
    dataDir,
    input: 'what did i play\nexit\n',
    env: { MOCK_TIME: '1' },
  })

  // 1st-call LLM called (we know this happened because we got to the tool parse)

  // no <tool> leaked
  expect(result.stdout).not.toContain('<tool>')
  expect(result.stdout).not.toContain('</tool>')
  // 1st-call "Let me check" was stripped (not in stdout)
  expect(result.stdout).not.toContain('Let me check')
}, 60_000)
```

## 13. 风险与边缘 case

| 风险 | 触发 | 处置 |
|---|---|---|
| **1 round-trip 加 token** | 每次 memory_search 加 ~200 token result + 2nd-call 完整回复 | topK=2 × ~100 token = 200 / call；3 calls / 30-min session = 600 token，state-machine 30-min cap 内 OK；v0.7.x context budget sprint 单独处理（已记 backlog）|
| **2nd-call LLM 又调 tool** | 2nd-call response 含 `<tool>` 块 | prompt 写"do not call any tool in second response" + cli strip safety pass（不 execute）|
| **2nd-call 失败（LLM 错误 / 网络）** | 2nd `client.chat` reject | try/catch 包整个 2nd-call；失败时回退到 1st-call 的 strip 文本 + stderr log |
| **Synthetic user message 污染 LLM context** | `[tool_result_v073]` marker 进入 history | marker 是设计意图；LLM 训练见过类似 function_calling 模式；不影响生成 |
| **Fixture match 误中** | 2nd-call fixture 的 `userSaysContains: "[tool_result_v073]"` 跟某 turn 用户输入撞 | marker 唯一，user 不会输入；substring 严格匹配 |
| **memory_search result 0 时 LLM 行为奇怪** | DB 无 embedding | result text 写 "No relevant past sessions found"；LLM 据此回应（live demo 验证）|
| **embedder 每次 call 都要嵌 query** | 同一 query 重复调 | cached <100ms；当前不缓存（v0.7.3 范围外）；v0.8+ 加 query→vec LRU |
| **2nd-call 文本含 tool 块被 strip** | 学生看到 stripped 文本（可能有奇怪空行）| stripToolCall 已 trim + collapse 3+ newlines → 2；输出仍连贯 |
| **tool 失败 result 含 stack trace** | embed 网络断开 throw | formatToolResult 对 `error` key 简化输出：`{ error: 'connect timeout' }` → "Tool error: connect timeout"（live demo 验证）|
