`z.coerce.number()` 自动把 string 变 number（dotenv 加载出来都是 string）。`default(6000)` 让 `.env` 没设也不会报错（fallback 静默）。`max(200_000)` 是 sanity check（超过这个值必然错）。

**加载链**（跟其他 `.env` 字段一致）：

1. shell 导出 `LLM_CONTEXT_BUDGET_TOKENS=N` → process.env 拿到
2. `.env` 文件里设了 → `dotenv/config` 加载后用
3. 都没设 → zod default 6000
4. 设了但 zod 校验 fail（比如 `abc`、负数、`200001`）→ `loadEnv()` 抛 Error，CLI 启动 fail

跟 `LLM_MAX_TOKENS` 现有的失败模式完全一致；operator 看到清晰错误信息。

## 3. Anthropic SDK: 捕获 usage + 支持 system blocks

### 3.1 改 `src/llm/anthropic.ts`

`message_start` 事件有 `message.usage`：

```ts
type AnthropicMessageStart = {
  type: 'message_start'
  message: {
    usage: {
      input_tokens: number
      output_tokens: number
      cache_creation_input_tokens?: number
      cache_read_input_tokens?: number
    }
  }
}
```

`client.messages.create({ stream: true })` 第一个 event 就是 `message_start`。需要在 `chatStream` 里 yield 一个新 chunk type：

```ts
async function* chatStream(opts: ChatOpts): AsyncIterable<ChatChunk> {
  const stream = await client.messages.create({
    model: env.LLM_MODEL_MAIN,
    max_tokens: opts.maxTokens ?? env.LLM_MAX_TOKENS,
    temperature: opts.temperature ?? env.LLM_TEMPERATURE,
    system: opts.systemBlocks ?? opts.system,   // ← string | SystemBlock[] 都接受
    messages: opts.messages.map((m) => ({
      role: m.role === 'system' ? 'user' : m.role,
      content: m.content,
    })),
    stream: true,
  })

  for await (const event of stream as unknown as AsyncIterable<AnthropicEvent>) {
    if (event.type === 'message_start') {
      const u = event.message.usage
      yield {
        type: 'usage',
        inputTokens: u.input_tokens,
        outputTokens: u.output_tokens,
        cacheReadTokens: u.cache_read_input_tokens ?? 0,
        cacheCreationTokens: u.cache_creation_input_tokens ?? 0,
      }
    } else if (event.type === 'content_block_delta') {
      const delta = event.delta
      if (!delta) continue
      if (delta.type === 'text_delta' && typeof delta.text === 'string') {
        yield { type: 'text', delta: delta.text }
      } else if (delta.type === 'thinking_delta' && typeof delta.thinking === 'string') {
        yield { type: 'thinking', delta: delta.thinking }
      }
    }
  }
}
```

### 3.2 改 `src/llm/types.ts`

```ts
export type SystemBlock = {
  type: 'text'
  text: string
  cache_control?: { type: 'ephemeral' | 'persistent' }
}

export interface ChatOpts {
  /** Legacy: pass a single string. No cache_control applied. */
  system?: string
  /** v0.7.5: pass an array of text blocks. First block can carry cache_control. */
  systemBlocks?: SystemBlock[]
  messages: Message[]
  temperature?: number
  maxTokens?: number
}

export type UsageChunk = {
  type: 'usage'
  inputTokens: number
  outputTokens: number
  cacheReadTokens: number
  cacheCreationTokens: number
}

export type ChatChunk =
  | { type: 'text'; delta: string }
  | { type: 'thinking'; delta: string }
  | UsageChunk

export interface ChatResult {
  content: string
  thinking?: string
  usage?: UsageChunk
}
```

`chat()` wrapper：

```ts
async function chat(opts: ChatOpts): Promise<ChatResult> {
  let text = ''
  let thinking = ''
  let usage: UsageChunk | null = null
  for await (const chunk of chatStream(opts)) {
    if (chunk.type === 'text') text += chunk.delta
    else if (chunk.type === 'thinking') thinking += chunk.delta
    else if (chunk.type === 'usage') usage = chunk
  }
  return {
    content: text,
    thinking: thinking.length > 0 ? thinking : undefined,
    usage: usage ?? undefined,
  }
}
```

## 4. Prompt builder 拆分

`src/agent/prompt-builder.ts` 加新函数 `buildFinalSystemSplit`，**旧 `buildFinalSystem` 不变**（既有 L1 + L3 测试不破）：

```ts
export interface FinalSystemSplit {
  static: string   // SOUL + AGENTS + STUDENT (+ TOOLS if present) — cacheable
  dynamic: string  // [System Context] block — changes every turn
}

export function buildFinalSystemSplit(
  systemPrompt: SystemPrompt,
  state: SessionState,
  lastReview: LastReview | null = null,
  activeTopics: TopicStat[] = [],
  recentMistakes: Mistake[] = [],
  relevantPast: RelevantSession[] = [],
): FinalSystemSplit {
  return {
    static: buildSystemString(systemPrompt),
    dynamic: '\n\n' + buildSystemContext(state, lastReview, activeTopics, recentMistakes, relevantPast),
  }
}

// Legacy: still returns joined string. Used by:
// - tests/agent/prompt-builder.test.ts (5 L1 tests)
// - src/cli.ts:339 (ctx-block dump for stderr)
export function buildFinalSystem(...): string {
  const { static: s, dynamic: d } = buildFinalSystemSplit(...)
  return s + d
}
```

**注意**：`buildFinalSystemSplit` 里 `dynamic` 前面加 `\n\n` 跟原 `buildFinalSystem` 的 join 行为保持一致，避免 ctx-block 测试对 spacing 的脆性。

## 5. Truncate history 算法

新文件 `src/agent/truncate-history.ts`：

```ts
import type { Message } from '../llm/types.js'

/**
 * Cheap token estimator: 1 token ≈ 4 characters. Conservative for English
 * (over-counts by ~50% for English, ~25% for Chinese mixed). Used for
 * pre-truncate decisions; SDK's actual usage is the ground truth.
 */
export function estimateTokens(text: string): number {
  return Math.ceil(text.length / 4)
}

export function estimateMessagesTokens(messages: Message[]): number {
  return messages.reduce((sum, m) => sum + estimateTokens(m.content), 0)
}

export interface TruncateResult {
  messages: Message[]
  dropped: number
}

/**
 * Drop oldest user/assistant pairs from `messages` until estimated total
 * (messages + system) ≤ `budget`. Always keeps the last 2 messages
 * (the most recent user + assistant pair) even if they exceed budget —
 * stripping to empty would break the chat loop.
 *
 * Truncation is in pairs of 2 messages (1 turn). Splits a pair never:
 * either both are kept or both are dropped. Synthetic user messages
 * (e.g. `[tool_result_v073]` from v0.7.3 A+B protocol) are treated the
 * same as natural user messages.
 */
export function truncateHistory(
  messages: Message[],
  budget: number,
  systemSize: number = 0,
): TruncateResult {
  let current = messages
  let dropped = 0
  while (current.length > 2 && estimateMessagesTokens(current) + systemSize > budget) {
    current = current.slice(2)
    dropped += 1
  }
  return { messages: current, dropped }
}
```

**示例**：

```
budget = 100 tokens (~ 400 chars)
system = 100 chars (~ 25 tokens), so messages budget ≈ 75 tokens (~ 300 chars)

input messages:
  [0] user:    "hi"                            (~ 2 tokens)
  [1] assistant: "Hi there! How are you?"      (~ 8 tokens)
  [2] user:    "i played minecraft"            (~ 5 tokens)
  [3] assistant: "Nice! What did you build?"   (~ 9 tokens)
  [4] user:    "castles and creeper walls"     (~ 6 tokens)
  [5] assistant: "Cool! Do you play survival?" (~ 8 tokens)
  total ≈ 38 tokens — under budget, no truncate

After 5 more turns (~ 50 tokens more): total ≈ 88 tokens — over budget
truncate: drop [0,1] (2 msgs = 1 turn), now total ≈ 78 tokens — under
truncate: keep going
```

## 6. CLI 集成

`src/cli.ts` 关键改动（伪代码，标出 diff 位置）：

```ts
// === boot: env 已含 LLM_CONTEXT_BUDGET_TOKENS（v0.7.5 新加字段） ===
import { buildFinalSystemSplit } from './agent/prompt-builder.js'
import { truncateHistory, estimateTokens } from './agent/truncate-history.js'

const env = loadEnv()
process.stderr.write(`[cli] context budget: ${env.LLM_CONTEXT_BUDGET_TOKENS} tokens\n`)   // NEW

let hasWarnedContext = false   // NEW: per-session warn guard
const TOKEN_WARN_RATIO = 0.8   // hardcoded; not user-configurable

// === turn loop, after history.push(user input) ===

// NEW: truncate history if over budget
const system = buildFinalSystemSplit(systemPrompt, state, ...)
const systemSize = estimateTokens(system.static) + estimateTokens(system.dynamic)
const { messages: trimmedHistory, dropped } = truncateHistory(history, env.LLM_CONTEXT_BUDGET_TOKENS, systemSize)
if (dropped > 0) {
  process.stderr.write(`[cli] truncated: dropped ${dropped} oldest turn(s) to fit budget (${env.LLM_CONTEXT_BUDGET_TOKENS} tokens)\n`)
}
history = trimmedHistory  // reassign (history is `let`, not `const`)

// NEW: build systemBlocks with cache_control on the static portion
const systemBlocks = [
  { type: 'text' as const, text: system.static, cache_control: { type: 'ephemeral' as const } },
  { type: 'text' as const, text: system.dynamic },
]

// === chatStream call: new systemBlocks param ===
let response = ''
let lastUsage: UsageChunk | null = null
for await (const chunk of client.chatStream({
  systemBlocks,   // NEW
  messages: history,
})) {
  if (chunk.type === 'text') response += chunk.delta
  else if (chunk.type === 'usage') lastUsage = chunk
  // 'thinking' chunks are not used in main turn (only in chat() for memory_search 2nd)
}

// === post-call: log usage + 80% warn ===
if (lastUsage) {
  process.stderr.write(
    `[cli] tokens: input=${lastUsage.inputTokens} cache_read=${lastUsage.cacheReadTokens} ` +
      `cache_creation=${lastUsage.cacheCreationTokens} output=${lastUsage.outputTokens}\n`,
  )
  if (
    !hasWarnedContext &&
    lastUsage.inputTokens / env.LLM_CONTEXT_BUDGET_TOKENS >= TOKEN_WARN_RATIO
  ) {
    const pct = ((lastUsage.inputTokens / env.LLM_CONTEXT_BUDGET_TOKENS) * 100).toFixed(0)
    process.stderr.write(
      `[cli] warn: context usage ${pct}% of budget (${lastUsage.inputTokens}/${env.LLM_CONTEXT_BUDGET_TOKENS} tokens)\n`,
    )
    hasWarnedContext = true
  }
}
```

**Stderr 输出总览**（正常 turn，无 truncate 无 warn）：

[cli] ctx: WARM_UP elapsed=0.0 silence=0.0

## 10. 风险细化

| 风险 | 触发 | 缓解 | 验证方式 |
|---|---|---|---|
| `message_start` 事件 schema 跟 SDK 版本对不上 | SDK 升级到 0.42+ | 已有 `as unknown as AsyncIterable<AnthropicEvent>` 转义；如 schema 变，runtime fail 立刻暴露 | L1 测 + live demo |
| `cache_control` 不被 SDK 接受 | 旧 SDK 版本 | 0.41 是已安装版本（[package.json](package.json) 验证） | 启动若 fail 立即知；改 SDK 升级不在本 sprint |
| `truncateHistory` 把 WARM_UP topic intro 截掉 | session 第 1 turn 设了 Minecraft 主题，第 30 turn 时 LLM 失忆 | v0.7.5 接受；v0.7.6 加 "anchor pair"（始终保留 first user/assistant 对） | Live demo 长 session 测 |
| chars/4 估算偏差 | 中文 / emoji / 重复词 / URL | 故意保守（chars/4 比真实略高）→ truncate 提前触发，但**不会**让 LLM 收到超 budget 的输入 | chars/4 vs SDK usage 散点对比（DoD 留 metric 但 v0.7.5 不实施） |
| `buildFinalSystem` 既有 5 个 L1 测试 fail | 签名微变 | 旧签名不变；新加 `buildFinalSystemSplit` | pnpm test 全绿 |
| `ChatResult.usage` 既有 Replay fixture 没填 usage | 老 fixture 缺字段 | `usage` 字段在 type 里 optional；CLI 走 "no usage → skip log" 路径 | L3 测试 |
