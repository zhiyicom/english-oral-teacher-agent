## 1. 总览

PRD §5.3 写了"单轮总输入 < 6K"是 v1.0 硬性起点。当前 ([v0.7.3](./v0.7.3-test-report.md)) 没有任何 token 计数器、history 截断、prompt cache。30 turn verbose session 可破 15K 撞 MiniMax-M3 上下文墙。

**v0.7.5 做 3 件事**：

1. **可配置 budget + token counter**：在现有 `.env` 加 `LLM_CONTEXT_BUDGET_TOKENS=6000`（默认 6000 per PRD §5.3，范围 1-200000）。`src/config/env.ts` 加 zod 字段，跟 `LLM_MAX_TOKENS` 同一模式。Anthropic API response 里的 `usage.input_tokens` 拿出来打 log（`[cli] tokens: input=X cache_read=Y output=Z`）。`ChatResult.usage` 字段已 type-def 好了，只是一直没填 —— **v0.7.5 一并填上**。
2. **Sliding window truncate**：预估 input tokens 超 budget 时，把 history 最旧的 user/assistant 对丢掉。预估用 `chars/4`（零依赖、保守），**不是** SDK 的 usage（SDK 只在 call 之后才知道，pre-truncate 用不了）。SDK usage 用来 **post-call 校验**。
3. **80% warn + cache_control**：实际 input > 80% budget 时 `[cli] warn: context usage X%` 一次（不每 turn 都打）。`buildFinalSystem` 重构返回 `{ static, dynamic }` 两段，static 段（SOUL + AGENTS + USER）加 Anthropic `cache_control: { type: 'ephemeral' }`，~90% 静态段成本下降。

**用户配置**：所有基础配置统一放在 `.env`（已 gitignored），新增 1 行 `LLM_CONTEXT_BUDGET_TOKENS=6000`。**配置错误 → CLI 启动 fail**，跟 `env.ts` 现在的行为一致。**配置缺失 → fallback 默认 6000**（zod default）。

## 3. 用户可配置字段

**写在 `.env`**（跟 API key 同一份，已 gitignored）：

```bash
# === Context budget (v0.7.5) ===
# Single-turn input token cap. CLI truncates oldest history when over.
# Lower = cheaper, more truncation. Higher = more context for the LLM.
LLM_CONTEXT_BUDGET_TOKENS=6000
```

| 字段 | 类型 | 默认 | 范围 | 说明 |
|---|---|---|---|---|
| `LLM_CONTEXT_BUDGET_TOKENS` | integer | 6000 | `1` ~ `200000` | 单轮 input token 上限。超此值时 sliding window truncate；80% 时 warn |

**`LLM_CONTEXT_BUDGET_WARN_RATIO` (0.8) 和 truncate threshold (1.0) 写死在代码里**，不做成配置。Less surface area；如果将来要可调，加 v0.7.6+ 配置项。

**加载链**（跟其他 `.env` 字段一致）：

1. `process.env.LLM_CONTEXT_BUDGET_TOKENS` 已设（shell 导出）→ 用 process.env 值
2. `.env` 文件里设了 → dotenv 加载后用
3. `.env` 没设 → zod default 6000
4. 设了但 zod 校验 fail（比如 `abc`、负数）→ CLI 启动 fail（跟 `env.ts:23-27` 现有行为）

## 4. 关键交互流程

### 4.1 启动

```
loadEnv()                                          // src/config/env.ts (modified: + LLM_CONTEXT_BUDGET_TOKENS)
  ↓
CLI 进入主循环（env.LLM_CONTEXT_BUDGET_TOKENS 已可用）
```

### 4.2 每 turn

```
1. 收到 user input
2. history.push(user input)
3. estimatedInputTokens = sum(messages) / 4       // chars/4 estimator
4. if estimatedInputTokens > contextBudgetTokens:
     history = truncateHistory(history, contextBudgetTokens)  // drop oldest
     [cli] truncated: dropped N pairs, history now K tokens (estimated)
5. buildFinalSystem(prompt, state, ...) → { static, dynamic }
6. chatStream({ system: [{type:'text', text: static, cache_control: {type:'ephemeral'}}, {type:'text', text: dynamic}], messages: history })
7. response, usage = ...
8. [cli] tokens: input=X output=Y
9. if X / contextBudgetTokens >= 0.8: [cli] warn: context usage X% (budget=Y)  // log once per session
10. 正常输出 response
```

## 6. 风险与缓解

| 风险 | 触发 | 缓解 |
|---|---|---|
| **chars/4 估算偏差大** | 英文偏短 (0.25 tok/char)，中文偏长 (0.5 tok/char) | estimator 走 **保守** (chars/4 ≈ 1.5x 真实)；pre-truncate 用 estimator，post-validate 用 SDK usage。如果 SDK usage 多次超 budget，加 metric 触发 v0.7.6 重新设计 |
| **Anthropic SDK 不暴露 `cache_read_input_tokens`** | 验证 cache hit 难度 | DoD #6 改为"看 SDK response 是否有 cache 字段"；不强求精确数字 |
| **`buildFinalSystem` 重构影响现有 L3 测试** | 既有测试可能 assert 整个 system 是 string | 把旧 `buildFinalSystem` 保留为 deprecated wrapper，**新 L3 测试用 `buildFinalSystemSplit()`**；或者用 default-arg 让旧调用仍可工作 |
| **Sliding window 丢掉 WARM_UP 早期 topic intro** | 学生说 "i played minecraft" 在第 1 turn，truncate 后 LLM 失忆 | v0.7.5 接受；v0.7.6 再加 "anchor pair"（始终保留 first user/assistant 对作为 topic intro） |
| **Per-session warning spam** | 80% warn 一次/每 turn？ | 改成**一次/每 session**（per-session bool flag） |
| **Env 配置被 shell 覆盖** | 用户在 shell 导出 `LLM_CONTEXT_BUDGET_TOKENS=2000` 会覆盖 .env | 这是 dotenv 标准行为；用户在 README 看到 .env.example 就知道；shell 优先级最高是惯例 |
