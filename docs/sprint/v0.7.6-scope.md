# Sprint v0.7.6 Scope — v0.7.x Wrap-up: Robustness + Context + Topics

> **v0.7.x 收官 sprint**。v0.7.5 是 v0.7.x 最后一大特性 sprint（context budget），本 sprint 把 v0.7.x 期间留的 backlog 一并清掉（A/B/D 三组），**之后直接进 v0.8 Web UI**，中间不再插别的 sprint。
> **不做**：C（PRD §7 错误处理剩余项 — 网络/磁盘/崩溃恢复）、E（mark_vocabulary/mark_homework tools）、F（F6 voice/F7 topic library/F8 session archive）、G（v0.8 本身）、H（CI/CHANGELOG）。

## 1. 总览

PRD §3.5.3（选题策略）、§5.3（in-session summarizer）、§7（错误处理）都有未完成的子项零散分布在 v0.7.4-v0.7.5 留的 backlog 里。v0.8 Web UI 之前的 CLI 必须稳定、上下文管理精细、选题逻辑完整 —— 否则 UI 上去之后用户能看到的第一个 bug 就是 PRD gap。

**v0.7.6 做 3 件事**：

1. **A — Production robustness**：V751-002 chatStream try/catch + 1x retry + 友好 stderr/stdout（PRD §7.1 LLM API 错误的最小集）；V741-001 live-validate scenario 4 fix（dev tool 覆盖率补全，1-2h）。
2. **B — Context refinement**（v0.7.5 sliding window 的补丁）：B1 anchor pair（永远保留 first user/assistant）/ B2 in-session summarizer tool（PRD §5.3 第二段，agent 主动压缩当前 history）/ B3 per-segment token cost logging（哪个 `[System Context]` 段最大）/ B4 `cache_control: ephemeral` 加到 messages[]（live demo 已验证 cache ✅）。
3. **D — Topic engine completion**：D1-D4 实现 PRD §3.5.3 完整选题策略（硬排除 30d / 软偏好低 count / 兴趣匹配 / 加权随机）+ D5 `topic_select(phase, exclude_recent_days)` tool 把策略包给 LLM 调用。

**用户配置**：无新增 `.env` 字段。V751-002 retry 次数、D5 topic_select exclude_recent_days 默认值都写死在代码里。Less surface area；如果将来要可调，再加 v0.7.7+ 配置项。

**LLM 协议 / Prompt / Fixture 变更**：B2 新加 1 个 tool（`summarize_history` 之类），需要在 `prompts/tools.md` 加 section；D5 新加 1 个 tool（`topic_select`），同样加 section。2 个新 fixture 用于 L3 测试。**B2/D5 不动 A+B 协议本身**，复用 v0.7.3 memory_search 的 A+B 路径。

## 4. 关键交互流程

### 4.1 V751-002 错误处理（A）

```
chatStream(...) throws
  ↓
catch (err):
  classification = classifyError(err)  // 4xx/5xx/network/content-filter/other
  process.stderr.write(`[cli] llm error: ${classification} (${err.message})`)
  ↓
  if retry-eligible (5xx/network) and attempts < 2:
    process.stderr.write(`[cli] retrying in 1s...`)
    sleep(1000)
    continue loop  // 不增加 attempts 计数器? 或增加? 设计时定
  else:
    // 持续失败降级
    process.stdout.write(`\n[Teacher]: Sorry, I lost my train of thought. Could you say that again? 😊\n\n`)
    summary = await summarizer.summarize(currentHistory)
    sessions.markEnded(sessionId, {summary, reason: 'llm_error'})
    process.stderr.write(`[cli] session auto-saved: ${sessionId}`)
    process.exit(1)
```

### 4.2 Anchor pair（B1）

```
启动:
  const firstPair = history.slice(0, 2)  // first user + first assistant
  ↓
每 turn:
  const truncated = truncateHistory(history, budget, {
    anchorPair: firstPair,
  })
  // truncateHistory: 在 drop oldest pairs 时永远跳过 anchor pair
  // 极端情况：anchor pair 本身就 > budget → log warning + 不 truncate（保留全部）
```

### 4.3 In-session summarizer tool（B2）

```
1st call: LLM 输出
  <tool>summarize_history({"target_tokens": 500})</tool>
  ↓
CLI parse → execute(tool, args)
  ↓
  // 用 v0.5 summarizer 把 history 压缩成 ≤ 500 token 的 summary
  const summary = await summarizer.summarizeToTokens(history, 500)
  // 把 history 替换成: [anchor_pair..., summary_message, ...recent_3_turns]
  ↓
CLI push assistant (with tool block) + synthetic user (with `[v076_history_summary]` marker)
  ↓
2nd call: LLM 收到 history 已压缩的提示；输出 without tool block
  ↓
CLI strip + write to stdout
```

### 4.4 Topic engine（D1-D5）

```
新 session 启动 / agent 调 topic_select tool:
  candidates = topics.list()  // 7 个 seed topics
  candidates = filterHardExclude(candidates, now, days=30)
  candidates = sortByCountAsc(candidates)
  candidates = boostByInterest(candidates, student.interests)
  winner = pickTopic(candidates, ..., rng)  // 加权随机
  return {slug, title, est_minutes}
```

## 6. 风险与缓解

| 风险 | 触发 | 缓解 |
|---|---|---|
| **大 sprint 范围失控** | 11 个 item / ~10 天；如某 axis 拖延影响全局 | 每个 axis 单独 commit；3 axis 内部按顺序 A → B → D 推进；A 必须最先（A 是 B/D 的前置 — chatStream 错误处理是 B2/D5 调 chat 的基础）|
| **B2 in-session summarizer 引入新 tool 协议复杂度** | A+B 协议 + history rewrite 逻辑 | 复用 v0.7.3 memory_search 的 A+B 路径；history rewrite 在 execute() 完成后由 CLI 做（不是 tool 自己）|
| **B4 cache_control on messages 可能 SDK 不支持** | 实测发现 SDK 报错 | 用 try/catch 包住；fallback 到不传 cache_control，log 一次 warn |
| **D5 topic_select 选错话题导致体验差** | 加权随机种子不稳 | L1 测覆盖 D1-D4 边界；live demo 验证 |
| **V751-002 retry 把 transient error 升级成 permanent** | retry 1x 后网络仍断 | 1x retry 是上限；失败立即 fallback（A3）|
| **3 axes 同时改 cli.ts 主循环** | 合并冲突 | A → B → D 顺序推进；每 axis 完成后立即 commit（中间态可运行）|
| **PRD §3.5.3 字段命名跟 v0.6 Jaccard 不一致** | 现有 `topic_stats.discussion_count` 是 UPSERT 计数，D2 soft preference 要按它排序 — 字段已存在 | 不需要 schema 变更；复用 `topic_stats` |
| **`prompts/tools.md` 加 2 section 影响 prompt token 数** | SOUL+AGENTS+USER prompt 总量涨 | 2 个 section 各 5-10 行；增量 < 200 tokens，cache_control: ephemeral 已验证有效，cost impact 可忽略 |
