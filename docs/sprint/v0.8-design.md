### 2.2 为什么不 monorepo（pnpm workspace）？

考虑过 pnpm workspace（`apps/server` + `apps/web` + `packages/shared`）。**最终选单 package + 独立 `web/package.json`**：

| 优 | 缺 |
|---|---|
| 不引入 workspace 复杂度 | web 的 deps 不会自动 hoist（pnpm 自动 symlink）|
| 用户 clone → `pnpm install` → `pnpm --dir web install` → `pnpm dev-web` 一气呵成 | 升级依赖需要 `pnpm --dir web update` |
| 跟 v0.7.x 一致（不破坏 git history）| shared types 没地方放（暂时通过 `web/src/lib/types.ts` 手动同步）|
| 1 个 root lockfile 简单 | |

**未来如果 types 共享变多**（v0.9+ 加 F6 语音类型 / v1.0 加 tool 类型），可以再升级到 monorepo。v0.8 MVP 不做。

## 3. Server API contract

### 3.1 REST endpoints

#### `GET /api/sessions`

**用途**：主界面 session 列表

**Response 200**:
```json
{
  "sessions": [
    {
      "id": "abc-123",
      "startedAt": "2026-06-10T18:30:00Z",
      "endedAt": "2026-06-10T18:55:00Z",
      "durationMin": 25,
      "phaseHistory": ["WARM_UP", "MAIN_ACTIVITY", "WRAP_UP", "END"],
      "summary": "Student practiced talking about Minecraft castles...",
      "keywords": ["minecraft", "castle", "creeper"],
      "topicMatch": "minecraft"
    },
    ...
  ]
}
```

**排序**：按 `startedAt DESC`。

#### `POST /api/sessions`

**用途**：[开始新练习] 按钮

**Request**:
```json
{ "interests": ["minecraft", "sports"] }  // 可选，默认 []
```

**Response 201**:
```json
{ "id": "abc-456" }
```

**副作用**：在 DB 插入一行 session（startedAt = now, endedAt = null, phase_history = ["WARM_UP"]）。

#### `GET /api/sessions/:id`

**用途**：历史详情页

**Response 200**:
```json
{
  "id": "abc-123",
  "startedAt": "...",
  "endedAt": "...",
  "durationMin": 25,
  "phaseHistory": [...],
  "summary": "...",
  "keywords": [...],
  "topicMatch": "minecraft",
  "messages": [
    { "id": 1, "role": "user", "content": "hi", "ts": "..." },
    { "id": 2, "role": "assistant", "content": "Hi there! ...", "ts": "..." },
    ...
  ]
}
```

#### `GET /api/settings`

**用途**：settings 页面加载

**Response 200**:
```json
{
  "voice_enabled": false,
  "voice_speed": 1.0,
  "voice_accent": "en-US",
  "font_size": 14,
  "show_debug": false
}
```

**来源**：USER.md frontmatter（持久化项）/ 内存默认（session-only 项）。

#### `PUT /api/settings`

**用途**：保存设置

**Request**:
```json
{
  "voice_enabled": true,         // 持久化到 USER.md
  "font_size": 16,                // 持久化到 USER.md (但 PRD §9.2 标 session-only；v0.8.4 实现为 session-only via localStorage，不调此 endpoint)
  "show_debug": true              // session-only
}
```

**Response 200**:
```json
{ "ok": true, "persisted": ["voice_enabled"] }
```

**副作用**：
- 持久化项 → 原子写 USER.md frontmatter（proper-lockfile 锁）
- session-only 项 → 内存更新（不写盘）

### 3.2 SSE endpoint

#### `GET /api/sessions/:id/stream?action=turn&input=...`

**用途**：单 turn 流式对话（v0.8.3 核心）

**Method**: GET（EventSource 只支持 GET）

**Query params**:
- `action` = `turn`（单 turn）/ `init`（建立连接时拿 initial state）
- `input` = 用户输入（action=turn 时必填）

**SSE events**（每行 `event: <name>\ndata: <json>\n\n`）：

| Event | Data shape | 触发时机 | 谁订阅 |
|---|---|---|---|
| `phase` | `{phase: "WARM_UP", elapsed: 0.0, silence: 0.0}` | turn 开始 / 阶段切换 | SessionPage phase tag |
| `text-chunk` | `{delta: "Hi "}` | LLM stream 每个 chunk | SessionPage 累积 assistant 文本 |
| `tool-call` | `{name: "mark_mistake", args: {...}}` | LLM 输出 `<tool>` 块 | 仅 console debug（UI 不显示）|
| `tool-result` | `{name: "memory_search", result: {...}}` | A+B 2nd-call 前 | 仅 console debug |
| `usage` | `{inputTokens, outputTokens, cacheRead, cacheWrite}` | LLM response done | 仅 debug（show_debug=true 时显示）|
| `segment` | `{phase, last, relevant, active, mistakes}` (v0.7.5 per-segment) | turn 结束 | 仅 debug |
| `error` | `{classification: "rate_limit", message: "..."}` | LLM 错误（v0.7.6 catch-all 不触发；in-turn 错才触发）| SessionPage error UI |
| `done` | `{endedReason: "user_exit" | "mark_ended" | "user_stop"}` | turn 完整结束 | SessionPage 解锁输入框 |

**Response 200** headers:
```
Content-Type: text/event-stream
Cache-Control: no-cache
Connection: keep-alive
X-Accel-Buffering: no  // 防 nginx 缓冲
```

**Edge cases**:
- Session 不存在 → 404（一次性 response，非 SSE）
- 输入为空 → 400（一次性 response）
- action=turn 但 session 已 ended → 200 + SSE `error` event `{classification: "session_ended"}` + `done`

### 3.3 SSE 端到端时序图（v0.8.3）

```
Browser                          Server (Hono)
  |                                |
  |-- GET /api/sessions/abc/stream?action=init -->|
  |                                | (load session, no turn yet)
  |<-- SSE phase WARM_UP ----------|
  |<-- SSE done (initial only) ----|
  |                                |
  | [user types "hi" + Enter]      |
  |-- GET /api/sessions/abc/stream?action=turn&input=hi ->|
  |                                | (runTurn: buildContext → chatStream → parseTool → ...)
  |<-- SSE phase WARM_UP ----------|
  |<-- SSE text-chunk "Hi" --------|
  |<-- SSE text-chunk " there!" ---|
  |<-- SSE done (turn complete) ---|
  |                                |
  | [user types "exit" + Enter]   |
  |-- GET /api/sessions/abc/stream?action=turn&input=exit ->|
  |                                | (runTurn: 'exit' triggers markEnded)
  |<-- SSE phase END --------------|
  |<-- SSE text-chunk "Bye!" ------|
  |<-- SSE done (ended) -----------|
  |                                |
  | [browser: router.push('/')]   |
```

### 4.3 State management

**结论：用 React 19 内置 hooks + 1 个轻量 store，不引 Redux/Zustand**。

- **Server state**: React Query (TanStack Query) 吗？**不引**。4-5 个 endpoint + 简单 cache → `useState` + `useEffect` + 自写 30 行 cache hook `useApi` 足够。**v0.9+ 如果 endpoint 涨到 10+ 再考虑**。
- **Client state**: `useState` + `useReducer`。
- **Settings persistence**: 启动时 `GET /api/settings` → 内存 state → 写时 `PUT /api/settings` + `localStorage` cache。
- **SSE state**: 组件内 `useState<SessionState>`（messages[] / currentPhase / streamingText / ended）。

### 4.4 CSS 策略

**Tailwind v4 zero-config**：
- `web/src/index.css` 1 行 `@import "tailwindcss";`
- 组件内 `className="bg-slate-100 dark:bg-slate-800 ..."`
- v0.8.5 polish 加 dark mode（v0.8.5+ backlog）

**无 CSS-in-JS** / **无 CSS modules** / **无 styled-components** — 纯 utility classes，bundle 最小。

## 5. CLI ↔ Server 共享：turn.ts refactor

### 5.1 当前 cli.ts 结构（v0.7.7）

```ts
// src/cli.ts (current, ~500 lines)
async function main() {
  // 1. 加载 env / DB / DAO / state / tool registry
  // 2. 主循环: while (true) { read input → runTurn → write output }
  //    - runTurn = build context + chatStream + parseTool + execute tool + maybe 2nd-call + summarize at END
  // 3. shutdown
}
```

### 5.2 重构后（v0.8.1）

**新文件 `src/agent/turn.ts`**：
```ts
// src/agent/turn.ts (~200 lines, no I/O)
export interface TurnInput {
  state: SessionState
  history: Message[]
  userInput: string
  env: Env
  deps: { db, llm, tools, ... }  // 显式 DI
}

export interface TurnOutput {
  state: SessionState            // updated
  history: Message[]             // updated
  displayText: string            // what student sees
  events: TurnEvent[]            // for SSE: text-chunk / tool-call / phase / ...
  endedReason?: EndedReason
}

export async function* runTurn(input: TurnInput): AsyncGenerator<TurnEvent, TurnOutput, void> {
  // 1. Apply user input to state
  // 2. Build system context
  // 3. Yield {type: "phase", ...}
  // 4. Stream chat
  // 5. Yield text-chunk events as they come
  // 6. Parse tool call
  // 7. Execute tool (with 2nd-call for A+B tools)
  // 8. Yield final displayText
  // 9. Return TurnOutput
}
```

**`src/cli.ts` 简化** (~50 lines)：
```ts
async function main() {
  const deps = await loadDeps()  // env/db/dao/llm/tools
  // ... REPL loop ...
  for await (const line of readStdinLines()) {
    const gen = runTurn({ state, history, userInput: line, env, deps })
    for await (const event of gen) {
      if (event.type === "text-chunk") stdout.write(event.delta)
      if (event.type === "phase") stderr.write(`[cli] ctx: ${event.phase} ...\n`)
    }
    const result = await gen.next()  // final return
    history = result.value.history
    state = result.value.state
    if (result.value.endedReason) break
  }
}
```

**`src/server.ts` 新建** (~150 lines)：
```ts
import { Hono } from 'hono'
import { runTurn } from './agent/turn.js'
// ...

const app = new Hono()
app.post('/api/sessions', ...)
app.get('/api/sessions', ...)
app.get('/api/sessions/:id', ...)
app.get('/api/sessions/:id/stream', async (c) => {
  // SSE: stream events from runTurn
  return streamSSE(c, async (stream) => {
    const gen = runTurn({...})
    for await (const event of gen) {
      await stream.writeSSE({ event: event.type, data: JSON.stringify(event) })
    }
  })
})
app.get('/api/settings', ...)
app.put('/api/settings', ...)

serve({ fetch: app.fetch, port: Number(env.PORT) })
```

**好处**：
- CLI 和 server 共享 100% 的 turn logic
- turn.ts 是纯函数 + 显式 DI → 容易 L1/L3 测试
- SSE events 自然对应 turn events → 无需额外 transform

**风险**：
- refactor 期间 CLI 可能 regression → **v0.8.1 第一件事**：抽 turn.ts → 跑 CLI 5-turn E2E → 验证没破
- 如果 refactor 暴露 bug → 回滚或修，**不进 v0.8.1.1**

### 6.2 USER.md 原子写

```ts
// src/prompts/loader.ts (新加)
import lockfile from 'proper-lockfile'

export async function updateUserSettings(updates: Partial<UserSettings>): Promise<void> {
  const path = resolvePromptsDir() + '/USER.md'
  const release = await lockfile.lock(path + '.lock', { retries: 3 })
  try {
    const raw = await fs.readFile(path, 'utf8')
    const { data, content } = matter(raw)  // gray-matter
    const newData = { ...data, preferences: { ...data.preferences, ...updates } }
    const newRaw = matter.stringify(content, newData)
    const tmpPath = path + '.tmp.' + Date.now()
    await fs.writeFile(tmpPath, newRaw, 'utf8')
    await fs.rename(tmpPath, path)  // atomic on POSIX, atomic-ish on Windows (NTFS)
  } finally {
    await release()
  }
}
```

**Lock file**：`proper-lockfile` 创建 `USER.md.lock`，防止 CLI 进程和 server 进程同时改。

**Windows atomic rename 限制**：NTFS rename 实际是 atomic 的（since Win Vista），`fs.rename` 在同 volume 上是 atomic。OK。

### 6.3 错误处理

- 写失败（磁盘满 / 权限）→ server 返 500 → 浏览器 Toast "保存失败"
- 读失败（USER.md 不存在）→ server 返 default settings → 浏览器 OK

## 8. Dev / Build / Deploy workflow

### 8.1 开发

```bash
$ pnpm install                    # 根 deps (server + agent + CLI)
$ pnpm --dir web install          # web deps (React 19 + Vite 6 + ...)
$ pnpm dev-web                    # concurrently: server (3000) + vite (5173)
                                  # 浏览器开 5173, vite proxy /api/* → 3000
```

### 8.2 生产

```bash
$ pnpm build                      # 1. vite build → web/dist/  2. tsc → dist/
$ pnpm serve                      # server 启动, listen 3000, serve web/dist/ as static
                                  # 浏览器开 3000, 直接看到 SPA
```

**生产模式 SPA fallback**：`src/server.ts` 启动时 detect `web/dist/index.html` → 用 Hono `app.use('/*', serveStatic({ root: './web/dist' }))` + 404 fallback 到 `index.html`（让 React Router 处理 client-side routing）。

### 8.3 单进程 vs 双进程

**MVP**: 单进程（server 既 serve API 又 serve static）。`pnpm serve` 一个命令搞定。
**未来**: 如果部署到云，可以拆 server / static hosting / CDN。v0.8 不做。
