# Sprint v0.8 Scope — Web UI (Full PRD §11)

> **路线图分隔线**。v0.7.x 收尾（v0.7.7 commit 52ca920）后，v0.8 把 CLI 升级到 Web 浏览器界面。
> **范围**：PRD §11 全集 — 主界面（session 列表）+ session 窗口（流式对话）+ 历史详情（只读 transcript）+ 设置面板（4 控件）。
> **架构决策**（已 user-confirmed 2026-06-11）：
> - **UI 框架**：React 19 + Vite 6 + TypeScript + Tailwind v4
> - **后端架构**：Standalone server — `src/cli.ts` 的 REPL 主体抽出到 `src/agent/turn.ts`，新建 `src/server.ts` 作为 HTTP+SSE 入口；CLI 调用同一个 `turn.ts`
> - **桌面壳**：纯 Web（localhost）；Tauri 不做（per DEV_PLAN §8 风险+Tauri Windows 麻烦）
> - **不做**：v0.9 语音（F6）/ F7 话题库 ≥20 / F8 会话归档 .md 写入 / PRD §9 设置项写回 USER.md 的 4 个**持久化**项留 v0.8.5+ 后续（v0.8.4 只做 UI 控件 + localStorage 临时存储，PRD §9.2 标明 "session-only" 的设置项可以走 localStorage）
> - **拆 5 个 sub-sprint**（v0.8.1-v0.8.5），每个 1-2 天，匹配本项目 "小步快跑" 模式

---

## 1. 总览

v0.1-v0.7.x 全是 headless CLI；学生要看历史 / 设置只能查 `data/oral-teacher.db` 或编辑 `prompts/USER.md`。PRD §11 + F9 明确：v0.8 要给"13 岁小张"一个能自己用的浏览器界面 —— 打开 `http://localhost:5173` → 看到历史列表 → 点 [开始新练习] → 流式对话 → [结束本次] → 回到列表。

**v0.8 做 1 大事**：

把现有的"CLI 单进程 REPL"扩展成"Server + Web UI"双形态，**复用同一 agent 核心**（state machine / tools / memory / summary / 7 seed topics）：

- **后端**：`pnpm serve` 启动 `src/server.ts`（HTTP + SSE），提供 RESTful session 管理 + SSE 流式对话
- **前端**：`pnpm dev`（开发） / `pnpm build` + `pnpm preview`（生产） 启动 Vite，提供 React 19 UI
- **CLI 不退役**：`pnpm dev`（CLI）继续可用（= `src/cli.ts` → 调用 `src/agent/turn.ts` → 直接 process stdin/stdout）
- **共享核心**：所有 agent logic（`src/agent/*`）不动；server.ts 和 cli.ts 都调 `turn.ts`

**用户配置**：
- 新增 `.env` 字段：`PORT`（server 端口，默认 3000）/ `WEB_DIST_DIR`（前端构建产物路径，server 用作静态资源）
- 其余不变

**测试策略**（per DEV_PLAN §4 + §3 v0.8 行）：
- L1 ≥ 5（纯函数/单元）— 组件、API client、turn.ts
- L3 ≥ 3（集成）— server endpoints 端到端、turn 与 server 集成
- **E2E ≥ 2**（Playwright）— 主界面 → session → 回到主；设置面板 → 重启 → 验证持久化

## 3. 用户可配置字段

**新增 2 个 `.env` 字段**：

| 字段 | 默认值 | 说明 |
|---|---|---|
| `PORT` | `3000` | `pnpm serve` 监听端口 |
| `WEB_DIST_DIR` | `./web/dist` | Vite build 产物路径；server 启动时若该目录存在则 serve 静态资源（生产模式）|

**开发态**：`pnpm dev-web` 同时跑 server (3000) + vite (5173)；浏览器开 5173，Vite proxy `/api/*` → 3000。

**生产态**：`pnpm build`（Vite 产物到 `web/dist/`）→ `pnpm serve`（server 启动时 detect `web/dist/`，挂 SPA fallback）→ 浏览器开 3000。

其余 .env 字段（`MINIMAX_API_KEY` / `LLM_CONTEXT_BUDGET_TOKENS` / etc.）不变。

## 4. 关键交互流程

### 4.1 v0.8.3 session 流程（SSE 流式）

```
浏览器: 点 [开始新练习]
  → POST /api/sessions → {id: "abc"}
  → router.push(`/session/abc`)
  → SessionPage mount
  → EventSource(`/api/sessions/abc/stream?phase=init`)
  ← SSE: {event:"phase", data:"WARM_UP"}
  ← SSE: {event:"text-chunk", data:"Hi"}
  ← SSE: {event:"text-chunk", data:" there!"}
  ← SSE: {event:"done", data:{}}

浏览器: 输入 "hi" → [发送]
  → POST /api/sessions/abc/turn {input:"hi"}
  → EventSource 持续收 SSE
  ← SSE: {event:"text-chunk", data:"Hi"}
  ← SSE: {event:"text-chunk", data:" there!"}
  ← SSE: {event:"done", data:{}}

浏览器: 点 [结束本次]
  → POST /api/sessions/abc/turn {input:"exit"}
  ← SSE: {event:"phase", data:"END"}
  ← SSE: {event:"done", data:{}}
  → router.push(`/`)
```

### 4.2 v0.8.4 settings 持久化

```
浏览器: 语音开关 OFF → ON
  → PUT /api/settings {voice_enabled:true}
  → server: 读 USER.md → 改 frontmatter.voice_enabled = true → 原子写 .tmp → rename
  → server: 返回 200 {ok:true}
  → 浏览器: 内存 state 更新 + 写 localStorage (cache)

重启 server:
  → server: loadUserMd() 读 USER.md
  → GET /api/settings → {voice_enabled:true, ...} (来自 frontmatter, 非 default)
  → 浏览器: 渲染 ON
```

### 4.3 CLI 不退役

```bash
$ pnpm dev          # CLI REPL (v0.7.7)
$ pnpm serve        # 新: Web server
$ pnpm dev-web      # 新: server + vite dev (concurrently)
$ pnpm build        # 新: vite build to web/dist/
```

CLI / server / web 三者共享 `src/agent/turn.ts`，**任何 agent logic 改动只需要改一处**。

## 6. 风险与缓解

| 风险 | 触发 | 缓解 |
|---|---|---|
| **大 sprint 失控**：5 sub-sprint × 1-2 天 = 5-10 天；某 sub-sprint 拖延影响全局 | turn.ts refactor 卡住 | v0.8.1 先做（refactor 是其他 sub-sprint 的基础）；每个 sub-sprint 独立可演示 |
| **CORS / proxy 配置问题** | 开发时 vite proxy 错 | v0.8.2 早期就配好，E2E 跑通才算 |
| **SSE 长连接断线** | 网络抖动 / server 重启 | client 端 auto-reconnect (3 次, 1s 退避) |
| **USER.md 原子写并发** | 浏览器和 CLI 同时改 | 写时用 file lock（proper-lockfile, ~3KB），避免 race |
| **Playwright 在 Windows 上慢** | 启动 chromium 慢 | v0.8.2 第一次跑就 baseline，后 sub-sprint 增量 |
| **React 19 生态不稳定** | 某些 lib 还没适配 | 选 lib 时挑 "React 19 compatible"；fallback 用 React 18.3 LTS（API 兼容）|
| **Vite 6 + Tailwind v4 配置踩坑** | Tailwind v4 是新 zero-config | v0.8.2 配好就锁定 |
| **E2E flake** | server / vite 启动时序 | 用 `webServer` 字段（Playwright 自带），等 ready 再 run |
| **i18n 引入成本** | v0.8.5 zh 翻译量大 | MVP: 只 zh + en stub；完整 i18n 留 v0.8.5+ 后续 |
| **PRD F9.4 语音按钮 disabled 但 UI 显示** | 用户期待能点 | UI 显式 disable + tooltip "v0.9+ 可用" |
| **Tauri 不做** | 用户 Windows 桌面体验差 | 纯 Web 127.0.0.1; v0.8.5+ 看反馈决定 |
