## 5. Issues encountered

### 5.1 EPERM on Windows `rmSync(dataDir, { recursive: true })` (L1 server test)

**Symptom**: `EPERM, Permission denied: \\?\C:\Users\samuel\AppData\Local\Temp\server-test-XXXX` thrown by `rmSync` in `afterEach`.

**Root cause**: `createApp()` opens its own `better-sqlite3` handle. If the test opens a second handle on the same file (to verify side effects), Windows file lock prevents cleanup.

**Fix**: Removed the second DB handle from the test harness; verify side effects via the API itself (GET endpoint round-trips the row). The `safeRm` helper in the L3 test also tolerates the EPERM (swallows after 3 retries).

### 5.2 `noImplicitAnyLet` on `let output` (turn.test.ts)

**Symptom**: biome `lint/suspicious/noImplicitAnyLet` on `let output` (no type annotation, no initializer).

**Fix**: Imported `TurnOutput` type from `src/agent/turn.ts` and typed the variable explicitly: `let output: TurnOutput | undefined`. Same fix for the "user says stop" test.

### 5.3 `initState(harness.db)` instead of `initState(harness.clock.now())` (turn.test.ts, first run)

**Symptom**: All 6 turn tests failed with `output.endedReason === 'phase_end'` and `output.state.phase === 'END'` — every test tripped the time-based END.

**Root cause**: I passed `harness.db` (a DbHandle object) to `initState(now: number)`. JS coerces the object to NaN. State machine's `elapsedMin = (now - NaN) / 60_000 = NaN`. `getPhase(NaN)` returns `'END'` (all `elapsedMin < N` comparisons are false).

**Fix**: Use `mockClock(Date.parse('2026-01-01T00:00:00Z'))` so `clock.now()` returns a fixed real number; pass that to `initState`. v0.7.x's existing L3 CLI tests use the same pattern.

### 5.4 Biome `import { MockClock }` not `import type`

**Symptom**: `lint/style/useImportType` — `MockClock` is only used as a type, but biome flagged the import for being non-`import type`.

**Fix**: Changed to `import type { MockClock } from './clock.js'`.
