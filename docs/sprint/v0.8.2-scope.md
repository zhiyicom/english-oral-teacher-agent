## Context

v0.8.1 delivered the **server skeleton**: `src/agent/turn.ts` (shared REPL loop), `src/server.ts` (Hono HTTP + SSE), 3 REST endpoints + 1 SSE stub, 18 new tests (344 total), and `pnpm serve` running on port 3000.

v0.8.2 is the **frontend skeleton** — the half the user actually sees in a browser. Open `http://localhost:5173` → see "English Oral Teacher" + [开始新练习] button + (empty) session list.

Three things must work end-to-end:

1. **Dev orchestration**: `pnpm dev-web` runs server (3000) + Vite (5173) together; Vite proxies `/api/*` → 3000 so the browser never hits CORS.
2. **4 routes with placeholders** + `MainPage` actually wired to `GET /api/sessions`.
3. **Playwright E2E #1** (main page) green.

## 6. Risk register (v0.8.2-specific)

| Risk | Mitigation |
|---|---|
| React 19 + Vite 6 peer-dep mismatch on Windows | Pin versions exactly; verify `pnpm --dir web install` succeeds first |
| Tailwind v4 zero-config surprises | v4 `@tailwindcss/vite` plugin handles config; fallback to inline `style` for one-offs |
| `concurrently` quoting on Windows cmd.exe | Use `concurrently -k -n srv,web` with simple command strings; npm-run-all2 as fallback if needed |
| Playwright chromium download slow first time | One-time `npx playwright install chromium`; document in test report |
| E2E flake on dev-web startup | Playwright `webServer.timeout: 60_000` + `reuseExistingServer: !CI` |
| CORS during dev | Vite proxy `/api` → 3000; browser only sees 5173 |
| TypeScript strict + JSX + DOM lib conflict with server tsconfig | Separate `web/tsconfig.json`; root `tsconfig.json` excludes `web/` |
| Tooling fragmentation (web has its own deps + lockfile) | Acceptable per v0.8-design §2.2; user runs `pnpm --dir web install` once |
