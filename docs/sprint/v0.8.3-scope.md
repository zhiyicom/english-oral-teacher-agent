## 背景

v0.8.2 交付了 **React 骨架**：Vite 6 + React 19 + 4 个路由 + MainPage 接入 `GET /api/sessions`。SessionPage、HistoryPage、SettingsPage 均为占位页面。服务端 SSE 端点（`GET /api/sessions/:id/stream`）是一个仅返回一条 `done` 事件的桩。

v0.8.3 是**关键集成 sprint**——将 SSE 桩替换为真实 `runTurn()` 调用，并构建 SessionPage UI，使学生能在浏览器中进行逐轮对话。

三件事必须端到端跑通：

1. **服务端**：`GET /api/sessions/:id/stream?action=turn&input=...` 调用 `runTurn()`，以 SSE 形式输出事件（phase / ctx / student-text / done / error）。
2. **SessionPage**：阶段标签 + 计时器 + 消息气泡（学生 + 老师）+ 输入框 + 发送按钮 + [结束本次]。
3. **E2E 测试 #2**：Playwright 完成 2 轮会话流程（hi → 回复 → stop → 结束 → 回到主页）。

## 5. 风险

| 风险 | 缓解措施 |
|---|---|
| `runTurn()` 集成破坏现有服务端测试 | 更新 SSE 桩测试为断言 `greeting.json` fixture 的真实事件 |
| 从 DB 重建 SessionState 较脆弱 | 首 turn 使用 `initState()`。后续 turn 使用内存存储。DB 重建仅为兜底方案 |
| EventSource URL 长度限制（长输入）| 对话输入通常 < 500 字符。超出时展示客户端验证错误 |
| Playwright E2E 等待 SSE 响应超时 | Replay fixtures 即时响应（无网络）。15 秒 timeout 足够充裕 |
| 输入框竞态条件（重复发送）| 从发送到 `done` 事件期间锁定输入（disabled）|
| 服务端进程状态泄漏（Map 无限增长）| 会话 30 分钟后自动结束。v0.8.3 暂不清理（顶多几个条目）|
| turn.ts 变更导致 CLI 回归 | 本 sprint 不改 turn.ts — 零回归风险 |
