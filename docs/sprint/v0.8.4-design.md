## 1. 设计决策

### 1.1 messages[] 内嵌在 GET /api/sessions/:id（不新建端点）

按 v0.8-design §3.1 约定，`GET /api/sessions/:id` 响应中包含 `messages[]` 字段。扩展 `toApiSession()` 而非新增独立 `/api/sessions/:id/messages` 端点。一次 HTTP 请求获取全部数据。

### 1.2 设置项：双层存储

| 字段 | 存储 | 原因 |
|---|---|---|
| `voice_enabled` / `voice_speed` / `voice_accent` | USER.md frontmatter | 跨服务端重启持久化 |
| `font_size` / `show_debug` | localStorage | session-only，永久 localStorage |

`PUT /api/settings` 仅处理 voice_* 字段的持久化。font_size 和 show_debug 纯客户端处理。

### 1.3 USER.md 原子写

使用 `proper-lockfile`（已在 v0.8.1 引入）确保并发安全：
1. `lockfile.lock(path + '.lock', { retries: 3 })`
2. 读取当前 USER.md → `gray-matter` 解析
3. 合并新 frontmatter 字段
4. `gray-matter.stringify(body, newData)` → 写入 `.tmp` 文件
5. `fs.rename(tmpPath, path)`（NTFS 上原子操作）
6. `release()`

### 1.4 语音控件全部 disabled

v0.8.4 呈现 voice_enabled 开关、voice_speed 滑块、voice_accent 下拉，但全部标记 `disabled` + tooltip "v0.9+ 可用"。值仍可修改并持久化到 USER.md，为 v0.9 语音功能做准备。

## 3. 服务端变更

### 3.1 扩展 GET /api/sessions/:id（含 messages[]）

扩展现有 handler，添加 messages 查询：

```typescript
app.get('/api/sessions/:id', (c) => {
  const id = c.req.param('id')
  const row = sessions.get(id)
  if (!row) return c.json({ error: 'session not found', id }, 404)

  const msgs = messages.getBySession(id)
  return c.json({
    ...toApiSession({...}),
    messages: msgs.map((m) => ({
      id: m.id,
      role: m.role,
      content: m.content,
      ts: m.ts,
    })),
  })
})
```

### 3.2 新增 GET /api/settings

```typescript
app.get('/api/settings', (c) => {
  // 从 loadUserFile() 读取当前设置，缺失字段回退到默认值
  const { data } = loadUserFile()
  return c.json({
    voice_enabled: data.voice_enabled ?? false,
    voice_speed: data.voice_speed ?? 1.0,
    voice_accent: data.voice_accent ?? 'en-US',
    font_size: 14,    // 默认值，客户端 localStorage 覆盖
    show_debug: false, // 默认值，客户端 localStorage 覆盖
  })
})
```

### 3.3 新增 PUT /api/settings

```typescript
app.put('/api/settings', async (c) => {
  const body = await c.req.json()
  // 仅持久化 voice_* 字段
  const persisted: string[] = []
  const updates: Record<string, unknown> = {}

  if (typeof body.voice_enabled === 'boolean') {
    updates.voice_enabled = body.voice_enabled
    persisted.push('voice_enabled')
  }
  if (typeof body.voice_speed === 'number') {
    updates.voice_speed = body.voice_speed
    persisted.push('voice_speed')
  }
  if (typeof body.voice_accent === 'string') {
    updates.voice_accent = body.voice_accent
    persisted.push('voice_accent')
  }

  if (persisted.length > 0) {
    await updateUserSettings(updates)
  }

  return c.json({ ok: true, persisted })
})
```

### 3.4 `src/prompts/loader.ts` — `updateUserSettings()`

```typescript
import lockfile from 'proper-lockfile'
import matter from 'gray-matter'

const PROMPTS_DIR = join(__dirname, '..', '..', 'prompts')

export async function updateUserSettings(
  updates: Partial<{ voice_enabled: boolean; voice_speed: number; voice_accent: string }>,
): Promise<void> {
  const path = join(PROMPTS_DIR, 'USER.md')
  const release = await lockfile.lock(path, { retries: 3 })
  try {
    let raw: string
    try {
      raw = await fs.readFile(path, 'utf8')
    } catch {
      // USER.md 不存在时从 .example 读取
      raw = await fs.readFile(join(PROMPTS_DIR, 'USER.md.example'), 'utf8')
    }
    const { data, content } = matter(raw)
    const newData = { ...data, ...updates }
    const newRaw = matter.stringify(content, newData)
    const tmpPath = `${path}.tmp.${Date.now()}`
    await fs.writeFile(tmpPath, newRaw, 'utf8')
    await fs.rename(tmpPath, path)
  } finally {
    await release()
  }
}
```

## 4. HistoryPage 组件契约

### 4.1 状态

| 状态 | 触发条件 | UI |
|---|---|---|
| **加载中** | 挂载，等待 fetch | "加载中…" |
| **活跃** | 数据获取成功 | 元数据卡片 + 消息列表 |
| **错误** | fetch 失败 | 错误横幅 + 重试 |

### 4.2 组件树

```
HistoryPage
├── ← 返回（Link 到 /）
├── 元数据卡片
│   ├── 日期 + 时长
│   ├── 摘要
│   ├── 关键词 chips
│   └── 阶段历史 timeline
└── 消息列表（只读，无输入框）
    ├── MessageBubble（学生 — 右对齐蓝色）
    └── MessageBubble（老师 — 左对齐灰色）
```

### 4.3 数据流

```
HistoryPage 挂载
  → getSessionDetail(id)
  → GET /api/sessions/:id（含 messages[]）
  → 渲染元数据卡片 + 消息列表
  → 错误时显示回退 UI
```

## 5. SettingsPage 组件契约

### 5.1 状态

| 状态 | 触发条件 | UI |
|---|---|---|
| **加载中** | 挂载，等待 fetch + localStorage | "加载中…" |
| **活跃** | 数据就绪 | 表单控件 |
| **保存中** | 点击保存后 | 按钮 disabled + "保存中…" |
| **已保存** | PUT 成功 | Toast "已保存"（2 秒后消失）|
| **错误** | fetch/PUT 失败 | 错误横幅 |

### 5.2 控件清单

| 控件 | 类型 | 值域 | 默认值 | 存储 | 状态 |
|---|---|---|---|---|---|
| 语音开关 | toggle | on/off | off | USER.md | **disabled**（v0.9+）|
| 语速 | slider | 0.5–2.0 | 1.0 | USER.md | **disabled**（v0.9+）|
| 口音 | select | en-US / en-GB | en-US | USER.md | **disabled**（v0.9+）|
| 字体大小 | slider | 12–20 | 14 | localStorage | 可用 |
| 显示调试 | toggle | on/off | off | localStorage | 可用 |

### 5.3 数据流

```
SettingsPage 挂载
  → getSettings() → GET /api/settings → 服务端默认值
  → 读取 localStorage → 覆盖 font_size / show_debug
  → 渲染控件

用户修改 → 保存
  → updateSettings({ voice_enabled, ... }) → PUT /api/settings
  → 写入 localStorage（font_size, show_debug）
  → Toast "已保存"
```

### 6.2 `web/src/lib/api.ts`

```typescript
export async function getSessionDetail(id: string): Promise<SessionApi> {
  const res = await checkOk(
    await fetch(`${BASE}/api/sessions/${encodeURIComponent(id)}`),
    'getSessionDetail',
  )
  return (await res.json()) as SessionApi
}

export async function getSettings(): Promise<SettingsApi> {
  const res = await checkOk(await fetch(`${BASE}/api/settings`), 'getSettings')
  return (await res.json()) as SettingsApi
}

export async function updateSettings(
  updates: Partial<SettingsApi>,
): Promise<{ ok: boolean; persisted: string[] }> {
  const res = await checkOk(
    await fetch(`${BASE}/api/settings`, {
      method: 'PUT',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(updates),
    }),
    'updateSettings',
  )
  return (await res.json()) as { ok: boolean; persisted: string[] }
}
```
