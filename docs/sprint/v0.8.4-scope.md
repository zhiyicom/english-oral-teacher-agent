## 背景

v0.8.3 交付了 SSE turn loop + 完整 SessionPage。v0.8.2 留下的两个占位页面是 HistoryPage 和 SettingsPage。v0.8.4 将它们变为真实功能。

两件事必须端到端跑通：

1. **历史详情**：点击会话行 → `/history/:id` → 所有消息以气泡形式渲染 + 元数据（摘要、关键词、日期、时长）
2. **设置持久化**：`/settings` → 切换语音开关 → 保存 → 重启服务端 → 语音开关仍为 ON

## 3. 设置项持久化策略（依据 v0.8-design §6.1）

| 字段 | 存储位置 | 说明 |
|---|---|---|
| `voice_enabled` | USER.md frontmatter | 布尔开关，持久化 |
| `voice_speed` | USER.md frontmatter | 0.5–2.0 滑块，控件 disabled（v0.9+ 启用）|
| `voice_accent` | USER.md frontmatter | en-US / en-GB，控件 disabled（v0.9+ 启用）|
| `font_size` | localStorage | 12–20 滑块，session-only |
| `show_debug` | localStorage | 布尔开关，session-only |

`PUT /api/settings` 仅持久化 voice_* 字段到 USER.md。font_size 和 show_debug 纯客户端 localStorage。

## 6. 风险

| 风险 | 缓解措施 |
|---|---|
| USER.md 原子写在 Windows 上失败 | proper-lockfile + tmp + rename 三层保护；L3 测试验证 |
| USER.md 不存在时首次写入 | `loadUserFile()` 已有回退逻辑（USER.md → USER.md.example）|
| proper-lockfile 锁文件未释放 | lock 操作带 `retries: 3`；`finally` 块确保 release |
| Settings 跨重启 L3 测试需重启 server | 复用现有 L3 `spawnServer()` 模式 |
| E2E settings 测试需要重启 webServer | Playwright 的 webServer 不支持中途重启；改为直接调 server API 验证持久化（绕过 Vite 代理）|
| HistoryPage 长消息列表渲染性能 | 单次会话通常 < 100 条消息，无分页即可 |
| 字体大小变更不实时生效 | localStorage 存储后需刷新页面；v0.8.5 加入 CSS 变量实时生效 |
