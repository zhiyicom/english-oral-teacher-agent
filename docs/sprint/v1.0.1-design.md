## 1. Phase 推进增强

### 1.1 问题

LLM 在阶段切换时不改变行为。`[System Context]` 里有 `Phase: MAIN_ACTIVITY` 但没有行为指令，LLM 继续闲聊。

### 1.2 方案：三层约束

**第一层 — System Context 动态块**（`context-injector.ts`）：
```
## You are in MAIN_ACTIVITY phase (5-25 min). Your task:
- TOPIC SELECTION (mandatory): Check "Active topics" below...
```

**第二层 — 用户消息前缀**（`turn.ts`）：
- 阶段首次切换：注入完整 Context 指令（`[PHASE CHANGE — ## You are in...]`）
- 后续同一阶段内：注入短 Reminder（`[Phase: MAIN_ACTIVITY — pick least-discussed...]`）

**第三层 — 可编辑文件**（`prompts/phases.md`）：
- Context：完整行为指令，注入 System Context 动态块
- Reminder：短指令，注入用户消息前缀

### 1.3 特殊处理

- **END 阶段立即结束**：LLM 告别后，再发任何消息直接 `return`，不调 LLM
- **30 分钟优雅结束**：time-based END 走 LLM 告别轮，`endedReason='phase_end'`
- **catch-up WRAP_UP**：沉默 ≥10min 且跳过 WRAP_UP 时强制补一轮总结

## 2. 话题追踪

### 2.1 问题

数据库 `topics` 表只有 7 个话题（minecraft, school, sports, food, family, movies, music），关键词过少，与 `prompts/topic-library.md` 的分类体系完全不对齐。摘要关键词无法命中。

### 2.2 方案

- **重建 topics 表**：30 个话题，完全对齐 text-library.md 的 A1-A2/B1/B2 分类
- **改良评分**：`max(Jaccard, sharedCount / sessionKeywordCount)`，单关键词也能匹配
- **阈值 0.1**
- **Web UI 编辑器**：`/topics` 路由，GET/PUT /api/topics，保存时自动重新生成 `prompts/topic-library.md`

## 3. 会话记忆

### 3.1 问题

- `lastReview` 服务端启动时一次查询，后续摘要不可见
- Web 会话结束不触发 summarize/markEnded/embedding

### 3.2 方案

- `lastReview` 改为每次新会话首轮动态查询 `loadLastReview(db)`
- Web 会话结束流水线：
  ```
  endedReason 非空
    → summarize(全部消息) → summary + keywords
    → logSummarize() → data/llm-debug/*_summarize.txt
    → extractStudentDiscoveries() → updateUserSettings() → USER.md
    → matchTopic() → topicStats.incrementAndUpdate()
    → embedder.embed() → sessions.setEmbedding()
    → sessions.markEnded()
  ```

## 4. 用户画像自动更新

### 4.1 方案

`profile-extractor.ts` 在每次摘要生成后：
1. 发送摘要给 LLM，请求提取新学生信息
2. LLM 返回 `{new_interests: [], body_update: "..."}`
3. `updateUserSettings()` 合并到 USER.md 的 frontmatter（interests）+ body

## 7. Phase 推进 Round 3（2026-06-16）

### 7.1 问题

前两轮尝试（System Context 指令、Phase Reminder 前缀、PHASE CHANGE 独立消息）均被 DeepSeek chat 模型无视。LLM 始终追随对话流，不响应结构化指令。

### 7.2 解决方案：工具调用代替文本指令

**MAIN_ACTIVITY 进入时自动调用 `topic_select` 工具**（`src/agent/tools/topic-select.ts`），将选中的话题合成为用户消息：

```
[TOPIC: food_drink] We are now in the main activity phase. Introduce the topic
"food_drink" to the student. Teach 2-3 relevant vocabulary words...
```

LLM 被迫响应具体的"任务消息"，而非可忽略的文本指令。`topic_select` 工具通过 `deps.toolRegistry.get('topic_select')` 调用，已注册在服务端 toolRegistry 中。

### 7.3 WARM_UP 独立消息

首轮不再用前缀，而是插入**独立用户消息**，包含完整摘要文本（非仅关键词）：

```
Last session (0 days ago): The student practiced casual conversation about...
Keywords: science, biology, blood, heart
Greet the student first, then make a natural connection to something from
last session before moving to a new topic.
```

策略从"避开上次话题"改为"关联上次话题"。

### 7.4 lastReview 过滤

`loadLastReview` 查询增加 `length(summary) > 30` 条件，跳过 `(summarization failed)` 占位符（22 字符）。

### 7.5 localStorage 双重持久化

设置项不再单独依赖 localStorage（浏览器重启可能清除）。新增服务端存储 `data/preferences.json`，`GET/PUT /api/settings` 读写此文件。`main.tsx` 启动时自动调 `GET /api/settings`，将服务端存储的值恢复到 localStorage。
