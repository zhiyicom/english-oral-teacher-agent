# Sprint v1.0.4 范围 — LLM Prompt 重复清理与可压缩空间优化

> **Sprint**: v1.0.4
> **上一 sprint**: v1.0.3（Web UI 体验打磨 + 阶段化话题策略）
> **分支**: `main`
> **日期**: 2026-06-28
> **状态**: §1 prompt 拼装层清理（in-scope）；§2/§3 视反馈追加

---

## 背景

v1.0.3 sprint 收尾后，对 `data/llm-debug/` 里 968 个 turn 快照做了一次系统抽样（turn1 / turn4 同会话对比 + 全局 Block 0 / Block 1 模板扫描）。结果发现 prompt 模板层存在多处**重复**与**可压缩空间**：

| 现象 | 出现位置 | 量化 |
|---|---|---|
| System Block 0 出现 `# SOUL` 两次、`# AGENTS` / `# AGENTS — operating manual` 双标题 | `src/prompts/loader.ts:109-127` | ~48 字节/turn |
| Block 1 `Last session (Nd ago, Nm min): ... summary ...` 与 Messages[0] user 的 `Last session (Nd ago): ... summary ...` 摘要全文重复 | `src/agent/context-injector.ts:87-95` + `src/agent/turn.ts:436-455` | ~250-400 字节/turn（仅 WARM_UP 第 1 轮）|
| Tool examples（`mark_mistake` / `memory_search` / `summarize_history` / `topic_select` 每个 3-5 个 example 块）每次 turn 全量重发 | `src/prompts/loader.ts`（拼装 TOOLS 段）| **~7 KB/turn × 64 turns = ~450 KB/session** |
| `[System Context]` 的 phase hint（当前阶段 + 任务描述）每 turn 重建 | `src/agent/turn.ts` Block 1 | 280 字节/turn |

其中**最大头是 tool examples**（每 session 省 ~400 KB 流量 + token 计费）；但这部分涉及 cache 策略调整 + LLM 行为验证，风险较大。

**核心约束**（来自用户审查反馈，2026-06-28）：v1.0.4 的目标是"整理和压缩发给 LLM 的最终 prompt"，**但不能影响任何 LLM 回复的效果，不能丢失当前 prompt 的任何语义**。所有改动必须：
- 保留每个关键词、每个时间戳、每段摘要
- LLM 第一句开场 / 对话风格 / 工具调用行为不可观测变化
- 必须有 A/B 验证证据（replay fixture 对比 v1.0.3 baseline LLM 输出）

本 sprint 聚焦**拼装层清洁 + Last session 单一来源**两个低风险改动，作为 prompt 优化基线；tool examples 拆层（§3）列为范围外待评估项。

## 1. 范围内

### 1.1 重构: System Block 0 H1 单一来源

| # | 事项 | 文件 |
|---|---|---|
| 1 | `buildSystemString()` 删除 3 个硬拼 H1：`# SOUL`、`# AGENTS`、`# TOOLS` | `src/prompts/loader.ts:109-127` |
| 2 | `prompts/USER.md` 文件首行（YAML frontmatter 之后）增加 `# STUDENT` H1 —— 让 4 个源文件自身都含 H1，loader 不需要任何特例 | `prompts/USER.md` |
| 3 | `buildSystemString()` 不再硬拼 `# STUDENT`（由 USER.md 自身 H1 承担）| `src/prompts/loader.ts` |
| 4 | 加运行时检查：4 个源文件 trim 后第一行必须是 `# <Name>` 开头，缺失则启动失败并明确报错（防止未来手维护文件忘写 H1 导致输出混乱）| `src/prompts/loader.ts` |
| 5 | 验证：当前所有 prompt 输出（cached + dynamic）的字节数变化、LLM 输入结构无回归 | unit test + A/B replay |

**变更详情**：
- 现状：loader 输出形如 `# SOUL\n\n# SOUL\n\nYou are Alex...`（H1 出现 2 次）；`# AGENTS` / `# AGENTS — operating manual`（措辞不同）；`# TOOLS` / `# Tool Calling`（措辞不同）
- 目标：loader 直接 `sections.push(sp.soul.trim(), '', sp.agents.trim(), '', sp.user, ...)`；每个 H1 只出现 1 次，且由源文件自身提供
- 风险评估：**低**。LLM 已忽略重复 H1；输出文本在语义上等价
- 字节节省：~48 字节/turn × 全 session
- 行为保证：所有源文件 H1 都保留（4 个 H1 中 3 个原本就在源文件，1 个新增到 USER.md）；LLM 看到的语义 100% 一致

**边界情况**：
| 情况 | 行为 |
|---|---|
| 源文件忘了写 H1 | loader 抛错 `[loader] <file> must start with '# <Title>' after trim()` —— 启动失败，不静默输出空白 |
| `tools` 为 null | 保持现状不输出 `# TOOLS` 段（runtime check 只检查 4 个段，tools 是可选）|
| `user` (STUDENT) 字段为空 | runtime check 仍然要求 H1 存在；body 为空也通过 |

**范围外**：
- 把 4 个 H1 改成更简洁的标签（如 `## Role` / `## Mechanics`）—— 风格变化，单独评估
- USER.md frontmatter 改字段（不动，只加 H1）

### 1.2 重构: Last session 摘要单一来源 + keywords 数量对齐

| # | 事项 | 文件 |
|---|---|---|
| 6 | `context-injector.ts` `buildSystemContext()` lastReview 段改造：从 2 行（摘要全文 + 4 个 keywords）改为 1 行（元数据 + 6 个 keywords + pointer）| `src/agent/context-injector.ts:87-95` |
| 7 | `turn.ts` Messages[0] keywords `slice(0, 4)` 改为 `slice(0, 6)` —— **对齐 Block 1 的 6 项**，两处 keyword 列表一致 | `src/agent/turn.ts:438` |
| 8 | Last session 摘要全文保留在 Messages[0]（WARM_UP 首轮 stand-alone user message）—— **LLM 唯一阅读入口** | `src/agent/turn.ts`（保持现状）|
| 9 | Block 1 新格式含 pointer 措辞 `(full summary in opening user message)`，让 LLM 知道全文在 Messages[0] | `src/agent/context-injector.ts` |
| 10 | 当 `lastReview === null` 时 Block 1 不输出空行（保持现状）；Messages[0] 也不注入 hint（保持现状）| — |
| 11 | A/B 验证：replay fixture 跑 v1.0.3 baseline + v1.0.4 新版各 10+ 个 session，对比 LLM 第一句开场 / WARM_UP 行为 / 工具调用模式无回归 | manual smoke + replay fixture diff |
| 12 | 测试：新增 1-2 个 unit 测试覆盖 Last session null / non-null 的 Block 1 字符串组装；keywords 数量断言 | `tests/agent/context-injector.test.ts`, `tests/agent/turn.test.ts` |

**变更详情**：

v1.0.3 现状（turn1 WARM_UP）：
- **Block 1** (`context-injector.ts:87-95`)：
  ```
  - Last session (11 days ago, 25 min): Jeremy had a low-energy session...homework task about naming one of his fish.
  - Last session keywords: short responses, low engagement, free conversation, cello, throat, homework task   ← 6 项
  ```
- **Messages[0]** (`turn.ts:436-455`)：
  ```
  Last session (11 days ago): Jeremy had a low-energy session...homework task about naming one of his fish.
  Keywords: short responses, low engagement, free conversation, cello                                        ← 4 项
  Greet the student first, then make a natural connection...
  ```

v1.0.4 目标：
- **Block 1**（改造）：
  ```
  - Last session (11 days ago, 25 min) — keywords: short responses, low engagement, free conversation, cello, throat, homework task (full summary in opening user message)
  ```
- **Messages[0]**（仅 slice 调整，4 → 6）：
  ```
  Last session (11 days ago): Jeremy had a low-energy session...homework task about naming one of his fish.
  Keywords: short responses, low engagement, free conversation, cello, throat, homework task                    ← 6 项（对齐）
  Greet the student first, then make a natural connection...
  ```

**语义零丢失保证**：
- 摘要全文：Messages[0] 保留（v1.0.3 已有，v1.0.4 不动）
- 6 个 keywords：Messages[0]（v1.0.4 新增 2 项）+ Block 1（v1.0.4 保留全部 6 项）
- duration `25 min`：Block 1 保留（v1.0.3 已有，v1.0.4 不动）
- daysAgo `11 days ago`：两处都保留
- opener 指令（warmUpHook 有/无分支）：Messages[0] 保留
- 唯一删除：Block 1 中摘要全文（但 Messages[0] 仍有全文；**信息无丢失**）

**风险评估**：**低**。所有信息仍存在于 prompt；Block 1 改为 pointer，LLM 能从 Messages[0] 读全文。

**边界情况**：
| 情况 | Block 1 输出 | Messages[0] 注入 | 备注 |
|---|---|---|---|
| `loadLastReview(db) === null`（首次会话）| 不输出 lastSeg | 不注入 hint | 保持现状 |
| Last session 有 summary 但 `keywords: []` | `- Last session (Nd ago, Nm min) — keywords:  (full summary in opening user message)` | 同上（Keywords 行为空字符串）| OK |
| `lastReview.durationMin === null` | `Nm min` → `unknown min`（保持 v1.0.3 fallback）| 同 v1.0.3，无 duration | OK |
| 非 WARM_UP 阶段 | Block 1 仍输出 lastSeg（dynamic，每 turn 重建）| turn.ts 仅 WARM_UP 注入 hint | 保持现状 |
| WARM_UP 但 turn >= 2 | Block 1 输出 lastSeg | turn.ts 不再注入 hint（首轮 only）| 保持现状 |

**范围外**：
- Last session 摘要改用 embedding 检索替代全量注入 —— 涉及新模块，留待 v1.0.5+ 评估
- Last session 摘要 LLM 压缩（缩短到 1-2 句） —— 改变行为，违反"语义零丢失"，明确排除
