## 背景

2026-06-28 practice session `dc50b481` 暴露 LLM 在 MAIN_ACTIVITY 阶段的**话题选择退化**问题：

| 现象 | 数据 |
|---|---|
| 连续不调 `topic_select` tool 的 turn 数 | 4（messages [47][49][51][53]）|
| LLM 凭空编的话题 | "Imagine you could travel anywhere in the world right now" |
| 数据来源 | 直接从 `# STUDENT` interests 抓（folk music、hiking 等）|
| 触发 | WARM_UP 阶段 prompt 主动引导 LLM 用 interests 选开场话题；MAIN_ACTIVITY 阶段无 anti-rule 阻止该行为延续 |
| 用户反应（[54]）| "Can you stop asking about? These kinds of questions." |

**根因**：
1. `topic_select` tool 返回形状只含 `{slug, title: slug, est_minutes, suggested_keyword: 单个英文词}`——LLM 拿到这种"裸数据"没具体内容可用，自然回到 `# STUDENT` 抓现成素材
2. `prompts/phases.md` MAIN_ACTIVITY Context 没有任何"不要从 # STUDENT 抓话题"的禁令

**修复（v1.0.5 范围内 3 段）**：
- §A 提示词禁令（"Do NOT use # STUDENT interests as a topic source"）—— 立即阻止 LLM 抓兴趣当话题
- §B tool 返回拓宽（加 `keywords[]` + `title` 改用 `description`）—— 给 LLM 具体词和中文标题当素材
- §C 默认 30 话题库 seed（`data/topics-default.json` + migration 007 + drift test）—— 让新机部署自带 30 话题而非 7

**为什么不上更大范围**（写 30 个 topic 的 opening/core_questions）—— 评估后**不需要**：LLM 拿到 `keywords[]`（~22 个英文词）已经能自己组开场问题，不需要手写剧本。

**为什么需要 §C**：

- **场景**：用户 2026-06-28 在另一台机器上从 GitHub clone + install，DB 只有 migration 003 的 7 个 starter topics；本地通过 Web UI 编辑器积累到 30 个。**这 23 个差额是 host 本地的运行时数据，不在 repo 里**。
- **修复**：把这 30 个作为项目资产 commit 到 repo + 用 migration 007 在新装时自动 seed。后续在任何主机 `git clone` 都有 30 个。
- **不 gitignore 这 30 个**：与"未来上 GitHub"目标有冲突——但 v1.0.5 的临时决定是把这 30 个作为本项目的固定出厂快照；未来如果发开源 fork，可以替换成通用 30 或拆 `prompts/topic-library.md` 流程（已写入 backlog）。

**数据流**：

```
data/topics-default.json     ← checked-in 30 条（人编辑 / 脚本导出）
       ↓
scripts/export-topics-default.ts  ← 一次跑：DB → JSON + SQL（物理同源）
       ↓
src/storage/migrations/007_topics_default.sql  ← migration 跑一次
       ↓
[新机部署] applyMigrations() → DB 有 30 topics
       ↓
tests/storage/topics-default.test.ts  ← drift test 防止 JSON/SQL 漂移
```

**行为矩阵**：

| 部署前 DB 状态 | 003 跑后 | 007 跑后 | 最终 |
|---|---|---|---|
| 空（新装）| CREATE TABLE only | 30 INSERT OR IGNORE → 30 new | 30 |
| 7 topics（v0.6 老用户 minecraft/school/.../music）| skipped（schema_migrations 已记录）| 30 new（007 名字与 v0.6 无重叠）| 37（用户可手动删 7 个 v0.6 旧名）|
| 30 topics（已编辑过，对应 007 基线名）| skipped | 30 OR IGNORE → 0 new | 30（用户编辑保留）|
| 35 topics（30 基线 + 5 用户自加）| skipped | 30 OR IGNORE → 0 new | 35（用户加的 5 保留）|

**验证**：
- `pnpm typecheck` 0 error
- `pnpm lint` 0 error
- `tests/storage/topics-default.test.ts`：4 个 case（JSON 完整性 / 跑 migrations 后 DB 与 JSON 一致 / 二次跑幂等 / OR IGNORE 保留用户编辑）全部 pass
- 全仓 vitest 400+/434+（基线不变，drift test 全部新增 pass）
- **手动 smoke §C**（建议）：删 `data/oral-teacher.db` → `pnpm serve` → `GET /api/topics` → 返回 30 条 → Web UI 看到 30 条
