**既有测试**（未改动）：
- 14 个老测试中有 13 个继续 pass
- 1 个 fail：`returns { error } when hard exclude removes every topic` (line 83)

**Pre-existing fail 根因分析**：
- 测试 setup：3 个 topic 都在过去 30 天内被讨论过，期望 tool 返回 `{ error: 'No topics available after hard exclude' }`
- 实际：tool 返回了成功结果（其中一个 topic）
- 原因：test 假设 `new Date()` 是 2026-06-10（与 `NOW` 常量对齐），但 vitest 跑测试时 `new Date()` 是实际当前时间 2026-06-28
- 计算：3 个 topic 中 `sports` (20 天前) < cutoff (currentTime - 30 天) → 在 window 内被 exclude
- 但 `minecraft` (5 天前) 和 `school` (10 天前) **不在** 30 天 window 内（因为今天是 6/28，30 天前是 5/29，5/18 和 6/08 都早于 5/29？— 让我重算）
- 实际：今天 2026-06-28，cutoff = 2026-05-29
  - minecraft 5 天前 = 2026-06-23 → 2026-06-23 > 2026-05-29 → **不在** hard-exclude window → 进入 pool
  - school 10 天前 = 2026-06-18 → 2026-06-18 > 2026-05-29 → **不在** window → 进入 pool
  - sports 20 天前 = 2026-06-08 → 2026-06-08 > 2026-05-29 → **不在** window → 进入 pool
- 实际：**3 个都不应该被 hard-exclude**，应该全进 pool 并正常返回 1 个 winner
- 测试 fail 是因为 **fixture 的 stat helper 写 `daysAgo: 20` 期望是 "in 30-day window"，但实际 `Date.now() - 20 days` 是 6/08，不在 30 天 window 内**
- 修复方案（不在本 sprint scope）：把 stat daysAgo 改成 35/40/45 才能确保 hard-exclude 触发

**结论**：pre-existing fail 与 v1.0.5 §B 改动**完全无关**——baseline `d577292` 也 fail。已在 baseline run 确认（`git stash` 后 `pnpm test` 同样 fail）。**v1.0.5 §B 零回归**。
