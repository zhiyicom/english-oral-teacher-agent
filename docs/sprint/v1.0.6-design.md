### 1.1 当前交付形态

- 数据写在 `APP_DATA_DIR` env var（默认平台 AppData）

### 1.2 终端用户 gap

| 步骤 | 开发者 | 终端用户 |
|---|---|---|
| 装 Node.js 22 | 已有 | **没装，不会装** |
| 装 pnpm | 已有 | **不知道 pnpm 是什么** |
| 克隆 repo / 下载 zip | 知道怎么做 | **不知道** |
| `pnpm install` | 一行命令 | **不知道命令** |
| `pnpm dev-web` 启服务 | 一行命令 | **不知道命令** |
| 编辑 `.env` 写 API key | 改文件 | **不会改** |
| 浏览器开 `localhost:3000/` | 知道 URL | **不知道** |

→ **每个步骤都是终端用户的阻塞点**。打包层要解决的就是：**用户只需要双击 .exe + Next + Next + Finish**。

### 1.3 打包目标

## 3. §1.1 — pkg 打包

### 3.1 选型：pkg vs 备选

| 方案 | 体积 | 维护状态 | 备注 |
|---|---|---|---|
| **`@yao-pkg/pkg`** (Vercel pkg 社区 fork) | ~25MB | 活跃 | **选这个** —— Vercel/pkg 已归档但仍可用；社区 fork 持续更新 |
| `pkg` (Vercel 官方) | ~25MB | 归档 | 备选；不再发布新版本 |
| `bun build --compile` | ~40MB | 活跃 | bun 嵌入完整 runtime；需要 bun 工具链（当前无）|
| `nexe` | ~25MB | 低活跃 | 备选 |
| Electron | ~150MB | 极活跃 | 远超 minimal 偏好 |

**决策**：v1.0.6 用 `@yao-pkg/pkg`。理由：活跃维护 + 与现有 Node 22 工具链一致 + 输出体积最小。

### 3.2 better-sqlite3 native binding 处理

**问题**：pkg 默认把 `.node` 文件打进 snapshot，但 Windows native ABI 必须匹配 host 系统的 Node 版本。

**方案**：
1. pkg 不打包 `node_modules/better-sqlite3/`（`--no-bytecode` + `--public` 选项排除）
2. 安装器把 `better_sqlite3.node`（编译好的 binding）作为单独文件复制到 `app/vendor/`
3. server.js 启动时检测 `__dirname/../vendor/better_sqlite3.node` 存在 → `process.dlopen` 手动加载

**简化方案（v1.0.6 选这个）**：
- pkg 用 `assets` 字段把 `node_modules/better-sqlite3/build/Release/*.node` 复制到 snapshot
- 运行时通过 `require('better-sqlite3')` 走 pkg 内置的 fs snapshot；pkg 已支持 .node 文件映射

**验证**：
```bash
# 打包
pnpm build:installer
# 启动
./installer/build/EnglishOralTeacher.exe
# 应能 open SQLite (创建 oral-teacher.db)
ls %APPDATA%\EnglishOralTeacher\oral-teacher.db
```

### 3.3 pkg 配置

`installer/pkg.config.json`：

```json
{
  "name": "EnglishOralTeacher",
  "version": "1.0.6",
  "bin": "dist/server.js",
  "pkg": {
    "scripts": [
      "dist/**/*.js"
    ],
    "assets": [
      "dist/web/**/*",
      "node_modules/better-sqlite3/build/Release/*.node"
    ],
    "targets": [
      "node22-win-x64"
    ],
    "outputPath": "installer/build"
  }
}
```

**关键选项**：
- `bin: dist/server.js` —— 入口；启动 .exe 时直接跑 server
- `targets: node22-win-x64` —— 嵌入 Node 22 + Windows x64
- `assets` —— 静态资源 + native binding；不走 require 解析
- `scripts` —— 仅打包 dist/ 输出；不打包 src/、tests/、docs/

### 3.4 构建脚本

`scripts/build-installer.sh`：

```bash
#!/usr/bin/env bash
set -euo pipefail
cd "$(dirname "$0")/.."

VERSION="${1:?usage: build-installer.sh <version>}"
echo "==> building installer for v$VERSION"

# 1. Build server + web
pnpm install --frozen-lockfile
pnpm build

# 2. Inject version into server bundle
# (Better: src/config/version.ts reads package.json directly)
# For v1.0.6 we hard-code via env var consumed by server.ts
VERSION="$VERSION" pnpm build:copy-web

# 3. pkg
pnpm exec pkg installer/pkg.config.json \
  --options "version=$VERSION" \
  --output installer/build/EnglishOralTeacher.exe

# 4. Verify
file installer/build/EnglishOralTeacher.exe
stat -c '%s bytes' installer/build/EnglishOralTeacher.exe

echo "==> server .exe built: installer/build/EnglishOralTeacher.exe"
```

`scripts/build-node.sh`：


## 5. §1.3 — 启动器 + 自动开浏览器 + 端口变更

### 5.1 启动器实现

**问题**：pkg 嵌入的入口是 `dist/server.js`，本身是 Hono server。直接启动 .exe 即可，不需要单独 launcher .exe。

**启动流程**：
```
用户双击桌面图标 (EnglishOralTeacher.exe)
  → Windows 执行 .exe
  → pkg snapshot 加载 Node 22 runtime
  → 跑 dist/server.js
  → loadEnv() + getAppDataDir() + startServer()
  → listen callback 触发
  → 检测 ENGLISH_ORAL_TEACHER_AUTO_OPEN=1 → exec('start http://localhost:8787/')
  → 浏览器自动开
  → 用户看到 /setup 或主界面
```

### 5.2 自动开浏览器

**方案**：在 server `serve()` callback 里检测 env var，触发 `start <url>`（Windows 命令）。

`src/server.ts` 改动：

```typescript
async function startServer(): Promise<void> {
  const env = loadEnv()
  const dataDir = getAppDataDir()
  const fixturesDir = getSubDir('replay')
  const app = createApp({ dataDir, fixturesDir })
  const port = env.PORT

  serve({ fetch: app.fetch, port }, (info) => {
    const url = `http://localhost:${info.port}`
    console.log(`[server] listening on ${url}`)
    console.log(`[server] data dir: ${dataDir}`)
    console.log(
      `[server] LLM mode: ${process.env.RUN_LIVE_LLM?.trim() === '1' ? 'live' : 'replay'}`,
    )

    // v1.0.6 §1.3 — auto-open browser (installer sets this env var).
    // Skipped in dev / CI / tests.
    if (process.env.ENGLISH_ORAL_TEACHER_AUTO_OPEN === '1') {
      try {
        if (process.platform === 'win32') {
          require('node:child_process').exec(`start "" "${url}"`)
        } else if (process.platform === 'darwin') {
          require('node:child_process').exec(`open "${url}"`)
        } else {
          require('node:child_process').exec(`xdg-open "${url}"`)
        }
      } catch (err) {
        console.warn(`[server] failed to auto-open browser: ${(err as Error).message}`)
      }
    }
  })
}
```

**关键决策**：
- 用 `ENGLISH_ORAL_TEACHER_AUTO_OPEN` env var 门控，**installer 在桌面对应的快捷方式里设这个 env var**（Inno Setup 的 `Icon Environment`）
- dev mode + CI + L3 tests 都不设 → 不自动开浏览器
- 用 `start "" "<url>"` 而不是 `start <url>` —— 前者是正确的 Windows `start` 用法（空标题）

### 5.3 端口 8787 + 冲突检测

**改动**：
- `.env.example` PORT 默认 `3000` → `8787`
- `src/config/env.ts` PORT 默认 `3000` → `8787`
- server 启动时端口被占 → 增强错误信息

`src/server.ts` listen error handler：

```typescript
import { EADDRINUSE } from 'node:constants'

serve({ fetch: app.fetch, port }, (info) => {
  // success callback
}).on('error', (err: NodeJS.ErrnoException) => {
  if (err.code === 'EADDRINUSE') {
    console.error(
      `\n[server] FATAL: port ${port} is already in use.\n` +
      `  To use a different port:\n` +
      `    1. Edit ${join(getAppDataDir(), '.env')} and set PORT=<other>\n` +
      `    2. Restart English Oral Teacher\n` +
      `  Or close the application that is using port ${port}.\n`,
    )
    process.exit(1)
  }
  throw err
})
```

### 5.4 边界情况

| 情况 | 行为 |
|---|---|
| 用户在 RDP session | `start <url>` 在 RDP 内可能不弹出本地浏览器；用户手动复制 URL |
| 浏览器未装 | `start <url>` 失败 → Windows 提示"找不到应用"；用户手动开浏览器输 URL |
| 8787 被占 | 启动失败 + 明确错误（见 §5.3）|
| dev mode（不用 installer）| 不设 `ENGLISH_ORAL_TEACHER_AUTO_OPEN` → 不自动开 |
| server 已运行（重复双击）| 第二次启动失败 → port in use → 用户关闭第一个 |
| 启动后立刻崩 | console 输出错误；用户关窗口；可看 `%APPDATA%\EnglishOralTeacher\logs\` |

### 5.5 测试覆盖

`tests/server/startup.test.ts`：

| Test | 验证 |

## 6. §1.4 — 自动更新检查

### 6.1 GitHub Releases API

**Endpoint**：`GET https://api.github.com/repos/zhiyicom/english-oral-teacher-agent/releases/latest`

**Response**（关键字段）：
```json
{
  "tag_name": "v1.0.7",
  "html_url": "https://github.com/zhiyicom/english-oral-teacher-agent/releases/tag/v1.0.7",
  "name": "v1.0.7 — ...",
  "published_at": "2026-07-XX..."
}
```

### 6.2 src/update-checker.ts

```typescript
const GITHUB_REPO = 'zhiyicom/english-oral-teacher-agent'
const API_URL = `https://api.github.com/repos/${GITHUB_REPO}/releases/latest`
const CACHE_TTL_MS = 24 * 60 * 60 * 1000  // 24h

interface UpdateInfo {
  currentVersion: string
  latestVersion: string | null
  updateAvailable: boolean | null  // null = unknown (API failed)
  releaseUrl: string | null
  checkedAt: string
}

let cache: { info: UpdateInfo; fetchedAt: number } | null = null

export async function checkForUpdate(currentVersion: string): Promise<UpdateInfo> {
  if (cache && Date.now() - cache.fetchedAt < CACHE_TTL_MS) {
    return cache.info
  }

  const info: UpdateInfo = {
    currentVersion,
    latestVersion: null,
    updateAvailable: null,
    releaseUrl: null,
    checkedAt: new Date().toISOString(),
  }

  try {
    const res = await fetch(API_URL, {
      headers: { 'User-Agent': 'EnglishOralTeacher-UpdateChecker' },
      signal: AbortSignal.timeout(5000),
    })
    if (!res.ok) {
      // 403 rate-limited, 404 no releases, etc.
      cache = { info, fetchedAt: Date.now() }
      return info
    }
    const data = await res.json() as { tag_name?: string; html_url?: string }
    if (typeof data.tag_name === 'string') {
      info.latestVersion = data.tag_name.replace(/^v/, '')
      info.releaseUrl = data.html_url ?? null
      info.updateAvailable = compareVersions(currentVersion, info.latestVersion) < 0
    }
  } catch {
    // network error / timeout — return null updateAvailable
  }

  cache = { info, fetchedAt: Date.now() }
  return info
}

function compareVersions(a: string, b: string): number {
  // Simple semver compare: split on '.' and compare numerically.
  const [aMaj, aMin, aPat] = a.split('.').map(Number)
  const [bMaj, bMin, bPat] = b.split('.').map(Number)
  if (aMaj !== bMaj) return aMaj - bMaj
  if (aMin !== bMin) return aMin - bMin
  return (aPat ?? 0) - (bPat ?? 0)
}
```

## 8. §1.6 — /setup 向导

> **原设计来源**：v1.0.5-design.md §6（2026-06-28）。2026-06-30 推迟到 v1.0.6。
> **v1.0.5.3 已落地**：`updateUserSettings` 已支持 name/age/level/goals 字段，
> 因此 `POST /api/setup/profile` 比原设计更简——直接调用 `updateUserSettings`，
> 无需 `writeProfileToUserMd()` 辅助函数。

### 8.1 设计原则

**API Key 加载优先级链**：
1. `process.env.API_KEY`（L3 tests + dev env var injection）
2. `{APP_DATA_DIR}/.env`（wizard 写入路径）
3. `<cwd>/.env`（legacy dev 配置）

**当全部缺失时**：server 启动成功（不抛错），但 `/api/setup/status` 返回 `needsApiKey: true`。Web 端 fetch status 后重定向到 `/setup` 页。

**Profile 加载**：复用 v1.0.5.3 的 `loadUserFile()`。首次运行无 USER.md → 自动 seed → `hasUserProfile: true`。

### 8.2 src/config/secrets.ts

```typescript
import { existsSync, readFileSync, writeFileSync, renameSync } from 'node:fs'
import { join } from 'node:path'
import { getAppDataDir } from './paths.js'

/**
 * v1.0.6 §1.6 — API key resolution priority chain:
 *   1. process.env.API_KEY (test + dev injection)
 *   2. {APP_DATA_DIR}/.env (wizard writes here)
 *   3. <cwd>/.env (legacy dev fallback)
 *
 * Returns null when no source has a non-empty value.
 */
export function getApiKey(): string | null {
  const fromEnv = process.env.API_KEY?.trim()
  if (fromEnv) return fromEnv

  const fromAppData = readEnvFile(join(getAppDataDir(), '.env')).API_KEY
  if (fromAppData?.trim()) return fromAppData.trim()

  const fromCwd = readEnvFile(join(process.cwd(), '.env')).API_KEY
  if (fromCwd?.trim()) return fromCwd.trim()

  return null
}

export function isSetupNeeded(): boolean {
  return getApiKey() === null
}

export function setApiKey(key: string): { persisted: string[] } {
  if (!key.trim()) throw new Error('API key cannot be empty')
  const envPath = join(getAppDataDir(), '.env')

  let current: Record<string, string> = {}
  if (existsSync(envPath)) current = readEnvFile(envPath)

  const persisted: string[] = []
  if (current.API_KEY !== key) {
    current.API_KEY = key
    persisted.push('API_KEY')
  }

  const content =
    Object.entries(current)
      .map(([k, v]) => {
        const needsQuote = /\s|["'=#]/.test(v)
        return needsQuote ? `${k}="${v.replace(/"/g, '\\"')}"` : `${k}=${v}`
      })
      .join('\n') + '\n'

  const tmp = `${envPath}.tmp.${Date.now()}`
  writeFileSync(tmp, content, 'utf-8')
  renameSync(tmp, envPath)
  process.env.API_KEY = key
  return { persisted }
}

function readEnvFile(path: string): Record<string, string> {
  if (!existsSync(path)) return {}
  const content = readFileSync(path, 'utf-8')
  const result: Record<string, string> = {}
  for (const line of content.split('\n')) {
    const trimmed = line.trim()
    if (!trimmed || trimmed.startsWith('#')) continue
    const m = trimmed.match(/^([A-Z_][A-Z0-9_]*)=(.*)$/)
    if (m) {
      const val = m[2].replace(/^["']|["']$/g, '')
      result[m[1]] = val
    }
  }
  return result
}
```

```typescript
// v1.0.6 §1.6 — /setup endpoints. The web UI fetches /setup/status on
// load and redirects to /setup if needsApiKey is true. The wizard
// posts to /setup/api-key and /setup/profile to write to AppData.

app.get('/api/setup/status', (c) => {
  const needsApiKey = isSetupNeeded()
  let hasUserProfile = false
  try {
    const { data } = loadUserFile()
    hasUserProfile = Boolean(
      typeof data.name === 'string' && data.name.length > 0 &&
      typeof data.age === 'number'
    )
  } catch { /* missing = false */ }
  return c.json({
    needsApiKey,
    hasUserProfile,
    appDataDir: getAppDataDir(),
    version: getVersion(),
  })
})

app.get('/api/setup/profile-default', (c) => {
  const { data } = loadUserFile()
  return c.json({
    name: data.name ?? '',
    age: data.age ?? 13,
    level: data.level ?? 'intermediate',
    goals: data.goals ?? [],
    interests: data.interests ?? [],
  })
})

app.post('/api/setup/api-key', async (c) => {
  const body = await c.req.json()
  const key = typeof body.apiKey === 'string' ? body.apiKey.trim() : ''
  if (!key) return c.json({ error: 'apiKey required' }, 400)
  try {
    const { persisted } = setApiKey(key)
    return c.json({ ok: true, persisted })
  } catch (err) {
    return c.json({ error: (err as Error).message }, 500)
  }
})

app.post('/api/setup/profile', async (c) => {
  const body = await c.req.json()
  const name = typeof body.name === 'string' ? body.name.trim() : ''
  const age = Number(body.age)
  const level = body.level
  if (!name) return c.json({ error: 'name required' }, 400)
  if (!Number.isFinite(age) || age < 3 || age > 120) {
    return c.json({ error: 'age must be 3..120' }, 400)
  }
  if (level !== 'beginner' && level !== 'intermediate' && level !== 'advanced') {
    return c.json({ error: 'level must be beginner|intermediate|advanced' }, 400)
  }
  // v1.0.5.3 already extended updateUserSettings with name/age/level/goals.
  // No writeProfileToUserMd() wrapper needed.
  await updateUserSettings({
    name,
    age,
    level,
    goals: Array.isArray(body.goals) ? body.goals.map(String) : [],
    interests: Array.isArray(body.interests) ? body.interests.map(String) : [],
  })
  return c.json({ ok: true })
})
```

```tsx
              headers: { 'Content-Type': 'application/json' },
              body: JSON.stringify(profile),
            })

          } finally { setBusy(false) }

          >
            <option value="beginner">Beginner</option>
            <option value="intermediate">Intermediate</option>
            <option value="advanced">Advanced</option>
          </select>
        </label>
        <label className="block">

            value={profile.goals.join(', ')}
            onChange={(e) => setProfile({ ...profile, goals: e.target.value.split(',').map((s) => s.trim()).filter(Boolean) })}
            className="mt-1 w-full rounded border px-3 py-2"
          />
        </label>
        <label className="block">
          <span className="text-sm text-slate-700">Interests (comma-separated)</span>

            value={profile.interests.join(', ')}

            className="mt-1 w-full rounded border px-3 py-2"
          />
        </label>
        {error && <p className="text-sm text-red-600">{error}</p>}
        <div className="flex gap-2">
          <button
            type="button"
            onClick={() => setStep('apiKey')}
            className="flex-1 rounded border px-4 py-2"
          >Back</button>
          <button
            type="submit"
            disabled={busy}
            className="flex-1 rounded bg-blue-600 px-4 py-2 text-white disabled:opacity-50"

        </div>

  )
```

### 8.7 web/src/App.tsx 改动

```tsx
import { useEffect, useState } from 'react'
import { BrowserRouter, Route, Routes, useNavigate } from 'react-router-dom'
import SetupPage from './components/SetupPage.tsx'

function SetupGate({ children }: { children: React.ReactNode }) {
  const navigate = useNavigate()
  const [checked, setChecked] = useState(false)

  useEffect(() => {
    if (window.location.pathname.startsWith('/setup')) {
      setChecked(true)
      return
    }
    fetch('/api/setup/status')
      .then((r) => r.json())
      .then((s) => {
        if (s.needsApiKey) {
          navigate('/setup', { replace: true })
        }
      })
      .catch(() => { /* network error: let through, may fail later */ })
      .finally(() => setChecked(true))
  }, [navigate])

  if (!checked) return null
  return <>{children}</>
}

export default function App() {
  return (
    <BrowserRouter>
      <SetupGate>
        <div className="flex h-screen bg-white">
          <SessionSidebar />
          <div className="flex-1 overflow-y-auto">
            <Routes>
              <Route path="/" element={<WelcomePage />} />
              <Route path="/session/:id" element={<SessionPage />} />
              <Route path="/history/:id" element={<HistoryPage />} />
              <Route path="/settings" element={<SettingsPage />} />
              <Route path="/topics" element={<TopicLibraryPage />} />
              <Route path="/setup" element={<SetupPage />} />
            </Routes>
          </div>
        </div>
      </SetupGate>
    </BrowserRouter>
  )
}
```

**为什么用 gate 而不是 redirect-only**：
- 避免 race condition（routes 已 render 时还没收到 status）
- 用户直接访问 `/setup?key=xxx` 不被 gate 拦截
- Gate 失败（network error）时不强制 redirect，让用户看到 SPA（LLM 调用时再报错）

### 8.8 边界情况

| 情况 | 行为 |
|---|---|
| 全部 API_KEY 来源缺失 | server 启动 OK；`/api/setup/status` 返回 needsApiKey=true；UI 跳 `/setup` |
| `process.env.API_KEY` 已设 | `getApiKey()` 直接返回；UI 不跳 `/setup` |
| AppData/.env 有 API_KEY 但 LLM 401 | 用户回 `/setup` 重输；setApiKey 覆盖写入 |
| Profile 部分字段缺 | POST 400，SetupPage 显示 inline error |
| 用户在 /setup 中途关闭浏览器 | 状态不持久化；下次启动 /setup 又走一遍 |
| `API_KEY=` （空字符串）| `getApiKey()` 视为缺失（trim 后空）|
| API key 含特殊字符（`#`、`=`、`"`）| `setApiKey` 自动加引号转义 |
| 多个进程同时写 AppData/.env | 后写者覆盖（无 lock）；INSTALLER + DEV 不应同时跑 |
| `API_KEY` env var 已设但用户想在 UI 改 | UI 不暴露编辑（env var 优先级最高）；要改需 unset env + restart |
