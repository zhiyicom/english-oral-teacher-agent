## 3. §1.1 — pkg 打包（实际实施）

### 3.1 选型：`@yao-pkg/pkg` v6

最终选型 `@yao-pkg/pkg`（Vercel pkg 社区 fork），原因：活跃维护 + Node 22 支持。

### 3.2 ESM → CJS 方案

pkg 无法解析 ESM package.json exports（如 `hono/streaming`）。解决方案：

1. **esbuild** 将 TypeScript/ESM 全量打包为单个 CJS 文件
2. **patch-bundle.cjs** 对 CJS bundle 做 5 项后处理
3. **pkg --public** 将 CJS 编译为 .exe

```
pnpm build → esbuild → patch-bundle.cjs → pkg --public → .exe
```

### 3.3 esbuild 命令

```bash
pnpm exec esbuild dist/server.js --bundle --platform=node --target=node22 \
  --format=cjs --outfile=dist/server-bundle.cjs --packages=external \
  --loader:.md=text --loader:.example=text
```

关键选项：
- `--format=cjs` — 输出 CommonJS，消除 ESM import.meta.url 问题
- `--loader:.md=text` — 将 prompt .md 文件内联为字符串（替代运行时 readFileSync）
- `--packages=external` — better-sqlite3 等 native module 保持外部引用

### 3.4 patch-bundle.cjs 五项修复

| # | 修复 | 原因 |
|---|---|---|
| 1 | `var import_meta = {}` → `{ url: require("url").pathToFileURL(__filename).href }` | esbuild CJS 输出 import.meta 为空 |
| 2 | PROMPTS_DIR / SUMMARIZER_PROMPT_PATH → `dirname(__filename)` | pkg VFS 下 `fileURLToPath(import.meta.url)` 返回空 |
| 3 | SQL migrations 内联到 `applyMigrations()` | pkg 不追踪 readdirSync/readFileSync |
| 4 | `dist/web/` 文件 base64 编码为 `WEB_ASSETS` 对象 | pkg 不追踪非 JS 文件的 readFileSync |
| 5 | `/assets/*` + `/*` SPA handler 替换为 embedded-first | 优先从 WEB_ASSETS 内存取，filesystem 仅 fallback |

### 3.5 WEB_ASSETS 嵌入

pkg 的 `--public` 包含 package.json 到 VFS，但**不追踪** `readFileSync` 调用。SPA 的 index.html / CSS / JS 无法从 VFS 读取。

解决：patch-bundle.cjs 遍历 `dist/web/` 目录，将所有文件 base64 编码，注入为全局 `WEB_ASSETS` 对象 + `serveWebAsset(path)` 函数。SPA handler 优先从内存取，仅在开发模式 fallback 到 filesystem。

### 3.6 better-sqlite3 native binding

`--public` 标志将 `package.json` 包含进 VFS，使 `bindings` npm 包的 `getRoot()` 能找到 `package.json`。`node24-win-x64` target 确保内嵌 Node ABI (v137) 与本地编译的 .node 匹配。

### 3.7 安全措施

打包前移除 `.env` 和 `data/` 目录，防止密钥泄露到 VFS。
```
mv .env .env.bak && mv data data.bak
# ... build ...
mv .env.bak .env && mv data.bak data
```

### 3.8 输出

| 产物 | 大小 | 用途 |
|---|---|---|
| `EnglishOralTeacher.exe` | ~102MB | 直接运行 |
| `EnglishOralTeacher-Setup-v1.0.6.exe` | ~37MB | Windows 安装器 |

## 5. §1.3 — 启动器 + 自动开浏览器 + 端口变更

### 5.1 启动器实现

**问题**：pkg 嵌入的入口是 `dist/server.js`，本身是 Hono server。直接启动 .exe 即可，不需要单独 launcher .exe。

**启动流程**：
```
用户双击桌面图标 (EnglishOralTeacher.exe)
  → Windows 执行 .exe
  → pkg snapshot 加载 Node 22 runtime
  → 跑 dist/server.js
  → loadEnv() + getAppDataDir() + startServer()
  → listen callback 触发
  → 检测 ENGLISH_ORAL_TEACHER_AUTO_OPEN=1 → exec('start http://localhost:8787/')
  → 浏览器自动开
  → 用户看到 /setup 或主界面
```

### 5.2 自动开浏览器

**方案**：在 server `serve()` callback 里检测 env var，触发 `start <url>`（Windows 命令）。

`src/server.ts` 改动：

```typescript
async function startServer(): Promise<void> {
  const env = loadEnv()
  const dataDir = getAppDataDir()
  const fixturesDir = getSubDir('replay')
  const app = createApp({ dataDir, fixturesDir })
  const port = env.PORT

  serve({ fetch: app.fetch, port }, (info) => {
    const url = `http://localhost:${info.port}`
    console.log(`[server] listening on ${url}`)
    console.log(`[server] data dir: ${dataDir}`)
    console.log(
      `[server] LLM mode: ${process.env.RUN_LIVE_LLM?.trim() === '1' ? 'live' : 'replay'}`,
    )

    // v1.0.6 §1.3 — auto-open browser (installer sets this env var).
    // Skipped in dev / CI / tests.
    if (process.env.ENGLISH_ORAL_TEACHER_AUTO_OPEN === '1') {
      try {
        if (process.platform === 'win32') {
          require('node:child_process').exec(`start "" "${url}"`)
        } else if (process.platform === 'darwin') {
          require('node:child_process').exec(`open "${url}"`)
        } else {
          require('node:child_process').exec(`xdg-open "${url}"`)
        }
      } catch (err) {
        console.warn(`[server] failed to auto-open browser: ${(err as Error).message}`)
      }
    }
  })
}
```

**关键决策**：
- 用 `ENGLISH_ORAL_TEACHER_AUTO_OPEN` env var 门控，**installer 在桌面对应的快捷方式里设这个 env var**（Inno Setup 的 `Icon Environment`）
- dev mode + CI + L3 tests 都不设 → 不自动开浏览器
- 用 `start "" "<url>"` 而不是 `start <url>` —— 前者是正确的 Windows `start` 用法（空标题）

### 5.3 端口 8787 + 冲突检测

**改动**：
- `.env.example` PORT 默认 `3000` → `8787`
- `src/config/env.ts` PORT 默认 `3000` → `8787`
- server 启动时端口被占 → 增强错误信息

`src/server.ts` listen error handler：

```typescript
import { EADDRINUSE } from 'node:constants'

serve({ fetch: app.fetch, port }, (info) => {
  // success callback
}).on('error', (err: NodeJS.ErrnoException) => {
  if (err.code === 'EADDRINUSE') {
    console.error(
      `\n[server] FATAL: port ${port} is already in use.\n` +
      `  To use a different port:\n` +
      `    1. Edit ${join(getAppDataDir(), '.env')} and set PORT=<other>\n` +
      `    2. Restart English Oral Teacher\n` +
      `  Or close the application that is using port ${port}.\n`,
    )
    process.exit(1)
  }
  throw err
})
```

### 5.4 边界情况

| 情况 | 行为 |
|---|---|
| 用户在 RDP session | `start <url>` 在 RDP 内可能不弹出本地浏览器；用户手动复制 URL |
| 浏览器未装 | `start <url>` 失败 → Windows 提示"找不到应用"；用户手动开浏览器输 URL |
| 8787 被占 | 启动失败 + 明确错误（见 §5.3）|
| dev mode（不用 installer）| 不设 `ENGLISH_ORAL_TEACHER_AUTO_OPEN` → 不自动开 |
| server 已运行（重复双击）| 第二次启动失败 → port in use → 用户关闭第一个 |
| 启动后立刻崩 | console 输出错误；用户关窗口；可看 `%APPDATA%\EnglishOralTeacher\logs\` |

### 6.2 src/update-checker.ts

```typescript
const GITHUB_REPO = 'zhiyicom/english-oral-teacher-agent'
const API_URL = `https://api.github.com/repos/${GITHUB_REPO}/releases/latest`
const CACHE_TTL_MS = 24 * 60 * 60 * 1000  // 24h

interface UpdateInfo {
  currentVersion: string
  latestVersion: string | null
  updateAvailable: boolean | null  // null = unknown (API failed)
  releaseUrl: string | null
  checkedAt: string
}

let cache: { info: UpdateInfo; fetchedAt: number } | null = null

export async function checkForUpdate(currentVersion: string): Promise<UpdateInfo> {
  if (cache && Date.now() - cache.fetchedAt < CACHE_TTL_MS) {
    return cache.info
  }

  const info: UpdateInfo = {
    currentVersion,
    latestVersion: null,
    updateAvailable: null,
    releaseUrl: null,
    checkedAt: new Date().toISOString(),
  }

  try {
    const res = await fetch(API_URL, {
      headers: { 'User-Agent': 'EnglishOralTeacher-UpdateChecker' },
      signal: AbortSignal.timeout(5000),
    })
    if (!res.ok) {
      // 403 rate-limited, 404 no releases, etc.
      cache = { info, fetchedAt: Date.now() }
      return info
    }
    const data = await res.json() as { tag_name?: string; html_url?: string }
    if (typeof data.tag_name === 'string') {
      info.latestVersion = data.tag_name.replace(/^v/, '')
      info.releaseUrl = data.html_url ?? null
      info.updateAvailable = compareVersions(currentVersion, info.latestVersion) < 0
    }
  } catch {
    // network error / timeout — return null updateAvailable
  }

  cache = { info, fetchedAt: Date.now() }
  return info
}

function compareVersions(a: string, b: string): number {
  // Simple semver compare: split on '.' and compare numerically.
  const [aMaj, aMin, aPat] = a.split('.').map(Number)
  const [bMaj, bMin, bPat] = b.split('.').map(Number)
  if (aMaj !== bMaj) return aMaj - bMaj
  if (aMin !== bMin) return aMin - bMin
  return (aPat ?? 0) - (bPat ?? 0)
}
```

## 8. §1.6 — /setup 向导（实际实施）

> 实施日期：2026-07-01 ~ 2026-07-02。基于 v1.0.5-design.md §6 设计稿，实际实现有显著扩展。

### 8.1 实施范围

比原设计多出的内容：
- **LLM 配置字段**：Base URL + 模型（原设计仅 API Key）
- **Live LLM 开关**：checkbox 控制 `RUN_LIVE_LLM`
- **LLM client 动态重初始化**：setup 保存后立即将 replay client 切换为 live client
- **getEnvVar() 三层读取**：process.env → AppData/.env → cwd/.env（`selectClient` 用此读取，保证重启后生效）
- **模型字段统一**：`LLM_MODEL_MAIN` + `LLM_MODEL_SUMMARIZER` 合并为 `LLM_MODEL`
- **placeholder 预设值**：Base URL 和模型用灰色 placeholder 显示默认值，用户填入后才变黑色

### 8.2 核心设计决策

**API Key + Env Var 持久化**：`src/config/secrets.ts`
- `getApiKey()` / `setApiKey()` — API 密钥专用
- `setEnvVar(key, value)` / `getEnvVar(key)` — 通用 env var 持久化（RUN_LIVE_LLM, ANTHROPIC_BASE_URL, LLM_MODEL）
- 原子写入：tmp file → rename，防止写一半崩溃损坏 .env
- `setEnvVar` 同步更新 `process.env[key]`，保证同进程内即时生效

**LLM Client 动态切换**：
- 启动时默认 replay mode（无 API key）
- POST `/api/setup/api-key` 保存 `RUN_LIVE_LLM=1` 后 → `client = selectClient(env, opts.fixturesDir)` 重建为 Anthropic live client
- PUT `/api/settings` 切换 Live LLM 时同样触发重建
- `selectClient` 使用 `getEnvVar('RUN_LIVE_LLM')` 而非 `process.env`，重启后从 AppData/.env 读回

**Setup 向导 Step 1 字段**：
| 字段 | 类型 | 默认值(placeholder) | 说明 |
|---|---|---|---|
| API Key | password | sk-... | 必填 |
| Base URL | text | https://api.minimaxi.com/anthropic | 灰色 placeholder |
| 模型 | text | MiniMax-M3 | 灰色 placeholder |
| 启用在线 LLM | checkbox | true | 默认勾选 |

function selectClient(env, fixturesDir): LLMClient {
  // 用 getEnvVar 读，覆盖 process.env + AppData/.env + cwd/.env

    return createAnthropicProvider(env)
  }
  // Default: replay mode
  if (!existsSync(fixturesDir)) throw new Error(...)
  return createReplayProvider(fixturesDir)
