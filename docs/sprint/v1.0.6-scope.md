# Sprint v1.0.6 范围 — Windows 一键安装包（pkg + Inno Setup）

> **Sprint**: v1.0.6
> **上一 sprint**: v1.0.5（单进程 + AppData + /setup 向导 — 必先合 main）
> **分支**: `main`
> **日期**: 2026-06-28
> **状态**: 5 项打包/分发层 in-scope；Electron / 代码签名证书采购 / macOS+Linux 安装器 = 后续评估

---

## 背景

v1.0.5 已合 main 后，架构层 4 项适配就绪：单进程 + AppData 隔离 + USER.md 自动 seed + /setup 向导。这些解决了"业务代码怎么打包"的问题，但**打包本身**（Node 22 runtime + 70MB deps 怎么塞进一个 .exe）和**安装器**（怎么让终端用户双击安装）还需要独立 sprint。

目标：用户从 GitHub Releases 下载 `EnglishOralTeacher-Setup-v1.0.6.exe`（Inno Setup 输出），双击 → 一路 Next → 桌面出现 `English Oral Teacher` 快捷方式 → 双击 → 浏览器自动开 `http://localhost:8787/` → /setup 向导 → 输入 API key + 学生信息 → 进入对话界面。**整个流程 0 命令行、0 编辑配置文件、0 Node.js 安装**。

**核心约束**：
- **业务代码零改动**（v1.0.5 已就绪；本 sprint 只动打包层）
- **Node 22 runtime 嵌入**：用户机器不装 Node 也能跑
- **数据 100% 隔离到 AppData**：卸载 / 升级不丢历史
- **不引入 Electron**：保持 minimal 偏好（packge size + 复杂度）
- **未签名 .exe 的 SmartScreen 警告**：CHANGELOG 标注"首次运行需要更多信息 → 仍要运行"；签名证书采购留待后续评估
- **仅 Windows 安装器**：macOS DMG / Linux AppImage 单独评估

**变更详情**：
- pkg 默认会把 better-sqlite3 的 .node 文件打进 snapshot，但 Windows 上 native ABI 必须匹配 host → 运行时加载失败
- **方案**：pkg 不打包 better-sqlite3；运行时通过 `node_modules_path` 解析；安装器把 .node 文件放在 `app/vendor/better-sqlite3.node` + server.js 启动时 `process.dlopen()` 加载
- 简化方案（v1.0.6 选这个）：pkg 配置 `assets` 包含 `node_modules/better-sqlite3/build/Release/*.node`；启动时通过 `require('better-sqlite3')` 走 pkg snapshot fallback

**范围外**：
- bun build / nexe 备选（先 pkg 跑通）
- 把 .exe 拆成多文件（pkg 支持但增加安装复杂度）
- macOS .app bundle（v1.0.7+）

**范围外**：

## 7. 风险与回退

| 风险 | 概率 | 影响 | 回退方案 |
|---|---|---|---|
| pkg 停止维护 / better-sqlite3 兼容性问题 | 中 | 高 | 备选 `@yao-pkg/pkg`（社区 fork）或 bun build --compile |
| Inno Setup 在 windows-latest runner 上安装失败 | 低 | 中 | 本地手动 smoke；CI 仅作安全网 |
| SmartScreen 拦截未签名 .exe | 高 | 中 | README 标注 "首次运行 → 更多信息 → 仍要运行"；后续 D6 解决 |
| better-sqlite3 native binding 与 Windows 版本不匹配 | 中 | 高 | 安装器检测 host arch (x64 vs x86) 选对应 .node；或打包源码 + Windows Build Tools 现场编译（不推荐） |
| Auto-open 浏览器在 RDP / Server Core 环境失败 | 低 | 低 | 仅 `ENGLISH_ORAL_TEACHER_AUTO_OPEN=1` 时启用；用户可手动开 |
| 端口 8787 被占用（罕见）| 低 | 低 | server 启动失败 + 明确错误；用户改 PORT env var |
| GitHub releases API 限速（60 req/h 未认证）| 低 | 低 | 24h 内存缓存；用户手动刷新 SettingsPage |
| Inno Setup 输出 .exe 体积超 50MB | 低 | 低 | pkg 排除 devDeps；确认 size < 35MB |
| 多用户共享一台 Windows（每个用户各自 AppData）| 低 | 无 | 标准 Windows 多用户行为；不解决也不恶化 |
