## 背景

v1.0.6 §1.13（commit `56e1348`，Web 侧 Chrome voice Edge hint）引入了语音识别失败时的提示：

```ts
// web/src/components/VoiceInput.tsx
if (err === 'unknown' || err === 'network') {
  setShowEdgeHint(true)
}
// ...
{showEdgeHint && (
  <span className="text-xs text-amber-600">
    Speech unavailable in Chrome. Try Microsoft Edge.
  </span>
)}
```

实际使用中暴露两个问题：

### Bug A：提示对中国 Edge 用户具有误导性

- Edge 浏览器在中国使用的是 **Azure 中国节点**（`*.azure.cn`），不需要翻墙即可直连
- 但 v1.0.6 代码把 `network`/`unknown` 错误一律归到"用错浏览器"
- 国内版 Edge 用户明明在用 Edge，也看到 "Try Microsoft Edge"，**完全没解决实际问题**
- 用户已反馈（2026-07-06）：在 Edge 下开/关 Clash Verge 时，语音输入时而工作、时而失败，提示信息毫无帮助

### Bug B：布局挤压

- `<VoiceInput>` 内部用 `<></>` 把 `<button>` 和错误提示 `<span>` 平级渲染
- 外层 `SessionPage.tsx:576` 的 `<div className="flex gap-2 border-t pt-3">` 是 flex 布局
- 错误提示直接出现在按钮和 `<textarea>` 之间，**把输入框挤到右边**
- 视觉上很乱，影响正常输入

### 根因分析

W3C Web Speech API spec 定义了 6 个 `SpeechRecognitionErrorEvent.error` 错误码 + Chromium 扩展 4 个。当前实现只粗粒度判了 `unknown`/`network`，把所有网络/云端故障一刀切 = "换浏览器"。

**用户实际遇到的错误分布（推测）**：
- 多数情况 = `network`（云端不可达）—— 国内 Edge 应走 Azure 中国节点，直连即可；不应触发"换浏览器"
- 麦克风权限问题 = `not-allowed` 或 `audio-capture` —— 应明确提示"给权限"而不是"换浏览器"
- 浏览器全局禁用 = `service-not-allowed` —— 应明确提示去设置里开
- 静音 = `no-speech` —— 应提示"没听到声音"而不是"换浏览器"

| # | 事项 | 文件 |
|---|---|---|

| 2 | `VoiceInput` 内部新增私有 `hintForError(err)` 函数，按错误码返回对应文案 | `web/src/components/VoiceInput.tsx` MODIFY |
| 3 | 删除硬编码的 "Speech unavailable in Chrome. Try Microsoft Edge." 文案 | `web/src/components/VoiceInput.tsx` MODIFY |
| 4 | 错误码 → 文案映射表 | 见下表 |

**错误码 → 文案映射**：

| 错误码 | STRINGS key | 中文文案 |
|---|---|---|
| `audio-capture` | `voiceErrorAudioCapture` | 无法访问麦克风 |
| `not-allowed` | `voiceErrorNotAllowed` | 需要麦克风权限 |
| `service-not-allowed` | `voiceErrorServiceNotAllowed` | 浏览器禁用了语音识别 |
| `network` | `voiceErrorNetwork` | 识别服务不可达，请检查网络 |
| `no-speech` | `voiceErrorNoSpeech` | 没有听到声音 |
| `language-not-supported` | `voiceErrorLangNotSupported` | 暂不支持该语言 |
| `unknown` / 其他 | `voiceErrorUnknown` | 语音输入出错，请重试 |

**变更详情**：
- `hintForError` 是 `VoiceInput` 内部纯函数，便于以后写单测
- 完全移除 "Try Microsoft Edge" 文案 —— 用户已经在中国版 Edge 上也能遇到 network 错误，提示"换浏览器"不解决问题
- 如果后续需要国际化（英文版），直接把 STRINGS 改成双 key 即可，组件代码不动

**验证**：
- 启动识别后**不说话** → 显示"没有听到声音"
- 拒绝麦克风权限 → 显示"需要麦克风权限"
- 拔掉网络 / 关掉 Edge STT 访问 → 显示"识别服务不可达，请检查网络"

### 1.2 提示位置移到横线上方居中（解决 Bug B）

| # | 事项 | 文件 |
|---|---|---|
| 5 | `<VoiceInput>` 增加 `onHint?: (msg: string \| null) => void` prop，把错误文案通过 callback 传出，组件本身不再渲染任何提示 | `web/src/components/VoiceInput.tsx` MODIFY |
| 6 | `SessionPage` 新增 `voiceHint: string \| null` state，接收 VoiceInput 的 hint 回调 | `web/src/components/SessionPage.tsx` MODIFY |
| 7 | 输入区容器结构调整：外层 `<div className="border-t pt-3">`（横线在这里），内层 `<div className="flex gap-2">`（按钮 + textarea + send），**上方**插入 `<div className="text-center text-xs text-amber-600 pb-2">{voiceHint}</div>` | `web/src/components/SessionPage.tsx` MODIFY |
| 8 | 启动识别 / 主动重试时调 `onHint?.(null)` 清空旧提示 | `web/src/components/VoiceInput.tsx` MODIFY |

**变更详情**：

旧结构（VoiceInput 内部渲染 hint，会挤压 textarea）：
```tsx
<div className="flex gap-2 border-t pt-3">
  <VoiceInput /> {/* 内部: <button/> <span>hint</span> 平级 */}
  <textarea />
  <button>发送</button>
</div>
```

新结构（hint 提升到 SessionPage，渲染在 border-t 上方居中）：
```tsx
<div className="border-t pt-3">
  {voiceHint && (
    <div className="text-center text-xs text-amber-600 pb-2" data-testid="voice-hint">
      {voiceHint}
    </div>
  )}
  <div className="flex gap-2">
    <VoiceInput onHint={setVoiceHint} />
    <textarea />
    <button>发送</button>
  </div>
</div>
```

**自动消失机制**：
- 沿用 v1.0.6 的 2.5 秒 `setTimeout` —— 但要把清空 hint 也加进去
- `rec.onerror` 内：`onHint?.(hintForError(err)); setTimeout(() => onHint?.(null), 2500)`

**验证**：
- 横线之上（不是行内）显示提示
- 水平居中（`text-center`）
- 输入框不被挤压，宽度正常
- 2.5 秒后自动消失

| 事项 | 原因 |
|---|---|
| Edge 本地离线识别（Windows Speech Platform 桥接）| Edge 当前没有把 `webkitSpeechRecognition` 桥接到 Windows 本地语音识别引擎；属于浏览器侧能力，Web 应用无法绕过 |
| 自建 STT 服务（vosk-browser / whisper.cpp WASM）| 体积 + 加载复杂度，不在本 sprint 范围 |
| 真实浏览器语音输入单测（Vitest + jsdom）| jsdom 不实现 SpeechRecognition；端到端验证靠 manual smoke |
| 移除 Chrome 不支持的 `processLocally` 尝试 | 规范存在但 Chromium 不实现，不需在代码里做防御 |
| 麦克风权限引导横幅（首次进入页面时弹"请允许使用麦克风"）| 单独 UX 改动，v1.0.8+ 评估 |
