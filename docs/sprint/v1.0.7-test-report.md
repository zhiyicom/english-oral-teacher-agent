## 4. 代码审查要点（自审）

### 4.1 `web/src/components/VoiceInput.tsx`

**新增导出**：私有 helper `hintForError(err: string): string`，switch 表覆盖 6 个错误码 + default。

**Prop 扩展**：`onHint?: (msg: string | null) => void`（optional，保持向后兼容）。

**状态清理**：删除 `useState` 中的 `showEdgeHint`，JSX 不再渲染行内 `<span>`。

**生命周期**：
- `start()` 开头：`onHint?.(null)` —— 用户主动重试时清旧提示
- `rec.onerror`：`onHint?.(hintForError(err))` —— 错误时传出细分文案
- `setTimeout` 2.5s 后：`onHint?.(null)` —— 自动消失

### 4.2 `web/src/components/SessionPage.tsx`

**State 新增**：`const [voiceHint, setVoiceHint] = useState<string | null>(null)`（紧挨 `voiceEnabled/Speed/Accent`）。

**结构调整**：
```tsx
// 旧：flex 行 + border-t 在行内
<div className="flex gap-2 border-t pt-3">
  <VoiceInput /> <textarea /> <button />
</div>

// 新：border-t 在外层，hint 单独一行居中，flex 行在下方
<div className="border-t pt-3">
  {voiceHint && (
    <div className="text-center text-xs text-amber-600 pb-2" data-testid="voice-hint">
      {voiceHint}
    </div>
  )}
  <div className="flex gap-2">
    <VoiceInput onHint={setVoiceHint} />
    <textarea />
    <button />
  </div>
</div>
```

**验证**：
- 横线（`border-t`）由外层 div 生成，位置在 hint 上方 ✅
- hint 用 `text-center` 水平居中 ✅
- 内部 flex 行只承担按钮 + 输入框 + 发送按钮的横向布局，textarea 宽度回到 `flex-1` 正常值 ✅
- `data-testid="voice-hint"` 便于后续 E2E 定位 ✅

### 4.3 `web/src/i18n/strings.ts`

7 条字段按错误码一对一映射，集中在 `voiceError*` 命名空间：

```ts
voiceErrorAudioCapture: '无法访问麦克风',
voiceErrorNotAllowed: '需要麦克风权限',
voiceErrorServiceNotAllowed: '浏览器禁用了语音识别',
voiceErrorNetwork: '识别服务不可达，请检查网络',
voiceErrorNoSpeech: '没有听到声音',
voiceErrorLangNotSupported: '暂不支持该语言',
voiceErrorUnknown: '语音输入出错，请重试',
```

**移除**：v1.0.6 §1.13 的 "Speech unavailable in Chrome. Try Microsoft Edge." 文案 —— 全代码 grep 已确认无残留。
