## 1. §1.1 Bracket 形式工具调用识别 + 防御剥离

### 1.1 新增函数（同文件 `tool-parser.ts`）

#### `parseBracketToolCall(response: string): ParsedToolCall | null`

```ts
const BRACKET_TOOL_REGEX = /^\[Tool call:\s*(\w+)\s*\]\s*(\{[\s\S]*?\})\s*$/

export function parseBracketToolCall(response: string): ParsedToolCall | null {
  const match = BRACKET_TOOL_REGEX.exec(response)
  if (!match) return null
  const [, name, jsonStr] = match
  if (!name || !jsonStr) return null
  let parsedJson: unknown
  try {
    parsedJson = JSON.parse(jsonStr)
  } catch {
    // v1.1.1 — bracket form is the LLM's fallback when it forgets <tool>.
    // Don't throw; treat as no parse so the caller can strip-and-display.
    return null
  }
  if (typeof parsedJson !== 'object' || parsedJson === null || Array.isArray(parsedJson)) {
    return null
  }
  return { name, args: parsedJson as Record<string, unknown>, rawMatch: match[0] }
}
```

**与 `parseToolCall` 的关键差异**：

| 维度 | `parseToolCall` | `parseBracketToolCall`（NEW）|
|---|---|---|
| 触发正则 | `<tool>name({...})</tool>` | `^\[Tool call: name\]\s*{...}\s*$` |
| JSON 解析失败 | **throws** `Error('tool args JSON parse failed for ...')` | **返回 null**（v1.1.1 fail-soft）|
| 锚定 | 多行 / 内嵌 / 后缀 | 行首到行尾（`^...$`）—— 防止误匹配闲聊里的 `[Tool call: ...]` 字样 |
| name 字符集 | `\w+` | `\w+`（与 `<tool>` 一致）|

**为什么 JSON 坏时不抛错**：
- bracket form 是 LLM 失误的兜底路径，抛错会让 `turn.ts` 的 try/catch 之外再炸；
- 防御剥离（下一步）只剥 `[Tool call: name]\n{garbage}` 前缀，不需要 parse 成功。

#### `stripBracketToolCall(response: string): string`

```ts
const BRACKET_PREFIX_REGEX = /^\[Tool call:[^\]]*\]\s*(?:\{[\s\S]*?\})?\s*/

export function stripBracketToolCall(response: string): string {
  return response.replace(BRACKET_PREFIX_REGEX, '').trim()
}
```

**正则拆解**：
- `^\[Tool call:` —— 行首锚定
- `[^\]]*\]` —— 匹配 `[Tool call: name]` 整段（含 name 中可能的空格，宽松）
- `\s*` —— 吃掉右括号后的换行 / 空格
- `(?:\{[\s\S]*?\})?` —— **可选** JSON 块（非贪婪，能匹配多行 JSON 也允许缺 JSON）
- `\s*` —— 末尾换行

**边界**：
- 输入 `[Tool call: name]\n{"k":"v"}\n` → 输出 `''`（剥干净）
- 输入 `[Tool call: name]\n{garbage}\n后续文本` → 输出 `后续文本`（即使 JSON 坏也剥）
- 输入 `Hey [Tool call: name]...` → 输出原字符串（`^` 锚定，必须行首）
- 输入 `Hey Jeremy` → 输出 `Hey Jeremy`（无前缀）

`turn.ts:677` 现状：

  // v1.1.1 P0-#2 — bracket-form fallback: when LLM forgets <tool>...</tool>
  // and emits "[Tool call: name]\n{json}" instead, still try to parse it as
  // a real tool call. If parse succeeds, fall through to the normal
  // tool-specific branches below. If parse fails (incl. malformed JSON),
  // strip the bracket syntax defensively so the student never sees the
  // raw "[Tool call: ...]" leak in their UI.
  const bracketParsed = parseBracketToolCall(response)

  } else {

- 不抛错 —— bracket parser 永不 throw

## 2. §1.2 Phase prefix 回显剥除

### 2.1 新增函数（同文件 `tool-parser.ts`）

```ts
const ECHOED_PHASE_PREFIX = /^\[Phase:[^\]]*\]\s*/

export function stripEchoedPhasePrefix(response: string): {
  cleaned: string
  stripped: boolean
} {
  const m = response.match(ECHOED_PHASE_PREFIX)
  if (!m) return { cleaned: response, stripped: false }
  return { cleaned: response.slice(m[0].length).trim(), stripped: true }
}
```

**正则拆解**：
- `^\[Phase:` —— 行首锚定
- `[^\]]*\]` —— 匹配到第一个 `]` 为止（覆盖 `[Phase: MAIN_ACTIVITY — ...\n\n---]` 和 `[Phase: END — ...]` 两种形态）
- `\s*` —— 吃掉右括号后的换行 / 空格

**覆盖的 5 种 observed 格式**（来自 `data/llm-debug/diag-bfc000c5.jsonl`）：

| # | 原始 rawHead | cleaned |
|---|---|---|
| 1 | `[Phase: MAIN_ACTIVITY — CALL \`topic_select\` TOOL ... 25 min\n\n---] I talked to you last time.` | `I talked to you last time.` |
| 2 | `[Phase: MAIN_ACTIVITY — CALL \`topic_select\` TOOL ... 25 min\n\n---] Every day in summer...` | `Every day in summer...` |
| 3 | `[Phase: WRAP_UP — Time remaining: ~1.6 min — Briefly review session highlights, then end in 1-2 sentences] Just chill time then...` | `Just chill time then...` |
| 4 | `[Phase: WRAP_UP — Time remaining: ~1.5 min — Briefly review session highlights, then end in 1-2 sentences] All good, no pressure!...` | `All good, no pressure!...` |
| 5 | `[Phase: END — Time remaining: 0.0 min — NO more questions. End the session now.]\n\nBye Jeremy, have an amazing trip to Europe — see you next time!` | `Bye Jeremy, have an amazing trip to Europe — see you next time!` |

### 3.2 v1.1.1 调整

blocked 分支提前 `return`，跳过 line 959-991：

```ts
} else if (parsed.name === 'topic_select') {
  const tool = deps.toolRegistry.get(parsed.name)
  if (!tool) { ... } else {
    const strippedPrefix = stripToolCall(response, parsed).trim()
    const minAge = getCurrentMinTopicAge()
    const lastUserInput = history[history.length - 1]?.content ?? ''
    const explicitSwitch = isExplicitTopicSwitch(lastUserInput)
    const currentCount = getTopicTurnCount(sessionId)
    let result: TopicSelectResult | { error: string }
    let blocked = false
    if (minAge > 0 && !explicitSwitch && currentCount < minAge) {
      const remaining = minAge - currentCount
      result = {
        error: `Topic too young (${currentCount}/${minAge} user turns on current topic). Stay on this topic for ${remaining} more turn${remaining > 1 ? 's' : ''}. The user has not asked to switch.`,
      }
      blocked = true
      logTurnDiagnostic(sessionId, phaseHistory.length, nextState.phase, {
        ev: 'topic_select_blocked',
        count: currentCount,
        minAge,
        lastUserInput: lastUserInput.slice(0, 80),
      })
    } else {
      // ... existing path: yield tool-call, execute, resetTopicTurnCount, recordAdoptedTopics
    }

    // v1.1.1 P1-#5/#6 — when blocked, short-circuit the 2nd LLM call to
    // avoid both (a) duplicated followup text (strippedPrefix + 2nd-call
    // followup both get pushed into history) and (b) the "复读机" effect
    // where the 2nd-call LLM tends to repeat the previous turn's wording.
    if (blocked) {
      displayText = strippedPrefix || "Let's keep going with this — tell me more."
      yield { type: 'student-text', text: `${displayText}\n\n` }

      deps.messages.append({ sessionId, role: 'assistant', content: displayText })
      yield {
        type: 'tool-2nd-call',
        name: parsed.name,
        argSummary: 'blocked (short-circuit)',
      }
      // Skip the normal 2nd-call path below (lines 959-991).
    } else {
      history.push({ role: 'assistant', content: response })
      const resultText = formatToolResult(parsed.name, result)
      history.push({ role: 'user', content: resultText })
      const followup = await deps.client.chat({ systemBlocks, messages: history })
      logTurnDiagnostic(sessionId, phaseHistory.length, nextState.phase, {
        ev: 'tool_2nd_call_done',
        toolName: parsed.name,
        followupLen: followup.content.length,
        followupHead: followup.content.slice(0, 200),
        followupEchoesPhasePrefix: /^\[Phase:/.test(followup.content.trim()),
        blocked,
      })
      const errOrSlug = 'error' in result ? 'error' : `slug=${result.slug}`
      yield { type: 'tool-2nd-call', name: parsed.name, argSummary: errOrSlug }
      let followupDisplay = followup.content
      const followupParsed = (() => {
        try {
          return parseToolCall(followup.content)
        } catch {
          return null
        }
      })()
      if (followupParsed) {
        followupDisplay = stripToolCall(followup.content, followupParsed)
      }
      if (strippedPrefix) {
        followupDisplay = `${strippedPrefix}\n\n${followupDisplay}`
      }
      yield { type: 'student-text', text: `${followupDisplay}\n\n` }
      history.push({ role: 'assistant', content: followupDisplay })
      deps.messages.append({ sessionId, role: 'assistant', content: followupDisplay })
    }
  }
}
```

### 3.3 关键不变量

- **blocked 时不再 push `response` 进 history**（line 959 原行为）—— 否则 history 仍然含 `<tool>...</tool>` 污染，下次 LLM 看到又糊涂
- **blocked 时不再 yield `'tool-call'` / `'tool-2nd-call-done'` 事件** —— 避免误导 diag 解读（`'tool-2nd-call'` 事件仍 yield，带特殊 argSummary `'blocked (short-circuit)'`）
- **displayText 必非空** —— `strippedPrefix || fallback` 二选一
- **不调 2nd LLM call** —— 节省 token + 消除复读机
- **fallback 句英文** —— 与 system prompt（SOUL.md:16 `speaks English with the student`）一致；未来 i18n 化只需替换这 1 行（T10）
