## 1. §1.1 会话内已选话题硬排除

### 1.1 `filterHardExclude` 新增 `excludeSlugs`

```ts
export function filterHardExclude(
  topics: Topic[],
  stats: TopicStat[],
  excludeDays: number,
  now: Date,
  excludeSlugs?: string[],  // NEW
): Topic[] {
  const cutoff = now.getTime() - excludeDays * 86_400_000
  const slugSet = new Set(excludeSlugs ?? [])
  return topics.filter((t) => {
    if (slugSet.has(t.name)) return false   // per-session dedup
    const stat = stats.find((s) => s.topic === t.name)
    if (!stat || !stat.lastDiscussedAt) return true
    return Date.parse(stat.lastDiscussedAt) < cutoff
  })
}
```

### 1.2 `selectTopic` 加参 + `TopicSelectArgsSchema` 加字段

`selectTopic` opts 加 `excludeSlugs?: string[]`，传给 `filterHardExclude`。

`TopicSelectArgsSchema` 加 `exclude_slugs: z.array(z.string()).optional()`——蛇形命名与既有 `exclude_recent_days` / `context_keywords` 一致。**不写入 tool description**，LLM 不可见。

`execute()` 内 `selectTopic` 调用加 `excludeSlugs: parsed.exclude_slugs`。

### 1.3 `turn.ts` 注入已选 slugs


## 4. §1.4 关键词评分信号强化

### 4.1 `computeFreshRatio`

```ts
export function computeFreshRatio(topic: Topic, keywordStats: KeywordHit[]): number {
  if (topic.keywords.length === 0) return 0
  const hitSet = new Set(
    keywordStats.filter(h => h.hitCount > 0).map(h => h.keyword.toLowerCase()),
  )
  const fresh = topic.keywords.filter(k => !hitSet.has(k.toLowerCase())).length
  return fresh / topic.keywords.length
}
```

- `hitCount > 0` 过滤（与 v1.1.2 §1.6 v2 数据模型一致：缺失 = 0 命中 = fresh）
- 大小写不敏感
- 空 keywords → 0

### 4.2 权重调整

```ts
const COUNT_WEIGHT       = 0.10  // 不变
const KEYWORD_WEIGHT     = 0.10  // 0.05 → 0.10
const FRESH_RATIO_WEIGHT = 0.15  // 新增
// INTEREST_WEIGHT       = 0.50  // 不变
// CONTEXT_WEIGHT        = 0.15  // 不变
// NOISE_RANGE           = 0.05  // 不变
```

新公式：
```
score = -count×0.10 - avgHit×0.10 + freshRatio×0.15 + interest×0.5 + context×0.15 ± 0.05
```

### 4.3 数值验证（生产 DB 7/23 数据）

| Topic | count | freshRatio | avgHit | 旧分 | 新分 |
|---|---|---|---|---|---|
| school_life | 1 | 0.90 | 0.10 | -0.105±0.05 | **+0.030±0.05** |
| gaming | 2 | 0.95 | 0.05 | -0.205±0.05 | **-0.060±0.05** |
| music | 7 | 0.83 | 0.39 | -0.739±0.05 | **-0.614±0.05** |

高新鲜低命中 topic 得分上升；高 count topic 仍靠后。排序更合理。

## 6. §1.6 topic_select 成功路径输出 topic + keyword

### 6.1 现状

`formatToolResult`（`turn.ts:285-291`）当前只输出 title + slug + est_minutes：

```ts
if (name === 'topic_select') {
  // ...error case...
  const r = result as TopicSelectResult
  return `[v076_topic_select_result]\nSelected topic: ${r.title} (slug=${r.slug}, ~${r.est_minutes} min). Start discussing it.`
}
```

`TopicSelectResult` 里 `keywords: string[]` 和 `suggested_keyword` 已由 `pickFreshestKeyword` 计算好（命中次数最低的关键词），但进了 `formatToolResult` 就被丢弃了。

### 6.2 修复后

```ts
if (name === 'topic_select') {
  if (typeof result === 'object' && result !== null && 'error' in result) {
    const r = result as { error: string }
    return `[v076_topic_select_result]\n${r.error}. Pick a topic manually or continue.`
  }
  const r = result as TopicSelectResult
  const kwPreview = r.keywords?.length
    ? r.keywords.slice(0, 12).join(', ') + (r.keywords.length > 12 ? '...' : '')
    : '(no keywords)'
  const angle = r.suggested_keyword
    ? `\nSuggested angle: ${r.suggested_keyword} (least discussed — use this for your opening question).`
    : ''
  return `[v076_topic_select_result]\nSelected topic: ${r.title} (slug=${r.slug}, ~${r.est_minutes} min).\nAll keywords: ${kwPreview}.${angle}`
}
```

输出示例：
```
[v076_topic_select_result]
Selected topic: Sports & health (B2) (slug=sports_health, ~15 min).
All keywords: football, exercise, fitness, hiking, team, compete, training, outdoor, gym, health, stamina, routine.
Suggested angle: hiking (least discussed — use this for your opening question).
```

### 6.3 `pickFreshestKeyword` 验证

已正确实现（`topic-select.ts:87-97`）：
- 缺失的 keyword → `hitCount = 0`（从未命中 = 最新鲜）
- 按 `hitCount` 升序排列，相同时按字母序
- 返回排序后的第一个

**不需修改**，只需在 `formatToolResult` 中使用其结果。
